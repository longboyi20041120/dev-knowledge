# 日报 / 周报 / 月报规则

> 知识采集报表的分周、命名、聚合与链接规范。AI 生成或补建报表时必须遵守。
> 制定于 2026-07-19。

---

## 层级结构

```
日报（每日采集） → 周报（聚合一周日报） → 月报（聚合当月周报）
```

## 分周规则（月内固定分周，周不跨月）

| 周 | 日期 |
|----|------|
| 第1周 | 1 ~ 7 号 |
| 第2周 | 8 ~ 14 号 |
| 第3周 | 15 ~ 21 号 |
| 第4周 | 22 ~ 28 号 |
| 第5周 | 29 号 ~ 月末（2~3 天；2 月平年无第5周） |

不使用 ISO 周（W27 这种），不跨月。

## 命名与位置

全部放在 `raw/网页剪藏/`：

- 日报：`YYYY-MM-DD-日报.md`
- 周报：`YYYY-MM-第N周-周报.md`
- 月报：`YYYY-MM-月报.md`

## 生成条件：没满不建

- **周报**：该周日期范围内的日报全部存在才生成，缺一天都不建。
- **月报**：该月的周报全部生成后才建（第1~5周齐）。
- 当前周/月还没过完 → 等，不提前建。

## 历史缺口回溯补建

- 缺失的历史日报可用公开 API 近似补建：**Hacker News**（Algolia API 按日期查）、**arXiv CS**（按提交日期查）。
- **Reddit r/programming、Dev.to、掘金**的当日热榜无法回溯，板块标注"（历史数据无法回溯）"。
- 补建日报顶部必须标注：
  ```
  <!-- backfilled: 补建日期 — 由 HN Algolia / arXiv API 历史数据回溯补建 -->
  ```
- 示例：[[2026-06-22-日报]]、[[2026-06-23-日报]]

## 链接规范（保证 Graph View 连通，无孤点）

- 周报头部引用块加一行，`[[]]` 链接该周全部日报：
  `> 日报: [[YYYY-MM-DD-日报]] · [[...]] · ...`
- 月报头部同理链接当月全部周报：
  `> 周报: [[YYYY-MM-第1周-周报]] · ...`
- 独立剪藏（如 [[2026-06-22-GitHub-Trending周报]]）挂到所属周的周报：
  `> 相关剪藏: [[...]]`

## 报表内容格式

1. 顶部标记：`<!-- consolidated: YYYY-MM-第N周 -->`（月报为 `YYYY-MM`）
2. 头部引用块：
   ```
   > 覆盖: YYYY-MM-DD ~ YYYY-MM-DD
   > 来源: N 篇日报 → 去重 X 篇 → 命中 Y 篇
   > 日报: [[...]] · [[...]]
   ```
3. 正文按知识体系分类（见 CLAUDE.md 过滤器），篇数降序；条目格式：
   ```
   - [标题](URL)
     *来源 · YYYY-MM-DD*
   ```
4. 分类之后依次：`## 其他`（未命中知识体系的条目）、`## 已过滤`（指向各日报末尾的明细表，不重复罗列）

## 去重规则

- 同一 URL 在多篇日报出现 → 聚合时只保留一条，日期标**首次出现日**。
- 同一条目可同时出现在多个分类下（多分类展示不算重复），但"命中/去重"统计按唯一 URL 计数。
