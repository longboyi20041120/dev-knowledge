# 数据库设计 — map_web

## 连接信息

- 数据库：`smart_campus`
- 地址：`localhost:3306`
- 用户：`root`
- 密码：见项目 `.env` 文件 `CAMPUS_DB_PASSWORD`
- 字符集：`utf8mb4`

## 核心表

### campus_building（建筑）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键 |
| model_id | varchar | 与 GLB 节点匹配的 ID |
| name | varchar | 正式名称 |
| alias | varchar | 俗称（如"八教"） |
| category | varchar | 分类，逗号分隔表示多分类 |
| description | text | 建筑描述 |

### path_waypoint（路点）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键 |
| x/y/z | double | 3D 坐标 |
| connections | json/varchar | 连接的路点 ID 列表 |

### building_service（建筑服务）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键 |
| building_id | bigint | 关联建筑 |
| name | varchar | 服务名称 |

### canteen_vendor / canteen_item（食堂）

| 表 | 说明 |
|-----|------|
| canteen_vendor | 食堂商户（档口） |
| canteen_item | 菜品 |

### student_course（课程）

学生课表，关联教室（room_name 匹配建筑）。

### sys_user / sys_role / user_role

用户认证和权限（JWT + BCrypt）。

## 关键字段设计

- `campus_building.alias` 存俗称，前端显示为"别名（正式名）"
- `category` 含逗号表示多分类，如"宿舍,食堂"
- `model_id` 与前端的 GLB 模型节点名称匹配
