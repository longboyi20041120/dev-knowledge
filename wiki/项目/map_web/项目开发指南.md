## 项目概述

校园三维导航系统，技术栈：Spring Boot + MyBatis-Plus + Three.js + Python FastAPI AI 服务 + MySQL。

## 项目结构

```
map_web/
├── src/main/java/.../controller/
│   ├── AdminController.java    # 管理员 API（服务/商家/菜品 CRUD）
│   ├── PublicController.java   # 公开 API（服务/食堂查询）
│   ├── AuthController.java     # 登录/注册（JWT + BCrypt）
│   ├── UserController.java     # 用户课程 CRUD
│   ├── CampusBuildingController.java  # 建筑列表
│   ├── PathController.java     # 导航路径
│   └── AiProxyController.java  # AI 代理 → Python :8000
├── src/main/resources/static/1.html  # 主前端（单文件 ~3500 行）
├── ai-service/
│   ├── main.py                 # FastAPI AI 服务 (:8000)
│   └── services/llm_service.py # LLM + 服务数据匹配
├── sql/                        # 数据库迁移 SQL
└── campus_storage              # MySQL 数据库名
```

## 关键文件

| 文件 | 说明 |
|------|------|
| `1.html` | 单文件前端，包含 CSS + HTML + JS（Three.js 3D 引擎 + UI + 业务逻辑） |
| `llm_service.py` | AI 对话服务，服务/食堂查询走代码精确匹配（不走 LLM） |
| `PathServiceImpl.java` | A* 导航算法，156 个路点 |

## 前端布局（1.html）

- **左上**：「建筑目录」按钮 → 悬浮面板（分类筛选、可拖动、可折叠）
- **中上**：搜索栏
- **右上**：天气 + 时间日期（固定位置）
- **左下**：「我的课程」按钮 → 悬浮面板（表格/列表视图、可拖动）
- **中下**：导航栏（起点/终点/路点采集）
- **右下**：AI 悬浮球
- **建筑信息**：点击建筑后出现的独立浮动面板（服务/食堂/今日课程、可拖动）

## 前端关键函数

- `showToast(msg, type)` — 通知提示
- `getToken()` / `authHeaders()` — JWT 认证
- `loadBuildingTypeContent(buildingId)` — 加载建筑服务+食堂+课程
- `loadBuildingServices()` / `loadCanteenContent()` / `loadBuildingTodayCourses()`
- `openBuildingByClassroom(roomName)` — 从课程教室打开建筑
- 管理员 CRUD：`addService/deleteService/saveService/addVendor/deleteVendor/saveVendor/addItem/deleteItem/saveItem`

## 数据库

- 连接：`localhost:3306/campus_storage`，用户 `root`，密码见 `.env` 中 `CAMPUS_DB_PASSWORD`
- 表：`campus_building`、`building_service`、`canteen_vendor`、`canteen_item`、`student_course`、`path_waypoint`、`sys_user`、`sys_role`、`user_role`
- `campus_building.alias` 存俗称（如"八教"），前端显示为 `别名（正式名）`

## 启动命令

```bash
# 后端
mvn spring-boot:run

# AI 服务
cd ai-service && python -m uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

## 重要约定

- 编辑 1.html 后用 `cp` 同步到 `target/classes/static/1.html`
- 改 Python AI 服务后需重启（或使用 `--reload`）
- 建筑分类含逗号（如"宿舍,食堂"）表示同时属于多个分类
- 前端 `h()` 函数用于 HTML 转义，防止 XSS
- `window.isAdmin` 由 `updateAuthUI()` 自动检测

## 1. 编码前先思考

- **不确定就问**：不做假设，明确说出你做了哪些假设，让用户确认。
- **有歧义列选项**：多种理解时列出所有选项，不要默默选一个。
- **该反驳就反驳**：有更简单、更好的方案时主动提出来。
- **评估影响面**：修改前理解代码的上下文和依赖关系。

## 2. 简洁优先

- **没被要求的功能不写**。
- **一次性代码不建抽象层**。
- **没人要的"灵活性"和"可配置性"不加**。
- **不可能发生的场景不做错误处理**。
- **200 行能 50 行搞定的就重写**。

## 3. 精准修改

- 只动必须动的，只清理自己造成的混乱。
- **不要顺手"改进"** 旁边的代码、注释、格式。
- **不要重构没坏的东西**。
- **匹配现有风格**，哪怕你更想用别的写法。
- **改动前先读文件**，确认当前内容。
- 每一行改动都能直接追溯到用户的需求。

## 4. 目标驱动执行

- 多步骤任务先列出计划，每步带验证点。
- 做完后汇报具体改了什么，列出刻意没改的东西。
