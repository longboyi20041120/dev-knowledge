# 自动知识采集工作流

## 一句话

定时从 GitHub Trending、技术博客等高质量源抓取内容，自动消化写入知识库。你只需要看结果。

## 流水线

```
定时触发器 → 采集源抓取 → 写入 raw/网页剪藏/ → AI 检测新文件 → 阅读 → 提取要点 → 写入 wiki/技术栈/
                                                                    ↓
                                                              更新 wiki/索引.md
                                                                    ↓
                                                              你去 Obsidian 看
```

## 高质量采集源

| 源 | URL | 说明 |

## 实现方案

### 方案：Claude Code Cron + WebFetch

最简单、不需要额外部署：

```bash
# 每天上午 10 点自动执行
# Claude Code 的 Cron 功能 + WebFetch 工具
```

流程：
1. Claude Code 定时触发
2. WebFetch 抓取 GitHub Trending / Hacker News 等
3. 提取标题和摘要
4. 对于有价值的项目/文章，深度抓取 README 或全文
5. 生成笔记写入 `raw/网页剪藏/`
6. 自动处理 → 写入 `wiki/技术栈/`
7. 更新 `wiki/索引.md`

### 采集策略（避免噪音）

不是什么都抓，有筛选条件：
- **Star 增速**：24h 内 > 500 star 的仓库才深度分析
- **相关性**：和你技术栈相关的优先（Java/Python/Three.js/Spring Boot）
- **重复过滤**：已经抓过的跳过
- **每日上限**：最多深度分析 3 篇，防止信息过载
