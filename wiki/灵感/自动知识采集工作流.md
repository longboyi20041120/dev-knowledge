# 自动知识采集工作流

## 一句话

定时从 GitHub Trending、技术博客等高质量源抓取内容，自动消化写入知识库。你只需要看结果。

## 流水线

```
定时触发器 → 采集源抓取 → 写入 raw/网页剪藏/ → AI 检测新文件 → 阅读 → 提取要点 → 写入 wiki/技术栈/
                                                                    ↓
                                                              更新 wiki/索引.md
                                                                    ↓
                                                              你去 Obsidian 看
```

## 实现方案

### 方案：Claude Code Cron + WebFetch

最简单、不需要额外部署：

```bash
# 每天上午 10 点自动执行
# Claude Code 的 Cron 功能 + WebFetch 工具
```

流程：
1. Claude Code 定时触发
2. WebFetch 抓取 GitHub Trending / Hacker News 等
3. 提取标题和摘要
4. 对于有价值的项目/文章，深度抓取 README 或全文
5. 生成笔记写入 `raw/网页剪藏/`
6. 自动处理 → 写入 `wiki/技术栈/`
7. 更新 `wiki/索引.md`

### 采集策略（知识体系过滤）

**只采集命中以下领域的内容**，不相关的直接丢弃：

| 领域 | 典型关键词 |
|------|-----------|
| AI/机器学习 | LLM, Agent, Transformer, 深度学习, 神经网络, NLP, 模型训练/推理, Prompt Engineering |
| 编程语言 | Python, Java, C/C++, R, Go, Rust, 编译器, 类型系统, JVM |
| 数据库 | SQL, MySQL, PostgreSQL, NoSQL, HBase, 索引, 查询优化, 窗口函数 |
| 大数据 | Hadoop, Spark, Hive, Flink, Kafka, 数据仓库, ETL, 数据管道 |
| 数据分析/可视化 | pandas, ECharts, matplotlib, BI, 数据可视化, 统计分析 |
| 操作系统/Linux | 进程, 内存管理, 文件系统, 内核, Shell, 系统调用 |
| 软件工程/架构 | 设计模式, 微服务, MVC, SOLID, CI/CD, DevOps, 测试, Git |
| 计算机基础 | 数据结构, 算法, 网络协议, 计算机组成, 编码/字符集 |
| 数学/统计 | 概率论, 线性代数, 微积分, 离散数学, MLE, 贝叶斯 |
| 容器/云原生 | Docker, Kubernetes, 容器化, 编排, 服务网格 |
| Web开发 | HTTP, REST, API设计, Spring, 前后端分离 |

**黑名单（直接丢弃）**：
- 个人博客（职业迷茫、心情分享、"我的旅程"、"I am behind"）
- 非技术新闻（政策社论、讣告、考古、历史）
- 纯自然科学（物理/化学/生物，与 CS 无关的）
- 硬件外设（键盘、显示器、鼠标测评）
- 游戏（非编程/教育类）
- 元内容（"本周精选"、"Top 10" 汇总帖）

**其他筛选条件**：
- **Star 增速**：24h 内 > 500 star 的仓库才深度分析
- **重复过滤**：已经抓过的跳过
- **每日上限**：最多深度分析 3 篇，防止信息过载
