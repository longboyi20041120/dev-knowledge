# Elasticsearch 基础

> 分布式全文搜索引擎，倒排索引是核心。面试必问：倒排索引原理、IK 分词、DSL 语法。 | **面试重要度：高** | 预计阅读：15 分钟

## 一、ES 是什么，解决什么问题

- **定位**：分布式全文搜索引擎
- **核心优势**：
  - **倒排索引 + 分词技术** → 高效全文检索（搜"鱼皮教程"能匹配"鱼皮的 ES 入门教程"）
  - **分布式架构** → 海量数据存储 + 高并发查询
- **检索效率高于 MySQL 的模糊查询**（`LIKE %x%` 无法用索引，ES 用倒排索引）

**为什么快？** 倒排索引 + 内存缓存 + 分布式并行。

## 二、核心概念

| ES 概念 | 类比 MySQL | 说明 |
|---------|-----------|------|
| 索引（Index） | 表 | 一类文档的集合 |
| 文档（Document） | 行 | JSON 格式的一条数据 |
| 字段（Field） | 列 | 支持 `text`（分词）/ `keyword`（不分词）类型 |

**中文分词**：推荐 **IK 分词器**——`ik_max_word`（建索引时细粒度切）、`ik_smart`（搜索时粗粒度切）。

## 三、常用 DSL 查询语法

| 语法 | 作用 |
|------|------|
| `match` | 全文检索（自动分词） |
| `term` | 精确匹配（不分词） |
| `bool` | 组合条件（`must` / `should` / `must_not`） |
| `range` | 范围查询（时间、数值） |

**实用特性**：
- **高亮显示**：命中关键词用 `<em>` 标签包裹
- **相关性排序**：**BM25 算法**（按词频 / 文档长度 / 稀有度算分）
- **聚合分析**：类似 `GROUP BY`，支持统计、求和

## 四、生产环境实践

### 数据同步（MySQL → ES）

| 方案 | 特点 | 适用 |
|------|------|------|
| 定时任务 | 简单，但有延迟 | 实时性要求低 |
| 双写 | 实时，但影响写入性能 | 一般场景 |
| **Canal** | 监听 MySQL Binlog，实时同步 | 高实时性场景 |

### 集群部署

- 至少 **3 节点**（主节点 + 数据节点）
- 启用**分片（Shard）+ 副本（Replica）**保障高可用
- 合理配置分片数（如 5 分片 + 1 副本）

## 五、应用场景（超出常规认知）

- **搜索引擎**：自带 DSL 查询语言，可直接搭建
- **数据库 / 监控存储**：可存监控指标数据，**SkyWalking、ELK** 等监控系统均采用
- **日志分析**：ELK（Elasticsearch + Logstash + Kibana）经典组合

ES 兼具**高性能检索**与**数据存储**能力。

## 六、学习建议

- 用 **Kibana Dev Tools** 练 DSL 查询
- 结合项目需求（日志分析、商品搜索、埋点数据分析）实战
- 核心问题驱动记忆：为什么快？写入/查询流程？倒排索引原理？
