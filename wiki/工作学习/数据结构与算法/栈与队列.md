
## 真题 1：有效的括号（LeetCode 20）

面试题里最简单的，但写不出来的大有人在。

**题目**：`s = "()[]{}"` → `true`，`s = "(]"` → `false`

```python
def is_valid(s: str) -> bool:
    pairs = {")": "(", "]": "[", "}": "{"}
    stack = []

    for ch in s:
        if ch in "([{":
            stack.append(ch)           # 左括号入栈
        else:                          # 右括号
            if not stack:              # 栈为空，没有能配对的
                return False
            if stack.pop() != pairs[ch]:  # 不匹配
                return False

    return len(stack) == 0  # 栈必须为空（所有括号都配对了）

# 验证
print(is_valid("()[]{}"))  # True
print(is_valid("([)]"))    # False — 嵌套顺序不对
print(is_valid("{[]}"))    # True — 正确的嵌套
```

### 变体：最长有效括号（LeetCode 32）

**题目**：`s = ")()())"`，最长有效括号子串是 `"()()"`，长度 4。

```python
def longest_valid_parentheses(s):
    # 栈里存索引，栈底始终保持"最后一个未匹配右括号的位置"
    stack = [-1]  # 初始哨兵
    max_len = 0

    for i, ch in enumerate(s):
        if ch == '(':
            stack.append(i)
        else:  # ch == ')'
            stack.pop()
            if not stack:
                # 栈空了，说明当前 ')' 没有匹配，把它作为新的"最后一个未匹配位置"
                stack.append(i)
            else:
                # 栈不为空，计算当前有效长度
                max_len = max(max_len, i - stack[-1])

    return max_len

# 验证
print(longest_valid_parentheses(")()())"))   # 4
print(longest_valid_parentheses("(()"))       # 2
print(longest_valid_parentheses(""))          # 0
```

**面试话术**："关键是用栈底存'最后一个未匹配的右括号位置'作为基准。遇到 '(' 索引入栈，遇到 ')' 先 pop，如果栈空了说明当前右括号也没有匹配，把它设为新的基准；如果栈没空，用当前索引减栈顶索引得到一段有效括号的长度。"

## 真题 2：表达式求值

### 逆波兰表达式求值（LeetCode 150）

后缀表达式是栈最经典的应用场景。逆波兰表达式把操作数放前面、运算符放后面。

**题目**：`tokens = ["2", "1", "+", "3", "*"]` → `9`（即 (2+1)×3 = 9）。

```python
def eval_rpn(tokens):
    stack = []

    for token in tokens:
        if token in "+-*/":
            b = stack.pop()  # 注意：先 pop 的是第二个操作数
            a = stack.pop()  # 后 pop 的是第一个操作数

            if token == '+':
                stack.append(a + b)
            elif token == '-':
                stack.append(a - b)
            elif token == '*':
                stack.append(a * b)
            elif token == '/':
                # 除法向零取整（Python 的 // 是向下取整，需要用 int()）
                stack.append(int(a / b))
        else:
            stack.append(int(token))

    return stack[0]

# 验证
print(eval_rpn(["2", "1", "+", "3", "*"]))     # 9
print(eval_rpn(["4", "13", "5", "/", "+"]))    # 6 (4 + 13/5 = 4 + 2 = 6)
print(eval_rpn(["10", "6", "9", "3", "/", "-", "*"]))  # -30
```

### 中缀表达式转后缀表达式（调度场算法）

这是面试中的延伸话题，问到表达式求值时可能会被追问"中缀怎么转后缀"。

```python
def infix_to_postfix(expression):
    """将中缀表达式转为后缀表达式（逆波兰表达式）"""
    precedence = {'+': 1, '-': 1, '*': 2, '/': 2}
    output = []
    operators = []

    for token in expression.split():
        if token.isdigit():
            output.append(token)
        elif token == '(':
            operators.append(token)
        elif token == ')':
            # 弹出直到遇到 '('
            while operators and operators[-1] != '(':
                output.append(operators.pop())
            operators.pop()  # 丢弃 '('
        else:  # 运算符
            while (operators and operators[-1] != '(' and
                   precedence.get(operators[-1], 0) >= precedence.get(token, 0)):
                output.append(operators.pop())
            operators.append(token)

    # 弹出剩余运算符
    while operators:
        output.append(operators.pop())

    return ' '.join(output)

# 验证
print(infix_to_postfix("3 + 4 * 2"))          # 3 4 2 * +
print(infix_to_postfix("( 1 + 2 ) * 3"))      # 1 2 + 3 *
print(infix_to_postfix("3 + 4 * 2 / ( 1 - 5 )"))  # 3 4 2 * 1 5 - / +
```

## 单调栈

### 什么是单调栈？

单调栈，栈内元素始终保持单调递增或递减。它的核心用途：**在一次遍历中，找到每个元素的下一个更大/更小的元素。**

思考模型：栈内元素是一群"还在等待答案的人"。当新元素到来时，比它矮的人终于等到了答案——新元素就是他们的答案。

### 真题 3：每日温度（LeetCode 739）

上一节有完整代码，这里补充一个可以直接用的模板：

```python
def find_next_greater(nums):
    """找每个元素的下一个更大元素（通用单调递减栈模板）"""
    n = len(nums)
    result = [-1] * n  # -1 表示没有更大的
    stack = []  # 存索引，栈底→栈顶对应的值递减

    for i, num in enumerate(nums):
        while stack and num > nums[stack[-1]]:
            prev_idx = stack.pop()
            result[prev_idx] = num  # 或 i - prev_idx（看题目要什么）
        stack.append(i)

    return result

# 验证
print(find_next_greater([2, 1, 2, 4, 3]))
# [4, 2, 4, -1, -1] — 2的下一个更大是4, 1的下一个更大是2, ...
```

### 真题 4：下一个更大元素 I（LeetCode 496）

**题目**：`nums1 = [4, 1, 2], nums2 = [1, 3, 4, 2]`。nums1 中的每个元素，在 nums2 中找到其下一个更大元素。输出 `[-1, 3, -1]`。

```python
def next_greater_element(nums1, nums2):
    # 先对 nums2 预处理，用单调栈求出每个元素的下一个更大元素
    next_greater = {}
    stack = []

    for num in nums2:
        while stack and num > stack[-1]:
            next_greater[stack.pop()] = num
        stack.append(num)

    # 栈里剩余的元素没有更大的
    # 拼接结果
    return [next_greater.get(num, -1) for num in nums1]

# 验证
print(next_greater_element([4, 1, 2], [1, 3, 4, 2]))  # [-1, 3, -1]
print(next_greater_element([2, 4], [1, 2, 3, 4]))      # [3, -1]
```

### 真题 5：下一个更大元素 II（LeetCode 503）

**题目**：数组是循环的，每个元素可以往右找，到了末尾可以回头。重复一次数组即可。

```python
def next_greater_elements_ii(nums):
    n = len(nums)
    result = [-1] * n
    stack = []

    # 遍历两次（模拟循环数组）
    for i in range(n * 2):
        num = nums[i % n]
        while stack and num > nums[stack[-1]]:
            prev = stack.pop()
            result[prev] = num
        if i < n:  # 只在第一轮中入栈
            stack.append(i)

    return result

# 验证
print(next_greater_elements_ii([1, 2, 1]))  # [2, -1, 2]
print(next_greater_elements_ii([1, 2, 3, 4, 3]))  # [2, 3, 4, -1, 4]
```

### 真题 6：接雨水（LeetCode 42）

面试里单调栈的终极考题。三个解法：暴力（O(n²)）、双指针（O(n) + O(1)）、单调栈（O(n) + O(n)）。

**单调栈思路**：维护一个递减栈。当遇到一个比栈顶高的柱子时，说明栈顶柱子形成了低洼，可以接水。

```
图示：
[0, 1, 0, 2, 1, 0, 1, 3, 2, 1, 2, 1]
       ↑ 此处可接水：左墙=1, 右墙=2, 高度=1 → 水量=1×1=1
```

```python
def trap(height):
    """单调栈解法"""
    stack = []  # 递减栈，存索引
    water = 0

    for i, h in enumerate(height):
        # 当前柱子比栈顶高 → 栈顶处形成了低洼，可以积水
        while stack and h > height[stack[-1]]:
            bottom = stack.pop()  # 低洼底部
            if not stack:
                break  # 没有左墙，无法积水

            left = stack[-1]      # 左墙
            width = i - left - 1  # 水面宽度
            bounded_height = min(height[left], h) - height[bottom]
            water += width * bounded_height

        stack.append(i)

    return water

# 验证
print(trap([0, 1, 0, 2, 1, 0, 1, 3, 2, 1, 2, 1]))  # 6
print(trap([4, 2, 0, 3, 2, 5]))  # 9
```

**双指针解法（O(1) 空间）**：

```python
def trap_two_pointers(height):
    if not height:
        return 0

    left, right = 0, len(height) - 1
    left_max = right_max = 0
    water = 0

    while left < right:
        left_max = max(left_max, height[left])
        right_max = max(right_max, height[right])

        if left_max < right_max:
            water += left_max - height[left]
            left += 1
        else:
            water += right_max - height[right]
            right -= 1

    return water
```

**三种解法对比**：

| 解法 | 时间复杂度 | 空间复杂度 | 面试推荐 |
|------|-----------|-----------|---------|
| 暴力（每个位置左右扫） | O(n²) | O(1) | 说出来就好，别写 |
| 双指针 | O(n) | O(1) | 空间最优，首选 |
| 单调栈 | O(n) | O(n) | 展示对栈的掌握 |

### 单调栈模板总结

```
初始化 stack = [], result = [-1]*n  (或 [0]*n)

for i, val in enumerate(数组):
    while stack and val 破坏了栈的单调性:
        prev = stack.pop()
        result[prev] = ...  (处理 prev 对应的答案)

    stack.append(i)
```

| 问题类型 | 栈单调方向 | while 条件 | 含义 |
|---------|-----------|-----------|------|
| 下一个更大 | 递减（小→大） | `val > 栈顶` | 栈顶等到了更大的值 |
| 下一个更小 | 递增（大→小） | `val < 栈顶` | 栈顶等到了更小的值 |
| 上一个更大 | 递减，反向遍历 | `val > 栈顶` | 换个遍历方向 |
| 接雨水 | 递减 | `val > 栈顶` | 栈顶形成低洼 |

## 队列(Queue)：先进先出（FIFO）

```python
from collections import deque

q = deque()
q.append(1)        # 入队   O(1)
q.append(2)
first = q[0]       # 看队首 O(1) → 1
val = q.popleft()  # 出队   O(1) → 1
```

**用 deque 不用 list**：`list.pop(0)` 是 O(n)（所有元素向前移动），`deque.popleft()` 是 O(1)。

### deque 用法大全

```python
from collections import deque

dq = deque([1, 2, 3])

# 两端操作（全是 O(1)）
dq.append(4)        # 右端入 → [1, 2, 3, 4]
dq.appendleft(0)    # 左端入 → [0, 1, 2, 3, 4]
dq.pop()            # 右端出 → 4
dq.popleft()        # 左端出 → 0

# 其他实用操作
dq.extend([5, 6])       # 右端批量入
dq.extendleft([-2, -1]) # 左端批量入（注意：逆序）
dq.rotate(2)            # 右移 2 位
dq.rotate(-1)           # 左移 1 位
dq.reverse()            # 原地反转

# 随机访问（O(1)，deque 内部是双向链表数组的混合结构）
print(dq[0], dq[-1])
```

### 队列的应用场景

| 场景 | 描述 | 代表题目 |
|------|------|---------|
| **BFS（广度优先搜索）** | 队列是 BFS 的核心数据结构 | 二叉树层序遍历、迷宫最短路径 |
| **任务调度** | 先来先服务，保证公平性 | 打印机队列、CPU 进程调度 |
| **消息队列** | 解耦生产者和消费者，异步处理 | RabbitMQ、Kafka（概念基础） |
| **滑动窗口** | 双端队列维护窗口内的极值 | 滑动窗口最大值 |
| **缓存** | FIFO 缓存淘汰策略 | 最基础的缓存实现 |

### BFS 模板代码

```python
from collections import deque

def bfs(start):
    queue = deque([start])
    visited = {start}  # 避免重复访问
    step = 0

    while queue:
        # 当前层级的节点数（如果需要分层处理）
        level_size = len(queue)

        for _ in range(level_size):
            node = queue.popleft()

            # 处理当前节点
            if 找到目标:
                return step

            # 把邻居加入队列
            for neighbor in get_neighbors(node):
                if neighbor not in visited:
                    visited.add(neighbor)
                    queue.append(neighbor)

        step += 1

    return -1  # 没找到
```

### 真题 7：滑动窗口最大值（LeetCode 239）

单调队列的经典问题，TCP 流量控制中也有类似概念。

**题目**：`nums = [1, 3, -1, -3, 5, 3, 6, 7], k = 3`，求每个滑动窗口的最大值。输出 `[3, 3, 5, 5, 6, 7]`。

```python
from collections import deque

def max_sliding_window(nums, k):
    result = []
    dq = deque()  # 存索引，维护队列头始终是当前窗口最大值的索引

    for i, num in enumerate(nums):
        # 1. 移除窗口外的元素（队首索引 < 窗口左边界）
        if dq and dq[0] <= i - k:
            dq.popleft()

        # 2. 维持递减：从队尾移除所有比当前元素小的
        while dq and nums[dq[-1]] < num:
            dq.pop()

        dq.append(i)

        # 3. 窗口形成后（i >= k-1），队首就是当前窗口的最大值
        if i >= k - 1:
            result.append(nums[dq[0]])

    return result

# 验证
print(max_sliding_window([1, 3, -1, -3, 5, 3, 6, 7], 3))  # [3, 3, 5, 5, 6, 7]
print(max_sliding_window([1, -1], 1))                      # [1, -1]
```

**模拟过程**（k=3）：

```
i=0, num=1:  dq=[0],                     窗口未形成
i=1, num=3:  dq移除0, dq=[1],            窗口未形成
i=2, num=-1: dq=[1,2],                   窗口 [1,3,-1] → 最大值 nums[1]=3
i=3, num=-3: dq移除1(窗口外), dq=[2,3],  窗口 [3,-1,-3] → 最大值 nums[2]=-1? 不对...
```

更细致的模拟：

```
i=0, num=1:  dq append 0 → dq=[0]
i=1, num=3:  3>1, pop 0, append 1 → dq=[1]
i=2, num=-1: -1<3, append 2 → dq=[1,2]
             窗口 [0,1,2] → nums[dq[0]]=nums[1]=3 ✓ result=[3]
i=3, num=-3: 移除 dq[0]=1? 1 > 3-3=0 不移除
             -3 < nums[dq[-1]]=-1, append 3 → dq=[1,2,3]
             窗口 [1,2,3] → nums[dq[0]]=nums[1]=3 ✓ result=[3,3]
i=4, num=5:  移除 dq[0]=1? 1 ≤ 4-3=1 移除 → dq=[2,3]
             5>-1, pop 3; 5>-3, pop 2; append 4 → dq=[4]
             窗口 [2,3,4] → nums[dq[0]]=nums[4]=5 ✓ result=[3,3,5]
...
```

**面试话术**："用双端队列维护窗口内元素的索引，队首始终是当前窗口最大值的索引。入队时从队尾弹掉所有比新元素小的（它们永远不会成为最大值了），出队时检查队首是否已滑出窗口。"

### 两个栈实现队列（LeetCode 232）

上一节有详细代码。核心是**均摊 O(1)**：每个元素最多从 in_stack 移到 out_stack 一次。

### 两个队列实现栈（LeetCode 225）

反向操作，用队列模拟栈：

```python
from collections import deque

class MyStack:
    def __init__(self):
        self.q = deque()

    def push(self, x):
        # 核心：每次 push 后，把新元素之前的元素都移到它后面
        # 这样队首始终是最后 push 的元素（栈顶）
        self.q.append(x)
        for _ in range(len(self.q) - 1):
            self.q.append(self.q.popleft())

    def pop(self):
        return self.q.popleft()

    def top(self):
        return self.q[0]

    def empty(self):
        return len(self.q) == 0

# 验证
stack = MyStack()
stack.push(1)
stack.push(2)
stack.push(3)
print(stack.pop())  # 3
print(stack.pop())  # 2
print(stack.top())  # 1
print(stack.empty())  # False
```

**复杂度**：push O(n)，pop O(1)，top O(1)。也可以用两个队列优化 push 到 O(1)，但 pop 就变成 O(n)。本质是栈和队列的语义互斥，无法同时做到所有操作 O(1)。

## 优先队列（堆）基础

虽然不是本章重点，但面试中和队列常常一起出现。优先队列的核心：**不是先进先出，而是优先级高的先出。**

```python
import heapq

# Python 的 heapq 是最小堆（堆顶最小）
heap = []
heapq.heappush(heap, 3)
heapq.heappush(heap, 1)
heapq.heappush(heap, 2)
print(heapq.heappop(heap))  # 1（最小的先出）
print(heapq.heappop(heap))  # 2
print(heapq.heappop(heap))  # 3

# 最大堆：取反
max_heap = []
heapq.heappush(max_heap, -3)
heapq.heappush(max_heap, -1)
print(-heapq.heappop(max_heap))  # 3（最大的先出）

# Top K 问题模板
def top_k(nums, k):
    """找数组中第 K 大的元素"""
    heap = []
    for num in nums:
        heapq.heappush(heap, num)
        if len(heap) > k:
            heapq.heappop(heap)  # 弹出最小的，保留 K 个最大的
    return heap[0]  # 堆顶是第 K 大的元素

print(top_k([3, 2, 1, 5, 6, 4], 2))  # 5
print(top_k([3, 2, 3, 1, 2, 4, 5, 5, 6], 4))  # 4
```

| 操作 | heapq | 说明 |
|------|-------|------|
| push | `heappush(heap, item)` | O(log n) |
| pop | `heappop(heap)` | O(log n)，返回最小值 |
| peek | `heap[0]` | O(1)，返回最小值 |
| heapify | `heapify(list)` | O(n)，原地建堆 |

## 面试专题

### 什么时候想到用单调栈？

遇到以下关键词时，单调栈大概率是正解：

1. **"下一个更大的/更小的"** — 直接的单调栈信号
2. **"找某个元素左右第一个比它大/小的"** — 单次遍历即可获得左右信息
3. **"柱状图/接雨水/最大矩形"** — 二维拓展，本质仍是找左右边界
4. **"维护某种单调性"** — 题目中有"依次处理""等待"等概念

**面试话术**："单调栈解决的问题都有一个共同特征——需要找到每个元素左边或右边第一个比它大/小的元素。如果在暴力解法中每个元素都要扫描左右两侧，那单调栈可以把 O(n²) 变成 O(n)。"

### 什么时候用栈 vs 队列？

| 特征 | 用栈 | 用队列 |
|------|------|--------|
| 处理顺序 | 后到的先处理（LIFO） | 先到的先处理（FIFO） |
| 典型场景 | 括号匹配、撤销、DFS、单调栈 | 任务调度、BFS、消息队列 |
| 嵌套/层级 | 有嵌套结构（括号、HTML 标签、函数调用） | 按层级展开（树的层序遍历） |
| 回溯 | 需要回溯到之前的状态 | 不需要回溯，线性处理 |
| "等待"语义 | 等待某个条件触发后弹出 | 按顺序排队等待 |

### 单调栈面试话术模板

"我维护一个从栈底到栈顶递减（或递增）的栈，栈里存的是索引。遍历数组时，如果当前元素比栈顶大，说明栈顶元素等到了答案——当前元素就是栈顶的下一个更大元素——弹出栈顶并记录结果。每个元素最多入栈一次、出栈一次，所以 O(n)。"

## 面试高频考点小结

| 考点 | 出现频率 | 核心技巧 | 易错点 |
|------|---------|---------|--------|
| 有效括号 | ★★★★★ | 栈匹配左右括号 | 最后忘检查栈是否为空 |
| 最长有效括号 | ★★★ | 栈底存基准位置 | 理解不了为什么栈底存-1 |
| 逆波兰表达式 | ★★★★ | 遇数入栈，遇运算符弹出两个计算 | a、b 弹出顺序不要弄反 |
| 每日温度 | ★★★★ | 递减栈存索引 | 搞混递增栈还是递减栈 |
| 下一个更大元素 I/II | ★★★ | 单调栈模板直接套 | 循环数组的处理 |
| 接雨水 | ★★★★★ | 单调栈 或 双指针 | 理解为什么按 min(左右) - 底 算 |
| 栈实现队列 | ★★★★ | 两个栈+均摊 | 不明白为什么是均摊 O(1) |
| 队列实现栈 | ★★★ | push时旋转 | 和栈实现队列搞混 |
| 滑动窗口最大值 | ★★★★ | 单调双端队列 | 队首过期移除条件 |
| BFS 模板 | ★★★★★ | 队列+visited | 忘了标记 visited 导致死循环 |
| Top K | ★★★★ | 小顶堆维护 K 个最大 | 什么时候用大顶堆、什么时候用小顶堆 |
