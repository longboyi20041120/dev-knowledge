# 数组与双指针

面试出镜率最高的技巧。核心思想：**用两个指针遍历数组，把 O(n²) 降到 O(n)。**

## 真题 1：两数之和（Two Sum）

LeetCode 1，面试问烂了但还问。

**题目**：`nums = [2, 7, 11, 15], target = 9`，找两个数之和等于 target，返回下标。

```python
# 暴力 O(n²) — 面试写了直接挂
def two_sum_brutal(nums, target):
    for i in range(len(nums)):
        for j in range(i+1, len(nums)):
            if nums[i] + nums[j] == target:
                return [i, j]

# 哈希表 O(n) — 正确解法
def two_sum(nums, target):
    seen = {}
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i
```

**面试话术**："暴力法需要 O(n²)，用哈希表把'找另一个数'从遍历变成 O(1) 查找。时间和空间的经典权衡——多用 O(n) 空间换 O(n) 时间。"

## 真题 2：盛最多水的容器

LeetCode 11。双指针经典题。

```
输入：[1, 8, 6, 2, 5, 4, 8, 3, 7]
图示：竖线高度为数组值，找两条线围出的最大面积
输出：49（第2根和第9根围出 7×7=49）
```

```python
def max_area(height):
    left, right = 0, len(height) - 1
    max_water = 0

    while left < right:
        # 宽度 × 较矮的高度
        water = (right - left) * min(height[left], height[right])
        max_water = max(max_water, water)

        # 关键：移动较矮的那根（移动高的不会让面积变大）
        if height[left] < height[right]:
            left += 1
        else:
            right -= 1

    return max_water

# 验证
height = [1, 8, 6, 2, 5, 4, 8, 3, 7]
print(max_area(height))  # 49
```

**为什么移动较矮的？** 宽度一直在缩小，如果移动高的那根，较矮的没变，面积只会更小。移动较矮的有可能碰到一根更高的，面积才有可能变大。

**面试话术**："双指针从两端向中间收缩，每次移动较矮的一侧。因为宽度递减，只有移动矮的有机会让面积变大。O(n) 时间 O(1) 空间。"

## 真题 3：三数之和

LeetCode 15。高频。

**题目**：`nums = [-1, 0, 1, 2, -1, -4]`，找所有和为 0 的三元组。输出 `[[-1, -1, 2], [-1, 0, 1]]`。

```python
def three_sum(nums):
    nums.sort()  # 先排序（双指针的前提）
    n = len(nums)
    result = []

    for i in range(n - 2):
        # 跳过重复的第一个数
        if i > 0 and nums[i] == nums[i-1]:
            continue
        # 如果最小的三个数之和 > 0，后面不可能了
        if nums[i] + nums[i+1] + nums[i+2] > 0:
            break
        # 如果当前 + 最大的两个 < 0，当前太小了跳过
        if nums[i] + nums[n-2] + nums[n-1] < 0:
            continue

        left, right = i + 1, n - 1
        while left < right:
            total = nums[i] + nums[left] + nums[right]
            if total < 0:
                left += 1
            elif total > 0:
                right -= 1
            else:
                result.append([nums[i], nums[left], nums[right]])
                # 跳过重复值
                while left < right and nums[left] == nums[left+1]:
                    left += 1
                while left < right and nums[right] == nums[right-1]:
                    right -= 1
                left += 1
                right -= 1

    return result
```

**复杂度**：排序 O(n log n)，外层循环 O(n) × 双指针 O(n) = O(n²)。

**面试话术**："先排序，固定第一个数，剩下的用双指针变成两数之和问题。注意跳过重复值，否则结果会有重复三元组。"

## 双指针总结

| 场景 | 指针位置 | 移动策略 |
|------|---------|---------|
| 两数之和 II | 左右两端 | 和小了移左，大了移右 |
| 盛水容器 | 左右两端 | 移动较矮的那个 |
| 三数之和 | 一固定 + 两端 | 固定最外层，内层双指针 |
| 快慢指针 | 同向 | 快指针找符合条件的元素，慢指针指向写入位置 |
| 滑动窗口 | 同向 | 右指针扩展窗口，左指针收缩窗口 |
