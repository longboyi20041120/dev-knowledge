# 数组与双指针

> 面试出镜率最高的技巧。核心思想：用两个指针遍历数组，把 O(n²) 降到 O(n)。 | **面试重要度：极高** | 预计阅读：25 分钟

## 双指针的三种模式

面试中所有双指针问题都逃不出这三种模式。先搞清楚每种模式的适用场景，然后做题就只需要往模板里套。

| 模式 | 指针位置 | 移动方向 | 典型场景 | 代表题目 |
|------|---------|---------|---------|---------|
| **左右指针** | 数组两端 | 向中间收缩 | 有序数组、回文、反转 | Two Sum II、盛水容器、反转字符串 |
| **快慢指针** | 同向，一快一慢 | 同向移动 | 去重、链表环检测、原地修改 | 移动零、删除重复项 |
| **滑动窗口** | 同向，维护窗口 | 同向，右扩左缩 | 子数组/子串问题 | 无重复字符最长子串、长度最小子数组 |

## 模式一：左右指针

### 真题：两数之和 II（LeetCode 167）

输入有序数组，这题不能用哈希表，要用左右指针。

**题目**：`numbers = [2, 7, 11, 15], target = 9`，返回 `[1, 2]`（下标从 1 开始）。

```python
def two_sum_ii(numbers, target):
    left, right = 0, len(numbers) - 1

    while left < right:
        current = numbers[left] + numbers[right]
        if current == target:
            return [left + 1, right + 1]  # 题目要求下标从 1 开始
        elif current < target:
            left += 1   # 和太小，左指针右移（让和变大）
        else:
            right -= 1  # 和太大，右指针左移（让和变小）

# 验证
numbers = [2, 7, 11, 15]
print(two_sum_ii(numbers, 9))   # [1, 2]
print(two_sum_ii(numbers, 18))  # [2, 3]
```

**复杂度**：O(n) 时间，O(1) 空间。跟哈希表方案（O(n) 时间 + O(n) 空间）相比，牺牲了一点通用性（要求有序），换来了 O(1) 空间。

**面试话术**："因为数组有序，和大了就缩小右边的数，和小了就放大左边的数。每次移动都能排除掉不可能的组合，总共只需遍历一次。"

### 真题：盛最多水的容器（LeetCode 11）

上一节已有完整代码。补充一个图解的直观理解：

```
[1, 8, 6, 2, 5, 4, 8, 3, 7]
 L                       R
 1×8=8  → 太小，L右移
    L                    R
    8×7=56 → 左边高了，R左移
    L                 R
    8×6=48 → ...
```

**为什么移动较矮的？** 宽度一直在缩小。如果移动高的那根，较矮的那根没变，新面积 = 新宽度 × 更矮的高度（要么跟原来一样矮，要么更矮），所以面积只会更小。只有移动较矮的才有可能碰到更高的，面积才有可能变大。

### 真题：反转字符串（LeetCode 344）

最典型的左右指针，简单但面试会问。

**题目**：原地反转字符数组 `["h","e","l","l","o"]` → `["o","l","l","e","h"]`。

```python
def reverse_string(s):
    left, right = 0, len(s) - 1
    while left < right:
        s[left], s[right] = s[right], s[left]  # 交换两端字符
        left += 1
        right -= 1
    # 原地修改，不需要返回值

# 验证
s = ["h", "e", "l", "l", "o"]
reverse_string(s)
print(s)  # ['o', 'l', 'l', 'e', 'h']
```

**复杂度**：O(n) 时间，O(1) 空间。

## 模式二：快慢指针

快慢指针的核心思想：**快指针在前面探路，慢指针在后面记录"有效位置"**。两者之间的区域是"已处理但不需要保留"的数据。

### 真题：移动零（LeetCode 283）

**题目**：`nums = [0, 1, 0, 3, 12]`，把所有 0 移到末尾，保持非零元素顺序。输出 `[1, 3, 12, 0, 0]`。

```python
def move_zeroes(nums):
    slow = 0  # 指向"下一个非零元素应该放的位置"

    for fast in range(len(nums)):
        if nums[fast] != 0:
            # 交换慢指针和快指针的值
            nums[slow], nums[fast] = nums[fast], nums[slow]
            slow += 1
    # 现在 slow 之前全是非零，slow 之后全是零

# 验证
nums = [0, 1, 0, 3, 12]
move_zeroes(nums)
print(nums)  # [1, 3, 12, 0, 0]
```

**模拟过程**：

```
初始: [0, 1, 0, 3, 12], slow=0
fast=0: nums[0]=0, 跳过, slow 不动        → slow=0
fast=1: nums[1]=1≠0, 交换 0和1
        [1, 0, 0, 3, 12], slow=1
fast=2: nums[2]=0, 跳过, slow 不动        → slow=1
fast=3: nums[3]=3≠0, 交换 slow(1)和 fast(3)
        [1, 3, 0, 0, 12], slow=2
fast=4: nums[4]=12≠0, 交换 slow(2)和 fast(4)
        [1, 3, 12, 0, 0], slow=3
```

**复杂度**：O(n) 时间，O(1) 空间。

**为什么要交换而不是覆盖？** 如果直接覆盖非零值到 slow 位置，零元素会丢失顺序信息。交换可以同时保证非零往前挪、零往后挪。

### 真题：删除有序数组中的重复项（LeetCode 26）

**题目**：`nums = [1, 1, 2]` → 返回长度 2，数组前两个元素变成 `[1, 2, _]`。

```python
def remove_duplicates(nums):
    if not nums:
        return 0

    slow = 0  # 指向"已去重部分"的最后一个位置

    for fast in range(1, len(nums)):
        if nums[fast] != nums[slow]:
            # 发现新元素，放到 slow 的下一个位置
            slow += 1
            nums[slow] = nums[fast]

    return slow + 1  # 去重后的长度

# 验证
nums = [1, 1, 2]
k = remove_duplicates(nums)
print(k, nums[:k])  # 2 [1, 2]

nums2 = [0, 0, 1, 1, 1, 2, 2, 3, 3, 4]
k2 = remove_duplicates(nums2)
print(k2, nums2[:k2])  # 5 [0, 1, 2, 3, 4]
```

**复杂度**：O(n) 时间，O(1) 空间。

**面试话术**："slow 维护已去重的最后一个位置，fast 去前面探路。遇到跟 slow 不一样的元素，说明找到了新元素，放到 slow+1 的位置。因为数组有序，所有重复元素都挨在一起。"

### 快慢指针小结

```
模板骨架：
slow = 0  (或 1，看具体题)
for fast in range(len(nums)):
    if 满足条件(nums[fast]):
        # 处理 slow 位置
        slow += 1
```

慢指针的含义取决于题目：移动零是"下一个非零元素的位置"，删除重复项是"已去重区域的最后一个位置"。

## 模式三：滑动窗口

滑动窗口可以理解为**快慢指针在维护一个区间 [left, right)**。当问题涉及"连续子数组"或"子串"时，优先考虑滑动窗口。

### 可变窗口模板

适用于"满足某个条件的最短/最长子串"：

```python
def sliding_window_variable(s):
    left = 0
    result = 0
    window_state = {}  # 根据题目选择数据结构

    for right in range(len(s)):
        # 1. 加入 right 位置的元素，更新窗口状态
        window_state[s[right]] = window_state.get(s[right], 0) + 1

        # 2. 当条件不满足时，收缩左边界
        while 窗口需要收缩:
            window_state[s[left]] -= 1
            if window_state[s[left]] == 0:
                del window_state[s[left]]
            left += 1

        # 3. 此时窗口有效，更新结果
        result = max(result, right - left + 1)  # 或 min

    return result
```

### 真题：无重复字符的最长子串（LeetCode 3）

**题目**：`"abcabcbb"` → 最长无重复子串是 `"abc"`，长度 3。

```python
def length_of_longest_substring(s):
    left = 0
    max_len = 0
    seen = {}  # 字符 → 最近出现的位置

    for right, char in enumerate(s):
        # 如果字符在窗口内出现过，left 跳到重复字符的下一个位置
        if char in seen and seen[char] >= left:
            left = seen[char] + 1

        seen[char] = right
        max_len = max(max_len, right - left + 1)

    return max_len

# 验证
print(length_of_longest_substring("abcabcbb"))  # 3
print(length_of_longest_substring("bbbbb"))     # 1
print(length_of_longest_substring("pwwkew"))    # 3 ("wke")
```

**模拟**：s = `"abcabcbb"`

```
right=0 'a': seen={a:0}, left=0, 窗口 [0,0] → max_len=1
right=1 'b': seen={a:0,b:1}, left=0, 窗口 [0,1] → max_len=2
right=2 'c': seen={a:0,b:1,c:2}, left=0, 窗口 [0,2] → max_len=3
right=3 'a': a 在窗口内(位置0), left 跳到 1
            seen={a:3,b:1,c:2}, 窗口 [1,3] → max_len=3
right=4 'b': b 在窗口内(位置1), left 跳到 2
            seen={a:3,b:4,c:2}, 窗口 [2,4] → max_len=3
...
```

**复杂度**：O(n) 时间，每个字符最多被 left 和 right 各访问一次。O(min(m, n)) 空间，m 是字符集大小。

### 固定窗口模板

适用于窗口大小固定的问题：

```python
def sliding_window_fixed(arr, k):
    left = 0
    window_sum = 0
    result = 0

    for right in range(len(arr)):
        window_sum += arr[right]

        # 窗口达到大小 k
        if right >= k - 1:
            result = max(result, window_sum)  # 或做其他操作
            window_sum -= arr[left]           # 移除左边元素
            left += 1

    return result
```

### 真题：长度最小的子数组（LeetCode 209）

**题目**：`target = 7, nums = [2, 3, 1, 2, 4, 3]`，找和 ≥7 的最短连续子数组，返回长度 2（子数组 `[4, 3]`）。

```python
def min_sub_array_len(target, nums):
    left = 0
    current_sum = 0
    min_len = float('inf')

    for right in range(len(nums)):
        current_sum += nums[right]

        # 窗口满足条件时，尝试收缩
        while current_sum >= target:
            min_len = min(min_len, right - left + 1)
            current_sum -= nums[left]
            left += 1

    return 0 if min_len == float('inf') else min_len

# 验证
print(min_sub_array_len(7, [2, 3, 1, 2, 4, 3]))  # 2
print(min_sub_array_len(4, [1, 4, 4]))            # 1
print(min_sub_array_len(11, [1, 1, 1, 1]))        # 0
```

**为什么时间复杂度是 O(n)？** 虽然有双层循环（for + while），但 left 指针从不后退，每个元素最多被加入一次（right 扫过）和移除一次（left 扫过），总共 2n 次操作 = O(n)。

## 真题：三数之和（LeetCode 15）

上一节已有完整代码。这里补充两个剪枝优化后的模板：

```python
def three_sum(nums):
    nums.sort()
    n = len(nums)
    result = []

    for i in range(n - 2):
        # 去重：跳过重复的第一个数
        if i > 0 and nums[i] == nums[i - 1]:
            continue
        # 剪枝1：最小的三个数之和大于 0，后面不可能了
        if nums[i] + nums[i + 1] + nums[i + 2] > 0:
            break
        # 剪枝2：当前数加上最大的两个数小于 0，当前 i 太小，跳过
        if nums[i] + nums[n - 2] + nums[n - 1] < 0:
            continue

        left, right = i + 1, n - 1
        while left < right:
            total = nums[i] + nums[left] + nums[right]
            if total < 0:
                left += 1
            elif total > 0:
                right -= 1
            else:
                result.append([nums[i], nums[left], nums[right]])
                # 跳过内层重复值
                while left < right and nums[left] == nums[left + 1]:
                    left += 1
                while left < right and nums[right] == nums[right - 1]:
                    right -= 1
                left += 1
                right -= 1

    return result

# 验证
nums = [-1, 0, 1, 2, -1, -4]
print(three_sum(nums))  # [[-1, -1, 2], [-1, 0, 1]]
```

**复杂度**：排序 O(n log n)，外层循环 O(n) 乘以内层双指针 O(n)，总 O(n²)。空间 O(1)（不计结果数组）。

## 真题：合并两个有序数组（LeetCode 88）

这题用**逆向双指针**，是左右指针的一个变体。

**题目**：`nums1 = [1, 2, 3, 0, 0, 0], m = 3, nums2 = [2, 5, 6], n = 3`，合并到 nums1 中。输出 `[1, 2, 2, 3, 5, 6]`。

**关键洞察**：从后往前填，避免覆盖 nums1 中尚未比较的元素。

```python
def merge(nums1, m, nums2, n):
    # 三个指针：p1 指向 nums1 有效部分的末尾，p2 指向 nums2 末尾
    # p 指向 nums1 最终位置的末尾
    p1, p2 = m - 1, n - 1
    p = m + n - 1

    # 从大到小逆序填入
    while p1 >= 0 and p2 >= 0:
        if nums1[p1] > nums2[p2]:
            nums1[p] = nums1[p1]
            p1 -= 1
        else:
            nums1[p] = nums2[p2]
            p2 -= 1
        p -= 1

    # 如果 nums2 还有剩余，直接复制（nums1 有剩余则不用管，已经在正确位置）
    while p2 >= 0:
        nums1[p] = nums2[p2]
        p2 -= 1
        p -= 1

# 验证
nums1 = [1, 2, 3, 0, 0, 0]
merge(nums1, 3, [2, 5, 6], 3)
print(nums1)  # [1, 2, 2, 3, 5, 6]
```

**复杂度**：O(m + n) 时间，O(1) 空间。

**面试话术**："如果从前往后合并，nums1 前面的元素可能被覆盖。从后往前填就能避免这个问题——大的先放到尾部，空出来的位置本来就是 0，随便填。"

## 双指针模式总结

| 模式 | 模板骨架 | 时间复杂度 | 空间复杂度 | 识别特征 |
|------|---------|-----------|-----------|---------|
| 左右指针 | `while left < right: if cond: left++ else: right--` | O(n) | O(1) | 有序数组、对称/回文 |
| 快慢指针 | `for fast: if ok: swap/assign slow; slow++` | O(n) | O(1) | 原地修改、去重、移动 |
| 滑动窗口 | `for right: add(right); while invalid: remove(left++)` | O(n) | 视情况 | "连续子数组/子串" |
| 逆向双指针 | `while p1>=0 and p2>=0: fill from end` | O(m+n) | O(1) | 合并到已分配空间的数组 |

## 面试专题

### 什么时候用双指针？

用一个问题自检：**"能不能在一次遍历中，根据当前状态决定下一步往哪走？"**

- 有序数组 → 左右指针（知道往哪移动）
- 需要原地修改数组 → 快慢指针（一个维护结果，一个扫描）
- 涉及子串/子数组 → 滑动窗口（维护一个可变区间）
- 合并/归并 → 多指针（各自遍历一部分）

### 时间复杂度为什么是 O(n)？

双指针的 O(n) 核心在于：**每个元素最多被每个指针访问一次，指针从不后退。**

以滑动窗口为例，虽然有 for + while 嵌套，但 left 从不后退，每个元素最多被加入和移出各一次，总共 2n 次操作。

以三数之和为例，外层 O(n)，内层双指针虽然有 while，但 left 和 right 总共只移动 O(n) 次。每次外层循环重置 left 和 right，但它们的总移动次数不超 O(n)。所以整体是 O(n) × O(n) = O(n²)，而非 O(n³)。

### 面试高频追问

**问：滑动窗口为什么不用 if 而用 while？**
答：因为窗口可能需要连续收缩多次。比如 target=7，nums=[2,3,1,2,4,3]，当 right 走到 4 时，[2,3,1,2,4] 的和是 12，移除一个 2 后和为 10 仍 ≥7，需要继续收缩。if 只会收缩一次，while 收缩到不满足条件为止。

**问：三数之和去重为什么要两层？**
答：外层去的是固定值 i 的重复，内层去的是 left 和 right 移动时的重复。两层分别保证：固定值不重复选同一个数，内层找到的三元组不重复。

**问：移动零为什么要交换而不是直接把非零覆盖到前面、末尾补零？**
答：交换更优雅，一次遍历完成。覆盖+补零也正确且可能更好理解，两种写法都行。面试时选你更不容易写错的。
