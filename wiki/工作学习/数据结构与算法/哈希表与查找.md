# 哈希表与查找

面试最常考的三种查找：哈希表 O(1)、排序+二分 O(log n)、暴力 O(n)。核心是**什么时候用哪个**。

哈希表是面试中出现频率最高的数据结构——没有之一。从 Two Sum 到 LRU Cache，从分组统计到去重，几乎每道题都能看到它的身影。

---

## 哈希表的本质

一个键值对集合，通过哈希函数把键映射到数组索引，实现 O(1) 的增删查。

```python
# Python 字典就是哈希表
seen = {}
seen["key"] = 1         # 插入 O(1)
val = seen.get("key")   # 查找 O(1)
del seen["key"]         # 删除 O(1)
```

### 哈希函数怎么工作？

哈希函数接收任意类型的键，输出一个固定范围内的整数（数组下标）。

```
输入："hello" → 哈希函数 → 整数 12345 → 12345 % 数组大小 = 5 → 存入 arr[5]
```

一个好的哈希函数要满足三点：
1. **确定性**：相同输入永远输出相同结果
2. **均匀分布**：不同的键尽量映射到不同索引
3. **高效**：计算速度快

Python 中 `hash()` 函数就是哈希函数的实现：

```python
print(hash("hello"))      # 每次运行的绝对值不同（有随机化），但在同一次运行中一致
print(hash("world"))
print(hash((1, 2, 3)))    # 元组可哈希
# print(hash([1, 2, 3]))  # TypeError! 列表不可哈希——因为可变对象哈希后内容变了就找不到了
```

### 哈希冲突：两个键映射到同一个位置怎么办？

这是面试最容易问到的基础知识。两种主流解决方案：

| 方案 | 思路 | 查找过程 | 优点 | 缺点 | 使用场景 |
|------|------|---------|------|------|---------|
| **链表法（拉链法）** | 每个槽位挂一个链表，冲突的元素都塞进去 | 先定位槽位，再遍历链表 | 实现简单、删除方便 | 链表指针占用额外内存，缓存不友好 | Java HashMap、Redis（listpack 之前） |
| **开放寻址法** | 冲突了就找下一个空位（线性探测/二次探测/双重哈希） | 从 hash 位置开始探测，直到找到或遇到空位 | 无指针开销，缓存友好 | 删除麻烦（要设墓碑标记）、负载因子高时性能差 | Python dict、C++ unordered_map |

**开放寻址的探测方式**：

```python
# 线性探测（最简单）
def linear_probe(hash_val, i, size):
    return (hash_val + i) % size

# 二次探测（减少聚集）
def quadratic_probe(hash_val, i, size):
    return (hash_val + i * i) % size

# 双重哈希（最均匀，但计算成本高）
def double_hash(hash_val, i, size, hash2_val):
    return (hash_val + i * hash2_val) % size
```

**面试话术**："哈希冲突有两种主流解法，链表法把冲突元素串成链表，开放寻址法在数组里找下一个空位。Python dict 用的是开放寻址，在负载因子约 2/3 时触发扩容。Java HashMap 在链表长度 > 8 时把链表转成红黑树防止退化。"

### Python dict 内部实现简介

Python 3.6+ 的 dict 实现很精巧（compact dict）：

1. **底层结构**：一个紧凑的 entries 数组，存哈希值 + 键 + 值
2. **冲突处理**：开放寻址（伪随机探测，用的是一种高性能变体）
3. **负载因子**：约 2/3，即数组填满到约 66% 时扩容（翻倍）
4. **有序性**：Python 3.7+ 保证 dict 按插入顺序遍历（但不是靠链表，而是靠 entries 数组本身的有序存储）

```python
# Python 3.7+ dict 保留插入顺序
d = {}
d["c"] = 1
d["a"] = 2
d["b"] = 3
print(list(d.keys()))  # ['c', 'a', 'b'] — 按插入顺序，不是按键排序

# 但要小心：这个"有序"只保证迭代时的顺序，不是数据结构本身有序
# 不支持按索引访问，不支持二分查找
```

## collections 工具：Counter 和 defaultdict

这两个是哈希表的得力助手，用得好代码量减半。

### Counter：计数器

```python
from collections import Counter

# 词频统计 — 一行搞定
words = ["apple", "banana", "apple", "orange", "banana", "apple"]
count = Counter(words)
print(count)  # Counter({'apple': 3, 'banana': 2, 'orange': 1})

# 最常用的 N 个元素
print(count.most_common(2))  # [('apple', 3), ('banana', 2)]

# 计数加减
count["grape"] += 1          # 也可以 count.update(["grape"])
count["apple"] -= 1
print(count)  # Counter({'apple': 2, 'banana': 2, 'orange': 1, 'grape': 1})

# 字符串字符计数
print(Counter("abracadabra"))  # Counter({'a': 5, 'b': 2, 'r': 2, 'c': 1, 'd': 1})
```

### defaultdict：带默认值的字典

```python
from collections import defaultdict

# 不用 defaultdict 的写法：
groups = {}
for word in ["eat", "tea", "tan", "ate", "nat", "bat"]:
    key = "".join(sorted(word))
    if key not in groups:
        groups[key] = []
    groups[key].append(word)

# 用 defaultdict 的写法 — 不需要检查 key 是否存在
groups = defaultdict(list)
for word in ["eat", "tea", "tan", "ate", "nat", "bat"]:
    key = "".join(sorted(word))
    groups[key].append(word)

# defaultdict(int) — 计数器简化
counts = defaultdict(int)
for ch in "hello":
    counts[ch] += 1  # 不存在的 key 自动初始化为 0
print(dict(counts))  # {'h': 1, 'e': 1, 'l': 2, 'o': 1}

# defaultdict(set) — 去重分组
seen = defaultdict(set)
pairs = [("a", 1), ("a", 2), ("b", 1), ("a", 1)]
for k, v in pairs:
    seen[k].add(v)
print(dict(seen))  # {'a': {1, 2}, 'b': {1}}
```

**默认值对比**：

| 写法 | 不存在的 key |
|------|-------------|
| `dict[key]` | KeyError |
| `dict.get(key)` | 返回 None |
| `dict.get(key, default)` | 返回 default |
| `defaultdict(int)[key]` | 返回 0 |
| `defaultdict(list)[key]` | 返回 [] |
| `defaultdict(set)[key]` | 返回 set() |

## 真题 1：两数之和及变体

### 原版 Two Sum（LeetCode 1）

前面章节已经详述，这里补充变体。

### 变体：返回所有和为 target 的数对

```python
def two_sum_all_pairs(nums, target):
    """返回所有和为 target 的不重复数对"""
    seen = {}
    pairs = set()  # 用 set 去重

    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            # 用 tuple 的排序版本去重，避免 (2,7) 和 (7,2) 都加入
            pairs.add(tuple(sorted((num, complement))))

        seen[num] = i

    return [list(p) for p in pairs]

# 验证
nums = [2, 7, 11, 15, 7, 2]
print(two_sum_all_pairs(nums, 9))  # [[2, 7]]
```

### 变体：Two Sum 的输入是数据流

```python
class TwoSumStream:
    """支持 add 和 find 的 Two Sum"""
    def __init__(self):
        self.counts = defaultdict(int)

    def add(self, number):
        self.counts[number] += 1

    def find(self, value):
        for num in list(self.counts.keys()):
            complement = value - num
            if complement == num:
                if self.counts[num] >= 2:
                    return True
            elif complement in self.counts:
                return True
        return False

# 验证
ts = TwoSumStream()
ts.add(1); ts.add(3); ts.add(5)
print(ts.find(4))  # True  (1 + 3)
print(ts.find(7))  # False
```

## 真题 2：同构字符串（LeetCode 205）

**题目**：`s = "egg", t = "add"` → `true`（e→a, g→d）。`s = "foo", t = "bar"` → `false`（o 映射了 r 但不能又映射 a）。

核心：**双向映射**，两个字符之间是一一对应关系。

```python
def is_isomorphic(s, t):
    if len(s) != len(t):
        return False

    s_to_t = {}  # s 的字符 → t 的字符
    t_to_s = {}  # t 的字符 → s 的字符（保证一对一）

    for cs, ct in zip(s, t):
        if cs in s_to_t:
            if s_to_t[cs] != ct:
                return False
        else:
            s_to_t[cs] = ct

        if ct in t_to_s:
            if t_to_s[ct] != cs:
                return False
        else:
            t_to_s[ct] = cs

    return True

# 验证
print(is_isomorphic("egg", "add"))      # True
print(is_isomorphic("foo", "bar"))      # False — o 对应了 a 和 r
print(is_isomorphic("paper", "title"))  # True
print(is_isomorphic("badc", "baba"))    # False — b↔b, a↔a, d↔b? 不对, b 已被映射
```

**复杂度**：O(n) 时间，O(1) 空间（字符集有限，ASCII 最多 128/256）。

## 真题 3：快乐数（LeetCode 202）

**题目**：判断一个数是不是快乐数。规则：每位数字平方和，反复计算，最终变成 1 就是快乐数，陷入循环就不是。
例如 19：1²+9²=82, 8²+2²=68, 6²+8²=100, 1²+0²+0²=1 → 快乐。

**核心技巧**：用哈希表检测循环。

```python
def is_happy(n):
    seen = set()

    while n != 1:
        if n in seen:
            return False  # 进入循环，不是快乐数

        seen.add(n)
        # 计算各位数字平方和
        n = sum(int(d) ** 2 for d in str(n))

    return True

# 验证
print(is_happy(19))  # True
print(is_happy(2))   # False (2→4→16→37→58→89→145→42→20→4 循环)
```

**快慢指针优化（O(1) 空间）**：这题也可以用快慢指针，不需要 set。

```python
def is_happy_fast(n):
    def get_next(num):
        return sum(int(d) ** 2 for d in str(num))

    slow = n
    fast = get_next(n)

    while fast != 1 and slow != fast:
        slow = get_next(slow)
        fast = get_next(get_next(fast))

    return fast == 1
```

## 真题 4：最长和谐子序列（LeetCode 594）

**题目**：`nums = [1, 3, 2, 2, 5, 2, 3, 7]`，和谐子序列是最大值和最小值相差恰好为 1 的子序列。最长的是 `[3, 2, 2, 2, 3]`，长度 5。

```python
from collections import Counter

def find_lhs(nums):
    count = Counter(nums)
    max_len = 0

    for num in count:
        if num + 1 in count:
            max_len = max(max_len, count[num] + count[num + 1])

    return max_len

# 验证
print(find_lhs([1, 3, 2, 2, 5, 2, 3, 7]))  # 5
print(find_lhs([1, 2, 3, 4]))               # 2
print(find_lhs([1, 1, 1, 1]))               # 0 (没有相差为1的数对)
```

**复杂度**：O(n) 时间，O(n) 空间。

**面试话术**："用 Counter 统计每个数的频率，然后只需要检查 num 和 num+1 这两种数的出现次数之和。不需要真的去构造子序列——题目问的是长度。"

## 真题 5：最长连续序列（LeetCode 128）

前文已有详解，这里补充关键点。这题的妙处在于**只从起点开始数**：如果 num-1 在集合中，num 肯定不是起点，跳过。每个元素最多被枚举一次，所以是 O(n)。

**面试追问应对**：

问："内层 while 循环不是让时间复杂度变成 O(n²) 吗？"
答："不是。每个元素只会作为某个连续序列的起点被访问一次——因为只有在 num-1 不在集合中时（即 num 是起点），才会进入内层 while。内层每个元素也最多被访问一次。总共每个元素最多被内层和外层各访问一次，所以是 O(2n) = O(n)。"

## 哈希表 vs 数组查找 vs 二分查找

这个对比表面试被问到的概率极高：

| 维度 | 哈希表 | 数组（顺序查找） | 排序数组 + 二分查找 |
|------|--------|----------------|-------------------|
| **查找** | O(1) 平均 | O(n) | O(log n) |
| **插入** | O(1) 平均 | O(1) 尾部 | O(n)（需要移动元素） |
| **删除** | O(1) 平均 | O(n) | O(n)（需要移动元素） |
| **额外空间** | O(n) | O(1) | O(1) |
| **有序性** | 无序 | 取决于插入方式 | 有序（支持范围查询） |
| **缓存友好** | 差（随机访问） | 极好（连续内存） | 好 |
| **最坏情况** | O(n)（大量冲突） | O(n) | O(log n) |
| **适用场景** | 频繁查找、不关心顺序 | 数据量极小 | 有序数据、范围查询、空间敏感 |
| **面试代表题** | Two Sum | 基本不考 | 搜索插入位置、旋转数组查找 |

**选择决策树**：
1. 需要范围查询（找所有介于 [a, b] 的元素）？→ 二分查找
2. 内存非常紧张？→ 二分查找（就地查找）
3. 数据量小（< 20）？→ 暴力遍历可能比哈希表快（常数因子小）
4. 其他情况 → 哈希表

## 面试专题

### 哈希表最坏情况 O(n) 是怎么来的？

**一句话回答**：所有键的哈希值都冲突了，哈希表退化成链表。

具体来说：
- **链表法**：所有键映射到同一个槽位，那个槽位的链表长度变成 n，查找效率退化为 O(n)
- **开放寻址法**：连续冲突导致每次插入/查找都要探测大量位置

**什么情况下会大量冲突？**
1. 哈希函数设计差，分布不均匀
2. 攻击者恶意构造输入（哈希碰撞攻击），比如精心构造一批 key 让它们映射到同一个槽位
3. 负载因子过高，数组太满

**如何避免？**
1. Python 3.x 使用了随机化的哈希种子（启动时随机生成），防止攻击者预测哈希值
2. 控制负载因子，超过阈值就扩容（Python dict 约在 2/3 时扩容）
3. Java HashMap 在链表长度 > 8 时转为红黑树（O(log n) 兜底）
4. 使用加密级哈希函数（SipHash 等）防止碰撞攻击

### 面试高频追问

**问：Python 的 dict key 为什么必须是不可变的？**
答：因为哈希值必须在对象的整个生命周期内保持不变。如果 key 是可变的（比如 list），修改内容后哈希值变了，dict 就找不到这个 key 了。这会导致内存泄漏和查找失败。

**问：哈希表为什么要扩容？扩容时发生了什么？**
答：负载因子过高会导致冲突增多、性能下降。扩容时创建一个更大的数组（通常是原来的 2 倍），然后对所有已有元素重新计算哈希并插入新数组（rehash）。这是一个 O(n) 的昂贵操作，但均摊到每次插入仍然是 O(1)。

**问：LRU Cache 为什么不用自带有序性的 OrderedDict 或者 Python 3.7+ 的普通 dict？**
答：Python 3.7+ 的 dict 虽然按插入顺序遍历，但不支持"把已有元素移到末尾"这样的 O(1) 操作（需要先删除再插入）。OrderedDict 有 `move_to_end()` 方法可以直接实现 LRU，一行代码就搞定。面试时如果允许用 OrderedDict 可以简化很多：

```python
from collections import OrderedDict

class LRUCache:
    def __init__(self, capacity):
        self.capacity = capacity
        self.cache = OrderedDict()

    def get(self, key):
        if key not in self.cache:
            return -1
        self.cache.move_to_end(key)
        return self.cache[key]

    def put(self, key, value):
        if key in self.cache:
            self.cache.move_to_end(key)
        self.cache[key] = value
        if len(self.cache) > self.capacity:
            self.cache.popitem(last=False)  # FIFO，删除最旧的
```

**问：Two Sum 用哈希表 vs 排序+双指针，选哪个？**
答：哈希表 O(n) 时间 + O(n) 空间；排序+双指针 O(n log n) 时间 + O(1) 空间。如果只问一对解，哈希表更简单。如果要求返回所有不重复数对，排序+双指针更好，因为去重逻辑更自然。如果数组已经有序，双指针 O(n) 时间 + O(1) 空间完胜。

## 面试高频考点小结

| 考点 | 出现频率 | 核心技巧 | 易错点 |
|------|---------|---------|--------|
| Two Sum 及变体 | ★★★★★ | 哈希表存储已遍历元素 | 忘记处理重复元素、返回值是下标不是数 |
| 字母异位词分组 | ★★★★ | 排序后作为 key 或计数数组作为 key | defaultdict 的用法 |
| 最长连续序列 | ★★★★ | set + 只从起点开始数 | 误认为 O(n²) |
| LRU Cache | ★★★★★ | 哈希表 + 双向链表 | 链表操作（删节点、插头部）容易写错 |
| 同构字符串 | ★★★ | 双向映射（两个字典） | 只做单向映射，漏掉一对一约束 |
| 快乐数 | ★★★ | set 检测循环 或 快慢指针 | 平方和计算逻辑 |
| 最长和谐子序列 | ★★ | Counter + 检查 num+1 | 没理解"子序列"可以不相邻 |
| 哈希冲突解决 | ★★★★ | 链表法 vs 开放寻址 | 搞混两种方案的具体实现 |
| Python dict 内部 | ★★★ | 开放寻址、约 2/3 扩容、3.7+ 有序 | 误以为 dict 天然支持随机访问 |
