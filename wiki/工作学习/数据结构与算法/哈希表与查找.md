# 哈希表与查找

面试最常考的三种查找：哈希表 O(1)、排序+二分 O(log n)、暴力 O(n)。核心是**什么时候用哪个**。

## 哈希表的本质

一个键值对集合，通过哈希函数把键映射到数组索引，实现 O(1) 的增删查。

```python
# Python 字典就是哈希表
seen = {}
seen["key"] = 1         # 插入 O(1)
val = seen.get("key")   # 查找 O(1)
del seen["key"]         # 删除 O(1)
```

**面试话术**："哈希表把任意类型的键通过哈希函数转成整数，用这个整数当数组下标。冲突用链表法（拉链）或开放寻址解决。Python 的 dict 平均 O(1)，最坏 O(n)（所有键都冲突）。"

## 真题 1：最长连续序列

LeetCode 128。考哈希表思维，不是代码量。

**题目**：`nums = [100, 4, 200, 1, 3, 2]`，最长连续序列是 `[1,2,3,4]`，长度 4。

```python
def longest_consecutive(nums):
    num_set = set(nums)  # O(n) 建集合
    longest = 0

    for num in num_set:
        # 关键优化：只从序列的起点开始数
        # 如果 num-1 在集合里，说明 num 不是起点，跳过
        if num - 1 not in num_set:
            current = num
            current_streak = 1

            while current + 1 in num_set:
                current += 1
                current_streak += 1

            longest = max(longest, current_streak)

    return longest
```

**为什么是 O(n)？** 外层循环每个元素访问一次，内层 while 看起来像嵌套循环，但**每个元素只在以它为起点时被内层访问一次**。总共每个元素走一次，O(n)。

**面试话术**："用集合存所有元素，只从序列起点开始向后数。关键优化是跳过非起点元素的检查，保证每个元素只被数一次。"

## 真题 2：字母异位词分组

LeetCode 49。用哈希表做分组。

**题目**：`strs = ["eat","tea","tan","ate","nat","bat"]`，把字母相同的词分一组。

```python
from collections import defaultdict

def group_anagrams(strs):
    groups = defaultdict(list)

    for s in strs:
        # 排序后作为键：eat→aet, tea→aet, tan→ant
        key = "".join(sorted(s))  # O(k log k)，k 是字符串长度
        groups[key].append(s)

    return list(groups.values())

# 返回 [["eat","tea","ate"], ["tan","nat"], ["bat"]]
```

**优化版（如果字符串很长）**：
```python
def group_anagrams_optimized(strs):
    groups = defaultdict(list)
    for s in strs:
        # 用 26 个字母的计数作为键，O(k)
        count = [0] * 26
        for ch in s:
            count[ord(ch) - ord('a')] += 1
        groups[tuple(count)].append(s)
    return list(groups.values())
```

## 真题 3：LRU 缓存

LeetCode 146。面试高频，考**哈希表 + 双向链表**的组合。

```python
class Node:
    def __init__(self, key=0, value=0):
        self.key = key
        self.value = value
        self.prev = None
        self.next = None

class LRUCache:
    def __init__(self, capacity: int):
        self.capacity = capacity
        self.cache = {}           # key → Node，O(1) 查找
        self.head = Node()        # 哨兵头节点
        self.tail = Node()        # 哨兵尾节点
        self.head.next = self.tail
        self.tail.prev = self.head

    def _remove(self, node):
        """从链表中移除节点"""
        node.prev.next = node.next
        node.next.prev = node.prev

    def _add_to_head(self, node):
        """把节点插到头部（最近使用）"""
        node.next = self.head.next
        node.prev = self.head
        self.head.next.prev = node
        self.head.next = node

    def get(self, key: int) -> int:
        if key not in self.cache:
            return -1
        node = self.cache[key]
        self._remove(node)        # 从当前位置移除
        self._add_to_head(node)   # 移到头部（标记为最近使用）
        return node.value

    def put(self, key: int, value: int) -> None:
        if key in self.cache:
            self._remove(self.cache[key])
        node = Node(key, value)
        self.cache[key] = node
        self._add_to_head(node)
        if len(self.cache) > self.capacity:
            # 删除尾部（最久未使用）
            lru = self.tail.prev
            self._remove(lru)
            del self.cache[lru.key]
```

**设计要点**：
- 哈希表负责 O(1) 查找
- 双向链表负责维护访问顺序（最近用的放头部，最久不用放尾部）
- 两个哨兵节点简化边界处理

## 哈希 vs 排序 vs 暴力

| | 哈希表 | 排序+二分 | 暴力 |
|------|--------|---------|------|
| 查找 | O(1) | O(log n) | O(n) |
| 额外空间 | O(n) | O(1) | O(1) |
| 适合场景 | 多次查找，不在意空间 | 有序数据，一次排序多次查找 | 数据量极小 |
| 面试高频 | Two Sum, LRU | 二分查找第一个/最后一个 | 基本不考 |
