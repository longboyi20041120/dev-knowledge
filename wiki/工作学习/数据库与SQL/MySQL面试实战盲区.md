# MySQL 面试实战盲区

> 锁机制、慢查询、大表拆分、主从延迟、HTAP——大厂面试都问生产问题。 | **面试重要度：极高** | 预计阅读：12 分钟

## 视频资源

- B站: [MySQL 面试连环炮](https://www.bilibili.com/video/BV1GT4y1U7vt/) — MySQL 面试高频题一网打尽

大厂 MySQL 面试的题，几乎都**与生产强绑定**——不是问语法，而是问"线上出了问题你怎么办"。这篇按四类盲区梳理高频问法和排查方向，配合基础笔记一起看。

## 一、锁机制类

| 问题 | 排查/解决方向 |
|------|--------------|
| 表锁什么时候触发 | 无合适索引导致行锁升级为表锁、MyISAM 引擎、DDL 操作 |
| 锁阻塞怎么定位 | 查 `information_schema.INNODB_TRX`、`INNODB_LOCKS`、`INNODB_LOCK_WAITS`；`SHOW ENGINE INNODB STATUS` |
| 死锁怎么排查 | 看死锁日志（`SHOW ENGINE INNODB STATUS` 的 LATEST DETECTED DEADLOCK） |
| 死锁通用解决 | 固定加锁顺序、缩小事务范围、加合适索引避免锁升级 |

## 二、查询优化类

| 问题 | 排查/解决方向 |
|------|--------------|
| 慢查询怎么观察 | 开慢查询日志（`slow_query_log`）、设 `long_query_time`、用 `EXPLAIN` 分析 |
| 联表查询怎么优化 | 小表驱动大表、被驱动表关联字段加索引、减少 JOIN 表数量 |
| 大表分页越翻越慢 | 深分页 `LIMIT 100000,10` 会扫大量行 → 用**游标/延迟关联**（先按主键定位再回表） |
| 索引没命中怎么查 | `EXPLAIN` 看 `type`/`key`；排查最左前缀、函数操作列、隐式类型转换、`OR` 等失效场景 |

## 三、生产风险类

| 问题 | 要点 |
|------|------|
| 大表查询的风险 | 全表扫描拖垮 IO、锁范围大、影响其他请求 |
| 大表怎么拆 | 垂直拆分（拆列/拆表）、水平拆分（分库分表，按 ID/时间 hash） |
| 高频写入主从延迟 | 主从复制延迟 → **限流**、读写分离时强制走主库读、半同步复制 |

## 四、选型对比类

| 问题 | 要点 |
|------|------|
| HTAP 数据库 | 同时支持 **OLTP（事务）+ OLAP（分析）**，省去 ETL 同步，适合实时分析场景（如 TiDB） |

## 排查心法

面试官问这些，本质是想看你**有没有生产经验**。回答套路：**先定位（用什么工具看）→ 再分析（根因是什么）→ 后解决（对应手段）**，别只背结论。
