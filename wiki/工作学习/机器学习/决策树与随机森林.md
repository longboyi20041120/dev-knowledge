## 一、决策树：三个核心概念

### 信息熵（衡量不确定性）

```
熵 H = -Σ p_i × log₂(p_i)

例子：
- 全是正例（p=1.0）：熵 = 0（完全确定）
- 一半正一半负（p=0.5）：熵 = 1（最不确定）
```

### 信息增益

分裂前的熵 - 分裂后的加权平均熵。**信息增益越大，说明这个特征越适合用来分裂。**

### 三种划分标准的详细对比

| 标准 | 公式 | 取值范围 | 计算复杂度 | 特点 | 使用算法 |
|------|------|----------|------------|------|----------|
| **信息熵（Entropy）** | -Σ p_i × log₂(p_i) | [0, 1]（二分类） | 含 log，稍慢 | 对不纯度变化敏感，偏向多值特征 | ID3, C4.5 |
| **基尼系数（Gini）** | 1 - Σ p_i² | [0, 0.5]（二分类） | 只有平方，快 | 与熵效果接近但计算更快 | CART（sklearn 默认） |
| **误分类率** | 1 - max(p_i) | [0, 0.5]（二分类） | 最简单 | 对概率变化不敏感，不推荐用于分裂 | 偶尔用于剪枝 |

#### 计算示例

假设一个节点有 100 个样本，60 个正类，40 个负类：

```
p_正 = 0.6, p_负 = 0.4

熵 = -(0.6 × log₂(0.6) + 0.4 × log₂(0.4))
   = -(0.6 × (-0.737) + 0.4 × (-1.322))
   = 0.971

基尼系数 = 1 - (0.6² + 0.4²) = 1 - (0.36 + 0.16) = 0.48

误分类率 = 1 - max(0.6, 0.4) = 1 - 0.6 = 0.4
```

```python
import numpy as np

def entropy(p):
    """计算二分类熵，p 为正类比例"""
    if p == 0 or p == 1:
        return 0
    return -(p * np.log2(p) + (1-p) * np.log2(1-p))

def gini(p):
    """计算基尼系数"""
    return 1 - (p**2 + (1-p)**2)

def misclassification(p):
    """计算误分类率"""
    return 1 - max(p, 1-p)

# 对比三种标准随 p 的变化
import matplotlib.pyplot as plt

ps = np.linspace(0.001, 0.999, 100)
entropies = [entropy(p) for p in ps]
ginis = [gini(p) for p in ps]
miscls = [misclassification(p) for p in ps]

print(f"p=0.5 时: 熵={entropy(0.5):.3f}, Gini={gini(0.5):.3f}, 误分类率={misclassification(0.5):.3f}")
print(f"p=0.9 时: 熵={entropy(0.9):.3f}, Gini={gini(0.9):.3f}, 误分类率={misclassification(0.9):.3f}")
# 输出:
# p=0.5 时: 熵=1.000, Gini=0.500, 误分类率=0.500
# p=0.9 时: 熵=0.469, Gini=0.180, 误分类率=0.100
# 可见 Gini 和熵的曲线形状非常接近，Gini 值约为熵的一半
```

**为什么 sklearn 默认用 Gini 而不是熵？**
1. Gini 只有乘法和加法运算，计算速度比熵（需要 log）快
2. 实际效果和熵差异极小，Gini 分裂的树和熵分裂的树几乎一样
3. Gini 天然适合二叉树（CART 标准配置）

## 二、CART 树生成过程详解

CART（Classification and Regression Tree）是 sklearn 使用的建树算法。它的特点是：**二叉树、基尼系数、递归分裂**。

### 手动走一遍 CART 建树过程

用经典的"天气-打球"数据集：

| 天气 | 温度 | 湿度 | 有风 | 打球？ |
|------|------|------|------|--------|
| 晴 | 热 | 高 | 否 | 不打 |
| 晴 | 热 | 高 | 是 | 不打 |
| 阴 | 热 | 高 | 否 | 打 |
| 雨 | 温和 | 高 | 否 | 打 |
| 雨 | 冷 | 正常 | 否 | 打 |
| 雨 | 冷 | 正常 | 是 | 不打 |
| 阴 | 冷 | 正常 | 是 | 打 |
| 晴 | 温和 | 高 | 否 | 不打 |
| 晴 | 冷 | 正常 | 否 | 打 |
| 雨 | 温和 | 正常 | 否 | 打 |

**根节点的基尼系数**：打球 6 次，不打 4 次 → Gini = 1 - (0.6² + 0.4²) = 0.48

**尝试按"天气"分裂**：
- 天气=晴（3 样本）：打 1，不打 2 → Gini = 1-(0.33²+0.67²)=0.444
- 天气=阴（2 样本）：打 2，不打 0 → Gini = 0（完美）
- 天气=雨（5 样本）：打 3，不打 2 → Gini = 1-(0.6²+0.4²)=0.48

加权 Gini = (3×0.444 + 2×0 + 5×0.48)/10 = 0.373

**按"有风"分裂**：
- 有风=是（3 样本）：打 1，不打 2 → Gini = 0.444
- 有风=否（7 样本）：打 5，不打 2 → Gini = 1-(0.714²+0.286²)=0.408

加权 Gini = (3×0.444 + 7×0.408)/10 = 0.419

比较：天气(0.373) < 有风(0.419) < 根节点(0.48)，选择"天气"作为根节点。

然后对每个分支递归重复这个过程，直到满足停止条件。

```python
# CART 建树的核心递归逻辑（伪代码形式）
def build_cart_tree(X, y, depth=0, max_depth=5):
    # 停止条件
    if depth >= max_depth:
        return majority_class(y)
    if len(np.unique(y)) == 1:       # 节点纯净，不分裂
        return y[0]
    if len(y) < min_samples_split:   # 样本太少，不分裂
        return majority_class(y)

    # 找到最佳分裂点
    best_feature, best_threshold, best_gini = find_best_split(X, y)

    if best_feature is None:  # 无法继续分裂
        return majority_class(y)

    # 按最佳阈值分裂数据
    left_idx = X[:, best_feature] <= best_threshold
    right_idx = ~left_idx

    # 递归建左右子树
    left_tree = build_cart_tree(X[left_idx], y[left_idx], depth+1, max_depth)
    right_tree = build_cart_tree(X[right_idx], y[right_idx], depth+1, max_depth)

    return {
        'feature': best_feature,
        'threshold': best_threshold,
        'left': left_tree,
        'right': right_tree
    }
```

## 三、从零构建决策树

```python
import numpy as np

def gini(y):
    """基尼系数：越小越纯"""
    _, counts = np.unique(y, return_counts=True)
    probs = counts / len(y)
    return 1 - np.sum(probs ** 2)

def best_split(X, y):
    """穷举所有特征和阈值，找到基尼系数最小的分裂"""
    best_gini = float("inf")
    best_feature, best_threshold = None, None

    for feat in range(X.shape[1]):
        thresholds = np.unique(X[:, feat])
        for thresh in thresholds:
            left = y[X[:, feat] <= thresh]
            right = y[X[:, feat] > thresh]
            if len(left) == 0 or len(right) == 0:
                continue
            g = (len(left) * gini(left) + len(right) * gini(right)) / len(y)
            if g < best_gini:
                best_gini = g
                best_feature = feat
                best_threshold = thresh

    return best_feature, best_threshold
```

## 四、过拟合与剪枝

决策树最大的问题：**容易过拟合**。如果不限制，树会一直分裂到每个叶子只有一个样本。

### 预剪枝（Pre-pruning）

在树生长过程中提前停止分裂。sklearn 中通过超参数控制：

```python
from sklearn.tree import DecisionTreeClassifier

# 不限制 → 过拟合（训练集 100%，测试集 70%）
dt = DecisionTreeClassifier()

# 加约束 → 泛化好
dt = DecisionTreeClassifier(
    max_depth=5,           # 最大深度：超过就不再分裂
    min_samples_split=10,  # 节点最少样本数：少于就不分裂
    min_samples_leaf=5,    # 叶子最少样本数：分裂后叶子样本少于这个数就不分裂
    max_features="sqrt",   # 每次分裂只考虑 sqrt(n) 个特征
    min_impurity_decrease=0.001,  # 不纯度降低小于此值就不分裂
)
```

| 参数 | 作用机制 | 调大/调小的效果 |
|------|----------|----------------|
| `max_depth` | 限制树的深度 | 调小防过拟合，调大让模型更复杂 |
| `min_samples_split` | 节点少于 N 个样本停止分裂 | 调大防过拟合，叶子更"粗" |
| `min_samples_leaf` | 叶子至少 N 个样本 | 调大减少噪声影响 |
| `min_impurity_decrease` | 不纯度降低不够就不分裂 | 过滤"无效"分裂 |

### 后剪枝（Post-pruning / CCP）

先让树长到最大（甚至过拟合），再从底部往上"砍掉"不重要的分支。

**代价复杂度剪枝（Cost Complexity Pruning，CCP）**：

```
R_α(T) = R(T) + α × |T|

R(T): 树的训练误差（误分类率）
|T|:  叶子节点数
α:    复杂度惩罚系数
```

α=0 时完全不剪枝（树最大），α 越大剪得越狠。sklearn 用 `ccp_alpha` 参数控制：

```python
# 通过 CCP 路径选择最佳 α
from sklearn.tree import DecisionTreeClassifier

# 先训练一棵完整的树
dt = DecisionTreeClassifier(random_state=42)
path = dt.cost_complexity_pruning_path(X_train, y_train)

alphas = path.ccp_alphas[:-1]  # 去掉最后一个（只剩根节点的 α）

# 遍历不同的 α 值，找到验证集上最好的
best_alpha = None
best_score = 0
for alpha in alphas:
    dt_pruned = DecisionTreeClassifier(random_state=42, ccp_alpha=alpha)
    dt_pruned.fit(X_train, y_train)
    score = dt_pruned.score(X_val, y_val)
    if score > best_score:
        best_score = score
        best_alpha = alpha

print(f"最佳 ccp_alpha: {best_alpha:.5f}, 验证集准确率: {best_score:.3f}")
```

### 预剪枝 vs 后剪枝对比

| 对比维度 | 预剪枝 | 后剪枝 |
|----------|--------|--------|
| 时机 | 边建树边剪 | 建完树再剪 |
| 计算量 | 小（省掉了长枝叶的开销） | 大（树先长满再砍） |
| 欠拟合风险 | 高（可能过早停止，错过好的分裂） | 低（先充分生长再修剪） |
| 实际使用 | sklearn 默认使用预剪枝超参数 | 用 `ccp_alpha` 显式启用 |

## 五、过拟合可视化演示

```python
import numpy as np
from sklearn.tree import DecisionTreeClassifier
from sklearn.model_selection import train_test_split

X, y = make_classification(

                   (15, train_scores[14], test_scores[14]),
```

## 六、随机森林：多棵树投票

### "随机"二字深入解释

随机森林有两个随机性来源，缺一不可：

**1. Bootstrap 采样（样本随机）**

从 n 个原始样本中有放回地抽样 n 次。一个样本不被抽中的概率 = (1 - 1/n)^n ≈ 1/e ≈ 0.368，即大约 **63%** 的样本会被抽中，约 37% 的样本从未出现在这棵树的训练集中——这些就是 OOB（Out-Of-Bag）样本。

```python
# 直观验证 Bootstrap 的 63% 规律
np.random.seed(42)
n = 10000
bootstrap_sample = np.random.choice(n, size=n, replace=True)
unique_ratio = len(np.unique(bootstrap_sample)) / n
print(f"Bootstrap 后不重复样本比例: {unique_ratio:.3f}")
# 输出: ~0.632，和理论值 1 - 1/e 非常接近
```

OOB 样本的好处：可以免费当验证集用，不用额外划分数据来评估模型。

**2. 随机特征子集（特征随机）**

每次分裂时，不是从所有 d 个特征中选最佳分裂，而是先随机选 k 个特征的子集，再从中选最佳。通常：
- 分类问题：k = sqrt(d)
- 回归问题：k = d/3

**为什么要两个随机？**

如果只用 Bootstrap 而不用特征随机：每棵树虽然训练数据不同，但最强的几个特征会在每棵树的根节点附近反复出现 → 树之间高度相关 → 投票效果大打折扣。

如果只用特征随机而不用 Bootstrap：样本完全相同，只是每次分裂看的特征不同 → 树的多样性不够。

两个随机结合起来 → 每棵树的训练数据和分裂依据都不同 → 树的多样性最大化 → Bagging 降方差效果最好。

```python
from sklearn.ensemble import RandomForestClassifier

rf = RandomForestClassifier(
    n_estimators=100,      # 100 棵树
    max_depth=10,
    random_state=42,
    oob_score=True,        # 启用 OOB 评估
)
rf.fit(X_train, y_train)
print(f"OOB 得分: {rf.oob_score_:.3f}")  # 不依赖测试集的评估

# 特征重要性（面试常问：怎么判断哪些特征重要）
for name, importance in zip(feature_names, rf.feature_importances_):
    print(f"{name}: {importance:.3f}")
```

### 特征重要性的两种计算方法

**方法一：基于基尼不纯度减少（MDI，Mean Decrease in Impurity）**

sklearn 默认使用这个方法。每个特征在树中每次分裂时，都会带来基尼不纯度的降低。把所有树、所有分裂中该特征"贡献"的不纯度减少加起来并归一化，就是特征重要性。

```python
# MDI 是 sklearn feature_importances_ 的默认计算方法
# 内部原理：每次用特征 f 分裂时，记录
# importance += (分裂前样本数 × Gini前 - 左子节点样本数 × Gini左 - 右子节点样本数 × Gini右)
# 最后除以所有 feature 的 importance 之和做归一化
```

**局限**：MDI 偏向于取值的特征（如编号类特征有太多取值，分裂点多，累计贡献就多）和高基数特征。

**方法二：基于 OOB 置换重要性（Permutation Importance）**

打乱某个特征的值，看 OOB 准确率下降多少。下降越多说明这个特征越重要。这个方法无偏、和模型无关。

```python
from sklearn.inspection import permutation_importance

# 在测试集上计算置换重要性
result = permutation_importance(
    rf, X_test, y_test,
    n_repeats=10,          # 重复打乱 10 次取平均
    random_state=42,
    scoring='accuracy'
)

for i in range(min(5, len(feature_names))):
    print(f"{feature_names[i]}: {result.importances_mean[i]:.4f} ± {result.importances_std[i]:.4f}")
```

| 对比 | MDI（默认） | 置换重要性 |
|------|-------------|------------|
| 计算时机 | 训练过程中累积 | 训练完成后 |
| 是否需要测试集 | 不需要 | 需要 |
| 偏向 | 偏向多值特征 | 无偏 |
| 速度 | 快 | 慢（需要多次打乱+预测） |

### 随机森林为什么比单棵决策树好？

① **Bagging 降低方差**：每棵树用不同数据训练，取平均后噪声互相抵消
② **随机特征选择降低相关性**：如果每棵树都一样，投票没意义；特征随机保证树之间有差异
③ **方差低 = 过拟合风险小**

用数学语言：单棵树的方差 = σ²，"棵棵独立"随机森林的方差 = σ² / n_estimators。虽然实际中树不是完全独立（只用了 63% 不同样本），但方差确实大幅降低。

## 七、Bagging vs Boosting

| 对比维度 | Bagging（随机森林） | Boosting（GBDT/XGBoost） |
|----------|---------------------|--------------------------|
| 训练方式 | 并行（各树独立训练） | 串行（每棵树依赖前一棵的结果） |
| 核心思想 | 投票/平均降低方差 | 逐步修正残差降低偏差 |
| 样本权重 | 每次 Bootstrap 采样，平等对待 | 每轮调整样本权重，错分样本权重变大 |
| 对噪声敏感度 | 低（平均抵消噪声） | 高（会过度学习噪声） |
| 过拟合风险 | 低（树越多越稳定） | 高（轮数太多会过拟合） |
| 典型算法 | Random Forest | AdaBoost, GBDT, XGBoost, LightGBM |
| GPU 加速 | 天然支持并行 | XGBoost/LightGBM 支持 |

```python
# Bagging vs Boosting 效果示意
from sklearn.ensemble import RandomForestClassifier, GradientBoostingClassifier

# Bagging（并行）
rf = RandomForestClassifier(n_estimators=100, random_state=42)
rf.fit(X_train, y_train)

# Boosting（串行）
gbdt = GradientBoostingClassifier(n_estimators=100, random_state=42)
gbdt.fit(X_train, y_train)

print(f"随机森林 训练: {rf.score(X_train, y_train):.3f} 测试: {rf.score(X_test, y_test):.3f}")
print(f"GBDT     训练: {gbdt.score(X_train, y_train):.3f} 测试: {gbdt.score(X_test, y_test):.3f}")
# 典型输出: GBDT 训练集更高（拟合能力强），但测试集差距也更大
```

### XGBoost / LightGBM 概念引入

**为什么工业界逐渐用 XGBoost / LightGBM 替代随机森林？**

1. **效果更好**：Boosting 逐棵修正残差，在大规模表格数据上通常比 Bagging 的随机森林效果好
2. **支持自定义损失函数**：可以优化任意可导函数（MAE、Huber、自定义业务损失）
3. **内置正则化**：XGBoost 的目标函数 = 损失 + 树复杂度正则化（叶节点数 + 叶权重平方和），天然防过拟合
4. **工程优化**：LightGBM 使用直方图算法（Histogram）和 leaf-wise 生长策略，训练速度快一个数量级，内存占用小
5. **缺失值处理**：自动学习缺失值分到左边还是右边，不用手动填充

```
XGBoost 目标函数 = Σ loss(y, y_pred) + γ × 叶节点数 + (1/2)λ × Σ 叶权重²

第一项：拟合误差
第二项：惩罚叶子太多（树太复杂）
第三项：惩罚叶子权重太大（个别叶子影响过大）
```

**什么时候仍然用随机森林？**
- 数据噪声大（Boosting 容易过拟合噪声）
- 需要并行训练且数据量极大
- 需要 OOB 免费评估
- 模型上线后不需要频繁更新

## 八、决策树 vs 线性模型

| | 决策树/随机森林 | 线性回归/逻辑回归 |
|------|------|------|
| 特征关系 | 自动捕捉非线性 | 只能学线性关系 |
| 特征工程 | 不需要标准化 | 需要 |
| 可解释性 | 树结构直观（特征重要性） | 系数直接表示影响 |
| 小数据 | 容易过拟合 | 更稳定 |
| 大数据 | 随机森林训练慢（100棵树） | 快 |
| 典型场景 | 表格数据分类、推荐系统 | 评分卡、风控 |

**面试话术**："线性模型假设线性关系，树模型自动学非线性。不知道数据什么关系就先上随机森林，效果通常不错。需要解释每个特征怎么影响结果就用逻辑回归。"

## 九、面试常见追问

**Q: 随机森林为什么随机？两个随机分别在哪？**

A: ① 样本随机：每棵树 Bagging 采样（有放回抽样约 63% 样本） ② 特征随机：每次分裂只考虑部分特征（通常 sqrt(n)）。两个随机让树之间多样化。

**Q: GBDT 和随机森林的区别？**

A: 随机森林 Bagging（并行训练多棵树取平均），GBDT Boosting（串行训练，每棵新树修正前一棵的残差）。Boosting 偏差更低但更容易过拟合。

**Q: 决策树怎么处理连续特征和类别特征？**

A: 连续特征：排序后遍历每个相邻值的中点作为候选分裂阈值，挑 Gini 最小的。类别特征：CART 做二叉分裂时，枚举所有"一类 vs 其他类"的组合（one-vs-rest）。

**Q: 随机森林中树的深度、数量怎么选？**

A: 深度先不设限制（或设一个较大值如 20），观察 OOB 误差随树数量（n_estimators）的收敛曲线——通常 100-300 棵后误差稳定。然后再调 max_depth 防止个别树过深。

## 面试高频考点小结

| 考点 | 频率 | 关键点 |
|------|------|--------|
| 熵 vs Gini vs 误分类率 | 极高 | 公式、取值范围、Gini 计算快 |
| 决策树过拟合原因 | 极高 | 无限分裂到叶子只有一个样本 |
| 预剪枝 vs 后剪枝 | 高 | pre：建树时停；post：建完再砍 |
| 随机森林两个随机 | 极高 | Bootstrap 采样 + 随机特征子集 |
| Bagging vs Boosting | 极高 | 并行降方差 vs 串行降偏差 |
| 随机森林比单棵好的原理 | 高 | Bagging 降方差，方差 = σ²/N |
| 特征重要性计算 | 高 | MDI（默认）vs 置换重要性（无偏） |
| OOB 误差是什么 | 中高 | 37% 未采样样本做免费验证集 |
| XGBoost 的目标函数 | 中 | 损失 + 叶节点数惩罚 + 叶权重惩罚 |
| CART 建树过程 | 中 | 二叉树、Gini 系数、递归分裂 |
