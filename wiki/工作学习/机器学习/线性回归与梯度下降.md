# 线性回归与梯度下降

ML 面试最基础的环节——手写公式、解释原理、代码实现。

## 一、线性回归的数学

**目标**：找一条直线 y = wx + b，最小化预测值和真实值的差距。

**损失函数（MSE）**：
```
MSE = (1/n) × Σ(y_true - y_pred)²
```

**为什么用平方？** ① 正负误差不抵消 ② 大误差惩罚更重（平方放大） ③ 数学上好求导。

### 矩阵形式推导（正规方程）

当特征不止一个时，用矩阵表示更方便。假设有 n 个样本，d 个特征：

```
X ∈ R^(n×d)    特征矩阵（第一列通常是全 1，对应截距项）
y ∈ R^(n×1)    标签向量
w ∈ R^(d×1)    参数向量
```

预测值：`y_pred = Xw`

MSE 损失函数写成矩阵形式：
```
J(w) = (1/n) × (y - Xw)ᵀ(y - Xw)
     = (1/n) × (yᵀy - yᵀXw - wᵀXᵀy + wᵀXᵀXw)
     = (1/n) × (yᵀy - 2wᵀXᵀy + wᵀXᵀXw)    ← yᵀXw 是标量，等于 wᵀXᵀy
```

对 w 求导并令其为 0：
```
∂J/∂w = (2/n) × (-Xᵀy + XᵀXw) = 0
→ XᵀXw = Xᵀy
→ w = (XᵀX)^(-1) Xᵀy        ← 正规方程的解
```

**什么时候正规方程不可用？**
- XᵀX 不可逆（特征数 > 样本数，或特征间存在完美共线性）
- 样本量 n 太大时，矩阵求逆 O(d³) 不现实

```python
import numpy as np

# 正规方程求解
np.random.seed(42)
X = np.random.randn(100, 3)          # 100 样本，3 特征
true_w = np.array([[2.0], [-1.0], [3.0]])
y = X @ true_w + np.random.randn(100, 1) * 0.5

# 正规方程一步到位
X_b = np.c_[np.ones((100, 1)), X]     # 加截距列
w_normal = np.linalg.inv(X_b.T @ X_b) @ X_b.T @ y
print("正规方程解:", w_normal.ravel())
# 输出: 正规方程解: [0.032  2.006 -1.012  3.008]  ← 第一项是截距 b
```

### 正规方程 vs 梯度下降

| 对比维度 | 正规方程 | 梯度下降 |
|----------|----------|----------|
| 求解方式 | 直接求解析解 | 迭代逼近 |
| 计算复杂度 | O(d³)（矩阵求逆） | O(nd × 迭代次数) |
| 适用场景 | d < 10000，n 不太大 | 大数据、大特征维度 |
| 需要调参 | 不需要 | 需要调学习率、迭代次数 |
| 特征缩放 | 不需要 | **必须做**，否则收敛慢 |
| 可扩展性 | 无法用于复杂模型 | 可用于神经网络等任意可导模型 |

### 面试专题：为什么用梯度下降而不是直接解正规方程？

> **面试官**："正规方程能直接算出来，为什么还要梯度下降？"

**标准回答**：
1. **计算量**：正规方程需要计算 (XᵀX)^(-1)，求逆的复杂度是 O(d³)。当特征维度 d 超过 10000 时，求逆在工程上不可接受。梯度下降每次迭代只需 O(nd)，可以通过控制迭代次数来平衡精度和速度。
2. **通用性**：正规方程只适用于线性回归。逻辑回归、神经网络等模型没有解析解，必须用梯度下降。面试中体现你对"优化方法"的通用理解。
3. **内存限制**：XᵀX 是一个 d×d 的矩阵，d=100万时内存直接爆了。梯度下降不需要存储这个矩阵。

> **追问**："那数据量小、特征少时用哪个？"

答：直接用正规方程，一步到位，没有学习率调参的烦恼。sklearn 的 `LinearRegression` 内部用的就是基于 SVD 的伪逆，比朴素正规方程更稳定。

## 二、梯度下降：从零实现

```python
import numpy as np

# 生成真实数据：y = 3x + 5 + 噪声
np.random.seed(42)
X = np.random.randn(100, 1)
y = 3 * X + 5 + np.random.randn(100, 1) * 0.5

# 梯度下降
w, b = 0.0, 0.0  # 初始化参数
lr = 0.01        # 学习率
n = len(X)

for epoch in range(1000):
    # 前向传播：预测
    y_pred = w * X + b

    # 计算梯度（MSE 对 w 和 b 求偏导）
    dw = (-2/n) * np.sum(X * (y - y_pred))
    db = (-2/n) * np.sum(y - y_pred)

    # 更新参数
    w -= lr * dw
    b -= lr * db

    if epoch % 200 == 0:
        loss = np.mean((y - y_pred) ** 2)
        print(f"Epoch {epoch}: w={w:.4f}, b={b:.4f}, loss={loss:.4f}")

print(f"\n最终: y = {w[0]:.2f}x + {b[0]:.2f}")
# 输出接近: y = 2.98x + 5.01
```

**面试口述梯度下降步骤**：
1. 随机初始化参数（w, b）
2. 算预测值 y_pred = wx + b
3. 算损失函数对每个参数的梯度（偏导数）
4. 参数沿着**负梯度方向**走一小步（学习率控制步长）
5. 重复 2-4，直到损失不再下降

### 梯度下降的三种变体

| 变体 | 每次用多少数据算梯度 | 优点 | 缺点 |
|------|----------------------|------|------|
| **BGD**（批量梯度下降） | 全部 n 个样本 | 梯度准确，收敛稳定 | 大数据集上每次迭代都很慢，内存吃不消 |
| **SGD**（随机梯度下降） | 1 个样本 | 超快，可在线学习 | 梯度方向不稳定，震荡剧烈，难收敛到最优点 |
| **Mini-batch GD** | m 个样本（通常 32/64/128） | 兼顾速度和稳定，GPU 可并行 | 需要调 batch size |

```python
# 三种梯度下降对比演示
import numpy as np
import matplotlib.pyplot as plt

np.random.seed(42)
X = np.random.randn(1000, 1)
y = 3 * X + 5 + np.random.randn(1000, 1) * 0.5

def bgd_step(X, y, w, b, lr):
    """批量梯度下降：用全部数据"""
    n = len(X)
    y_pred = w * X + b
    dw = (-2/n) * np.sum(X * (y - y_pred))
    db = (-2/n) * np.sum(y - y_pred)
    return w - lr * dw, b - lr * db

def sgd_step(X, y, w, b, lr):
    """随机梯度下降：随机选一个样本"""
    i = np.random.randint(0, len(X))
    y_pred = w * X[i] + b
    dw = -2 * X[i] * (y[i] - y_pred)
    db = -2 * (y[i] - y_pred)
    return w - lr * dw, b - lr * db

def mini_batch_step(X, y, w, b, lr, batch_size=32):
    """小批量梯度下降"""
    idx = np.random.choice(len(X), batch_size, replace=False)
    X_batch, y_batch = X[idx], y[idx]
    y_pred = w * X_batch + b
    dw = (-2/batch_size) * np.sum(X_batch * (y_batch - y_pred))
    db = (-2/batch_size) * np.sum(y_batch - y_pred)
    return w - lr * dw, b - lr * db

# 运行对比
w_bgd, b_bgd = 0.0, 0.0
w_sgd, b_sgd = 0.0, 0.0
w_mbgd, b_mbgd = 0.0, 0.0

losses = {'BGD': [], 'SGD': [], 'Mini-batch': []}

for epoch in range(500):
    # BGD
    w_bgd, b_bgd = bgd_step(X, y, w_bgd, b_bgd, lr=0.01)
    losses['BGD'].append(np.mean((y - (w_bgd * X + b_bgd))**2))

    # SGD
    w_sgd, b_sgd = sgd_step(X, y, w_sgd, b_sgd, lr=0.01)
    losses['SGD'].append(np.mean((y - (w_sgd * X + b_sgd))**2))

    # Mini-batch
    w_mbgd, b_mbgd = mini_batch_step(X, y, w_mbgd, b_mbgd, lr=0.01)
    losses['Mini-batch'].append(np.mean((y - (w_mbgd * X + b_mbgd))**2))

print(f"BGD 最终: w={w_bgd[0]:.3f}, b={b_bgd[0]:.3f}, loss={losses['BGD'][-1]:.4f}")
print(f"SGD 最终: w={w_sgd[0]:.3f}, b={b_sgd[0]:.3f}, loss={losses['SGD'][-1]:.4f}")
print(f"MBGD 最终: w={w_mbgd[0]:.3f}, b={b_mbgd[0]:.3f}, loss={losses['Mini-batch'][-1]:.4f}")
# 输出: BGD 最平滑收敛，SGD 震荡但前期下降快，Mini-batch 介于两者之间
```

## 三、学习率的影响

### 学习率调参实验

```python
import numpy as np

np.random.seed(42)
X = np.random.randn(100, 1)
y = 3 * X + 5 + np.random.randn(100, 1) * 0.3

def train_with_lr(lr, epochs=200):
    w, b = 0.0, 0.0
    losses = []
    for epoch in range(epochs):
        y_pred = w * X + b
        dw = (-2/len(X)) * np.sum(X * (y - y_pred))
        db = (-2/len(X)) * np.sum(y - y_pred)
        w -= lr * dw
        b -= lr * db
        losses.append(np.mean((y - y_pred)**2))
    return w, b, losses

# 三种学习率对比
for lr, label in [(0.001, "太小"), (0.01, "刚好"), (0.5, "太大")]:
    w, b, losses = train_with_lr(lr)
    print(f"lr={lr} ({label}): w→{w[0]:.3f} b→{b[0]:.3f} "
          f"epoch10 loss={losses[9]:.4f} epoch200 loss={losses[-1]:.4f}")

# 典型输出:
# lr=0.001 (太小): w→1.23  b→0.46  epoch10 loss=3.52  epoch200 loss=1.84 — 收敛极慢
# lr=0.01  (刚好): w→2.97  b→5.01  epoch10 loss=0.31  epoch200 loss=0.09 — 快速收敛
# lr=0.5   (太大): w→nan   b→nan   epoch10 loss=nan   — 直接发散
```

**面试话术**："学习率太小收敛慢，太大可能跳过最优点甚至发散。实践中一般从 0.01 或 0.001 开始试，或者用 Adam 这类自适应优化器。"

### 学习率衰减策略

```python
# 学习率衰减：前期大步快速收敛，后期小步精细搜索
def train_with_decay(X, y, initial_lr=0.1, decay=0.99, epochs=500):
    w, b = 0.0, 0.0
    lr = initial_lr
    for epoch in range(epochs):
        y_pred = w * X + b
        dw = (-2/len(X)) * np.sum(X * (y - y_pred))
        db = (-2/len(X)) * np.sum(y - y_pred)
        w -= lr * dw
        b -= lr * db
        lr *= decay  # 每轮乘以衰减因子
    return w, b

w, b = train_with_decay(X, y)
print(f"带动量衰减: w={w[0]:.3f}, b={b[0]:.3f}")
```

## 四、特征缩放的必要性

**为什么必须做特征缩放？** 不同特征量纲不同（如"年龄"20-60，"收入"5000-500000），不做缩放会导致：
1. 梯度下降的等高线是极扁的椭圆，zig-zag 震荡收敛极慢
2. 正则化对不同特征惩罚不均（大数值特征被过度惩罚）

### 标准化 vs 归一化

| 方法 | 公式 | 结果 | 适用场景 |
|------|------|------|----------|
| **标准化**（Standardization） | (x - μ) / σ | 均值 0，标准差 1 | 假设数据服从正态分布；PCA、线性回归、逻辑回归、SVM |
| **归一化**（Min-Max） | (x - min) / (max - min) | 缩到 [0, 1] | 数据有固定边界；神经网络（sigmoid/tanh 激活函数）、图像像素 |
| **Robust Scaling** | (x - 中位数) / IQR | 对异常值鲁棒 | 数据有明显异常值 |

```python
from sklearn.preprocessing import StandardScaler, MinMaxScaler
import numpy as np

# 构造不同量纲的数据
np.random.seed(42)
age = np.random.randint(20, 60, 100).reshape(-1, 1)        # 20~60
salary = np.random.randint(5000, 50000, 100).reshape(-1, 1) # 5000~50000
X = np.hstack([age, salary])

# 不做缩放的后果：梯度更新时 salary 方向的梯度远大于 age 方向
print(f"age 均值={age.mean():.0f} 标准差={age.std():.0f}")
print(f"salary 均值={salary.mean():.0f} 标准差={salary.std():.0f}")

# 标准化
scaler_std = StandardScaler()
X_std = scaler_std.fit_transform(X)
print(f"标准化后 age: mean={X_std[:,0].mean():.3f}, std={X_std[:,0].std():.3f}")
print(f"标准化后 salary: mean={X_std[:,1].mean():.3f}, std={X_std[:,1].std():.3f}")
# 输出: 两列均值为 0，标准差为 1

# 归一化
scaler_mm = MinMaxScaler()
X_mm = scaler_mm.fit_transform(X)
print(f"归一化后 age 范围: [{X_mm[:,0].min():.3f}, {X_mm[:,0].max():.3f}]")
print(f"归一化后 salary 范围: [{X_mm[:,1].min():.3f}, {X_mm[:,1].max():.3f}]")
# 输出: 两列范围均为 [0, 1]
```

**面试话术**："特征缩放是梯度下降类模型的前置操作。树模型（决策树、随机森林、XGBoost）不需要缩放——它们按值的大小排序分裂，跟量纲无关。"

## 五、正则化：对抗过拟合

在损失函数后面加一个"惩罚项"，约束参数不要太大。参数小 → 模型简单 → 泛化好。

### 三种正则化对比

| 正则化 | 惩罚项 | 特点 | 适用场景 |
|--------|--------|------|----------|
| **L1（Lasso）** | λ × Σ\|w\| | 产生稀疏解，自动做特征选择 | 特征很多但只有少数有用（高维稀疏数据） |
| **L2（Ridge）** | λ × Σ w² | 参数趋向于 0 但不会正好为 0，所有特征都有贡献 | 特征间存在共线性、大多数特征都有用的场景 |
| **ElasticNet** | λ₁ × Σ\|w\| + λ₂ × Σ w² | L1 和 L2 的折中 | 特征多且有关联时（如基因数据） |

数学形式：
```
Lasso:  J(w) = MSE + λ × Σ|wⱼ|
Ridge:  J(w) = MSE + λ × Σ wⱼ²
ElasticNet: J(w) = MSE + λ₁ × Σ|wⱼ| + λ₂ × Σ wⱼ²
```

```python
from sklearn.linear_model import Ridge, Lasso, ElasticNet
from sklearn.model_selection import train_test_split
import numpy as np

# 生成 50 个特征但只有 5 个真正有用的数据
np.random.seed(42)
n_samples = 200
X = np.random.randn(n_samples, 50)
true_w = np.zeros(50)
true_w[:5] = [3.0, -2.0, 1.5, 0.0, 4.0]  # 只有前 5 个特征有效
y = X @ true_w + np.random.randn(n_samples) * 0.5

X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

# Ridge：所有系数都缩小但不为 0
ridge = Ridge(alpha=1.0)
ridge.fit(X_train, y_train)
print(f"Ridge - 非零系数个数: {(np.abs(ridge.coef_) > 1e-5).sum()}")
print(f"Ridge - 前 10 个系数: {ridge.coef_[:10].round(3)}")
# 输出: 50 个系数几乎全非零，但都很小

# Lasso：大量系数被压缩到 0
lasso = Lasso(alpha=0.1)
lasso.fit(X_train, y_train)
print(f"Lasso - 非零系数个数: {(np.abs(lasso.coef_) > 1e-5).sum()}")
print(f"Lasso - 系数: {lasso.coef_[:10].round(3)}")
# 输出: 只有少数（接近 5 个）系数非零，自动做了特征选择

# ElasticNet：介于两者之间
elastic = ElasticNet(alpha=0.1, l1_ratio=0.5)
elastic.fit(X_train, y_train)
print(f"ElasticNet - 非零系数个数: {(np.abs(elastic.coef_) > 1e-5).sum()}")
```

**面试话术**："L1 产生稀疏解适合特征选择，L2 让所有参数都缩小但不断崖。实践中不确定就用 ElasticNet，让模型自己平衡。另外 L1 的不可导问题用次梯度或坐标下降解决。"

## 六、损失函数对比：MSE vs MAE

| 损失函数 | 公式 | 对异常值的敏感度 | 梯度行为 | 适用场景 |
|----------|------|------------------|----------|----------|
| **MSE** | (1/n) × Σ(y - y_pred)² | 极高（误差平方放大） | 误差大时梯度大，误差小时梯度小 | 数据干净，无极端异常值 |
| **MAE** | (1/n) × Σ\|y - y_pred\| | 较低（线性增长） | 梯度恒为 ±1，不随误差变化 | 数据有异常值时更鲁棒 |
| **Huber** | 小误差用 MSE，大误差用 MAE | 平衡两者 | 平滑过渡 | 不想手动筛异常值 |

```python
# 异常值对 MSE 和 MAE 的影响实验
import numpy as np

np.random.seed(42)
X = np.linspace(0, 10, 100)
y_true = 3 * X + 5

# 正常模型预测（接近真实值）
y_pred_normal = y_true + np.random.randn(100) * 2

# 计算损失
mse_normal = np.mean((y_true - y_pred_normal)**2)
mae_normal = np.mean(np.abs(y_true - y_pred_normal))
print(f"无异常值: MSE={mse_normal:.2f}, MAE={mae_normal:.2f}")

# 加入一个极端异常值
y_pred_outlier = y_pred_normal.copy()
y_pred_outlier[0] += 1000  # 第一个样本的预测偏离 1000

mse_outlier = np.mean((y_true - y_pred_outlier)**2)
mae_outlier = np.mean(np.abs(y_true - y_pred_outlier))
print(f"有异常值: MSE={mse_outlier:.2f}, MAE={mae_outlier:.2f}")
# 输出: MSE 从 ~4 暴涨到 ~10000，MAE 只是从 ~1.6 涨到 ~11.6
# MSE 被异常值"绑架"了，MAE 受影响小得多
```

**面试话术**："MSE 对大误差的平方惩罚会让模型拼命去迁就异常值，导致整体拟合变差；MAE 对所有误差一视同仁。实际项目中先用 MSE 训练，发现异常值再换 Huber Loss。"

## 七、评估指标

| 指标 | 公式 | 含义 | 量纲 |
|------|------|------|------|
| **MSE** | (1/n) Σ(y - y_pred)² | 均方误差，越大越差 | 原始单位²（不好解释） |
| **RMSE** | sqrt(MSE) | 均方根误差，和 y 同量纲 | 与 y 相同 |
| **MAE** | (1/n) Σ\|y - y_pred\| | 平均绝对误差，直观 | 与 y 相同 |
| **R²** | 1 - SS_res / SS_tot | 解释了多少方差（0~1），越接近 1 越好 | 无 |
| **调整 R²** | 1 - (1-R²)(n-1)/(n-d-1) | 加入特征数惩罚，防止堆特征刷 R² | 无 |

```python
from sklearn.metrics import mean_squared_error, mean_absolute_error, r2_score
import numpy as np

y_true = np.array([3.0, -0.5, 2.0, 7.0, 4.2])
y_pred = np.array([2.5, 0.0, 2.0, 8.0, 4.0])

mse = mean_squared_error(y_true, y_pred)
rmse = np.sqrt(mse)
mae = mean_absolute_error(y_true, y_pred)
r2 = r2_score(y_true, y_pred)
n, d = len(y_true), 1  # 样本数 5，特征数 1
adj_r2 = 1 - (1 - r2) * (n - 1) / (n - d - 1)

print(f"MSE: {mse:.3f}")      # MSE: 0.258
print(f"RMSE: {rmse:.3f}")    # RMSE: 0.508
print(f"MAE: {mae:.3f}")      # MAE: 0.440
print(f"R²: {r2:.3f}")        # R²: 0.949
print(f"调整 R²: {adj_r2:.3f}")  # 调整 R²: 0.931
```

**面试话术**："R² 告诉你模型比'用均值直接猜'好了多少。但 R² 有个坑——加再多无关特征 R² 也不会降，调整 R² 加了对特征数量的惩罚，特征选择时看它。"

## 八、从线性回归到逻辑回归

线性回归输出连续值，逻辑回归输出概率（0到1之间），用于分类。

**核心**：在线性组合外面套一层 sigmoid 函数，把输出压到 [0, 1]。

```python
def sigmoid(z):
    return 1 / (1 + np.exp(-z))

# 逻辑回归预测
# z = wx + b（线性部分）
# p = sigmoid(z)（概率输出）
# if p >= 0.5 → 正类，else → 负类

# 损失函数从 MSE 换成交叉熵（Cross-Entropy）
# 因为 MSE + sigmoid 的梯度会消失（饱和区梯度接近0）
```

**为什么逻辑回归用交叉熵而不用 MSE？**

sigmoid 在两端（接近 0 或 1）梯度接近 0，MSE 的导数又乘了这个接近 0 的数 → 梯度消失，参数不更新。交叉熵的导数把 sigmoid 的导数消掉了，不会消失。

```python
# 直观验证：MSE + Sigmoid 的梯度消失问题
def sigmoid(z):
    return 1 / (1 + np.exp(-z))

def mse_loss_with_sigmoid(y_true, z):
    """MSE + Sigmoid 的梯度"""
    p = sigmoid(z)
    # ∂MSE/∂z = 2(p - y) * p * (1 - p)
    # 当 p→0 或 p→1 时，p*(1-p)→0，梯度消失
    return 2 * (p - y_true) * p * (1 - p)

def cross_entropy_loss_with_sigmoid(y_true, z):
    """交叉熵 + Sigmoid 的梯度"""
    p = sigmoid(z)
    # ∂CE/∂z = p - y  （sigmoid 的导数被消掉了！）
    return p - y_true

# 预测很极端时（z 很大或很小）
print(f"z=10, p≈1.0: MSE梯度={mse_loss_with_sigmoid(1.0, 10):.8f}")      # 接近 0
print(f"z=10, p≈1.0: CE梯度={cross_entropy_loss_with_sigmoid(1.0, 10):.8f}")  # 正常
# MSE 的梯度 ≈ 0，参数不更新；CE 的梯度正常
```

## 九、面试完整回答模板

> "请解释线性回归和梯度下降"

"线性回归是找一个线性函数拟合数据，最小化均方误差。求解可以用正规方程直接求解析解，但数据量大时用梯度下降迭代求解——算梯度、沿负方向更新参数。逻辑回归在线性回归基础上加了 sigmoid 做分类，损失函数换成交叉熵避免梯度消失。"
