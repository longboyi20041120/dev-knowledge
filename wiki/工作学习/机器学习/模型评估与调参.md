# 模型评估与调参

面试必问：**过拟合 vs 欠拟合，交叉验证，准确率/精确率/召回率的区别。**

## 一、过拟合 vs 欠拟合

```python
# 用多项式回归演示
from sklearn.preprocessing import PolynomialFeatures
from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_squared_error
import numpy as np

# 真实关系: y = sin(x)，加一点噪声
np.random.seed(42)
X = np.linspace(0, 2*np.pi, 20).reshape(-1, 1)
y = np.sin(X).ravel() + np.random.randn(20) * 0.1

# 欠拟合：用线性模型拟合曲线 → 训练集和测试集都很差
model_under = LinearRegression()
model_under.fit(X, y)  # MSE ≈ 0.5（高偏差）

# 刚刚好：用 degree=3 多项式
X_poly3 = PolynomialFeatures(degree=3).fit_transform(X)
model_good = LinearRegression()
model_good.fit(X_poly3, y)  # MSE ≈ 0.01

# 过拟合：用 degree=19 多项式（20 个点 20 个参数 → 完美拟合每个点）
X_poly19 = PolynomialFeatures(degree=19).fit_transform(X)
model_over = LinearRegression()
model_over.fit(X_poly19, y)  # 训练 MSE ≈ 0，但测试集会很差
```

| | 偏差(Bias) | 方差(Variance) | 训练误差 | 测试误差 |
|------|------|------|------|------|
| 欠拟合 | 高 | 低 | 高 | 高 |
| 刚刚好 | 中 | 中 | 低 | 低 |
| 过拟合 | 低 | 高 | 极低 | 高 |

**面试话术**："欠拟合是模型太简单没学到规律，过拟合是把噪声也学了。判断方法：训练误差远小于测试误差就是过拟合；两个都高就是欠拟合。"

### 学习曲线诊断

学习曲线（Learning Curve）是诊断过拟合/欠拟合最直观的工具。

```python
import numpy as np
from sklearn.model_selection import learning_curve
from sklearn.ensemble import RandomForestClassifier
import matplotlib.pyplot as plt

# 生成模拟数据
from sklearn.datasets import make_classification
X, y = make_classification(n_samples=1000, n_features=20,
                           n_informative=5, random_state=42)

# 绘制学习曲线
train_sizes, train_scores, val_scores = learning_curve(
    RandomForestClassifier(n_estimators=100, random_state=42),
    X, y,
    train_sizes=np.linspace(0.1, 1.0, 10),  # 10% → 100% 训练数据
    cv=5, scoring='accuracy'
)

train_mean = train_scores.mean(axis=1)
val_mean = val_scores.mean(axis=1)

print("训练数据量 | 训练准确率 | 验证准确率 | 诊断")
print("-" * 55)
for size, tr, val in [(train_sizes[1], train_mean[1], val_mean[1]),
                       (train_sizes[4], train_mean[4], val_mean[4]),
                       (train_sizes[7], train_mean[7], val_mean[7]),
                       (train_sizes[9], train_mean[9], val_mean[9])]:
    gap = tr - val
    diagnosis = "过拟合，需要更多数据/正则化" if gap > 0.05 else "正常"
    print(f"  {int(size):4d}    |    {tr:.3f}    |    {val:.3f}    | {diagnosis}")

# 典型解读:
# 训练分低 + 验证分也低 → 欠拟合：加特征/换更复杂的模型
# 训练分高 + 验证分低（大裂隙）→ 过拟合：正则化/更多数据/更简单的模型
# 两条线都上升且接近 → 收集更多数据可能有帮助
# 两条线都已经趋于平缓 → 更多数据帮助不大
```

**学习曲线诊断速查表**：

| 现象 | 诊断 | 对策 |
|------|------|------|
| 训练和验证都低 | 欠拟合（高偏差） | 加特征、换更复杂模型、减少正则化 |
| 训练高、验证低，差距大 | 过拟合（高方差） | 加正则化、加数据、降模型复杂度 |
| 两条线还在上升且接近 | 需要更多数据 | 收集更多数据（最有价值） |
| 两条线平缓且接近 | 已达模型上限 | 换更好的模型、做特征工程 |

## 二、交叉验证（Cross-Validation）

**为什么不能只分一次训练/测试集？** 可能运气好分到简单数据，看起来准确率高，其实泛化差。

### 五折交叉验证示例

```python
from sklearn.model_selection import cross_val_score, KFold
from sklearn.ensemble import RandomForestClassifier

model = RandomForestClassifier()

# 5 折交叉验证
scores = cross_val_score(model, X, y, cv=5, scoring="accuracy")
print(f"各折: {scores}")
print(f"平均: {scores.mean():.3f} ± {scores.std():.3f}")
# 输出: 各折: [0.88 0.92 0.85 0.90 0.87]
#       平均: 0.884 ± 0.024
```

标准差小 = 模型稳定。如果某折特别低，说明那折数据有特殊情况。

### 交叉验证策略横向对比

| 策略 | 原理 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|----------|
| **K-Fold** | 数据分 K 份，轮流用一份验证，K-1 份训练 | 所有数据都被训练和验证过 | 不保持类别比例，不均衡数据可能某折全是一类 | 均衡数据 |
| **Stratified K-Fold** | 和 K-Fold 一样但保持每折类别比例 | 类别比例一致，评估更稳定 | 不适用于回归问题 | **分类问题的默认选择** |
| **Leave-One-Out (LOO)** | 每次留 1 个样本验证，其余训练 | 几乎无偏（用了几乎所有数据） | 极慢（n 个样本训练 n 次），方差大 | 小数据集（n<100） |
| **时间序列 CV** | 按时间顺序训练，只用"过去"预测"未来" | 符合真实业务场景 | 数据利用不充分（早期数据用得少） | 股票预测、销量预测等时序问题 |
| **Group K-Fold** | 同一个 group 的样本不会被分到不同折 | 防止数据泄露（同一个人/同一用户） | 需要知道 group 信息 | 用户行为预测、医疗数据（同一病人的记录） |

```python
from sklearn.model_selection import (
    KFold, StratifiedKFold, LeaveOneOut,
    TimeSeriesSplit, GroupKFold, cross_val_score
)

# Stratified K-Fold（分类问题首选）
skf = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
scores = cross_val_score(model, X, y, cv=skf, scoring='f1')

# 时间序列 CV（时序预测必须用）
tscv = TimeSeriesSplit(n_splits=5)
# 第 1 折：用前 20% 训练，预测接着的 15%
# 第 2 折：用前 35% 训练，预测接着的 15% ...类推

# Group K-Fold（防数据泄露）
# 假设 groups 标识每个样本属于哪个用户
groups = np.array([0,0,0,1,1,2,2,2,2,3,3,3, ...])  # 每个用户的 ID
gkf = GroupKFold(n_splits=5)
scores = cross_val_score(model, X, y, groups=groups, cv=gkf)
```

**面试话术**："分类问题默认 StratifiedKFold，保证每个折的类别比例和全量一致。做离线时序预测必须用 TimeSeriesSplit——如果用随机 K-Fold，模型会用'未来'数据训练，结果看起来好但上线就崩。有用户维度分组时一定要用 Group K-Fold，否则同一个用户的数据分散在训练集和测试集里就是数据泄露。"

## 三、混淆矩阵与分类指标

### 从混淆矩阵推导所有指标

```
             预测正    预测负
实际正      TP        FN
实际负      FP        TN

TP (True Positive):  真正——预测对了的正例
TN (True Negative):  真负——预测对了的负例
FP (False Positive): 假正——把负例错误预测为正（误报）
FN (False Negative): 假负——把正例错误预测为负（漏报）
```

```python
from sklearn.metrics import (
    accuracy_score, precision_score, recall_score,
    f1_score, confusion_matrix, fbeta_score,
    matthews_corrcoef
)

y_true = [0, 1, 0, 1, 0, 1, 0, 0, 1, 1]
y_pred = [0, 1, 1, 1, 0, 0, 0, 0, 1, 1]

cm = confusion_matrix(y_true, y_pred)
TP = cm[1, 1]  # 3
TN = cm[0, 0]  # 4
FP = cm[0, 1]  # 1
FN = cm[1, 0]  # 1
print(f"混淆矩阵:\n{cm}")
print(f"TP={TP}, TN={TN}, FP={FP}, FN={FN}")
```

### 全部指标公式与含义

| 指标 | 公式 | 含义 |
|------|------|------|
| **Accuracy**（准确率） | (TP+TN) / (TP+TN+FP+FN) | 总体预测对了多少 |
| **Precision**（精确率） | TP / (TP+FP) | 预测为正的样本里，多少是真的正 |
| **Recall**（召回率/TPR） | TP / (TP+FN) | 真正的正例，找回了多少 |
| **Specificity**（特异度/TNR） | TN / (TN+FP) | 真正的负例，找回了多少 |
| **FPR**（假阳性率） | FP / (TN+FP) = 1 - Specificity | 负例中被误判为正的比例 |
| **F1 Score** | 2×P×R / (P+R) | P 和 R 的调和平均，两者都重要时用 |
| **F_beta** | (1+β²)×P×R / (β²×P+R) | β>1 更重视 Recall，β<1 更重视 Precision |
| **MCC** | (TP×TN-FP×FN)/sqrt((TP+FP)(TP+FN)(TN+FP)(TN+FN)) | [-1, 1]，+1 完美，0 随机，-1 完全相反。最全面的二分类指标 |

```python
accuracy = accuracy_score(y_true, y_pred)
precision = precision_score(y_true, y_pred)
recall = recall_score(y_true, y_pred)
f1 = f1_score(y_true, y_pred)
f2 = fbeta_score(y_true, y_pred, beta=2)    # 更重视 Recall
f0_5 = fbeta_score(y_true, y_pred, beta=0.5)  # 更重视 Precision
mcc = matthews_corrcoef(y_true, y_pred)

print(f"Accuracy:   {accuracy:.3f}")
print(f"Precision:  {precision:.3f}")
print(f"Recall:     {recall:.3f}")
print(f"F1:         {f1:.3f}")
print(f"F2:         {f2:.3f}")     # Recall 权重是 Precision 的 4 倍
print(f"F0.5:       {f0_5:.3f}")   # Precision 权重是 Recall 的 4 倍
print(f"MCC:        {mcc:.3f}")
```

**面试必问：准确率 vs 精确率 vs 召回率**

| 指标 | 公式 | 问题 | 场景 |
|------|------|------|------|
| Accuracy | (TP+TN)/全部 | 正负样本不均时无用 | 均衡数据集 |
| Precision | TP/(TP+FP) | 预测为正的里面几个真的？ | 垃圾邮件检测（不想误删正常邮件） |
| Recall | TP/(TP+FN) | 真正的正例找到几个？ | 癌症检测（宁可误诊也不能漏诊） |
| F1 | 2×P×R/(P+R) | Precision 和 Recall 的调和平均 | 两者都重要时 |

**面试话术**："精确率和召回率是跷跷板。提高阈值 → 预测更保守 → Precision 升 Recall 降。降低阈值 → 预测更激进 → Recall 升 Precision 降。选哪个指标取决于业务——误报的代价大还是漏报的代价大。"

## 四、ROC 曲线与 AUC

### ROC 曲线是怎么画出来的

1. 模型对每个样本输出一个概率（或分数）
2. 把阈值从 1.0 逐步降到 0.0
3. 每个阈值下计算一组 (FPR, TPR)，画到坐标系里
4. 连接所有点，就是 ROC 曲线

```
阈值=1.0 → 全判负 → FPR=0, TPR=0 → 左下角
阈值=0.5 → 中等 → 某个 (FPR, TPR)
阈值=0.0 → 全判正 → FPR=1, TPR=1 → 右上角
```

```python
from sklearn.metrics import roc_curve, roc_auc_score
import numpy as np

np.random.seed(42)
# 模拟：正类得分高一些，负类得分低一些
y_true = np.array([0]*50 + [1]*50)
y_scores_good = np.concatenate([
    np.random.normal(0.3, 0.15, 50),    # 负类：得分集中在 0.3
    np.random.normal(0.7, 0.15, 50)     # 正类：得分集中在 0.7
])

# 计算 ROC 曲线的点
fpr, tpr, thresholds = roc_curve(y_true, y_scores_good)
auc = roc_auc_score(y_true, y_scores_good)

print(f"AUC: {auc:.3f}")
print("阈值变化过程（摘取）:")
for i in [0, len(thresholds)//4, len(thresholds)//2, 3*len(thresholds)//4, -1]:
    print(f"  阈值={thresholds[i]:.3f} → FPR={fpr[i]:.3f}, TPR={tpr[i]:.3f}")
```

### AUC 的三种典型情况

| AUC 值 | 含义 | 场景 |
|--------|------|------|
| AUC = 1.0 | 完美分类器，存在一个阈值完全分开两类 | 理想情况，现实中几乎不存在（可能是过拟合/数据泄露） |
| AUC = 0.5 | 随机猜测，对角线 | 模型没有学到任何信息 |
| AUC = 0.0 | 完全反向——总是把正判负、负判正 | 把标签反过来就变成 AUC=1，说明特征有效但理解反了 |
| AUC < 0.5 | 比随机猜还差，但比 0 好 | 反向预测即是正向，调转标签即可得到 >0.5 的模型 |

**AUC 的直观含义**：随机选一个正样本和一个负样本，正样本排在负样本前面的概率。AUC=0.8 意味着 80% 的情况下模型给正样本的打分高于负样本。

## 五、PR 曲线 vs ROC 曲线

### 样本不均衡时 PR 曲线比 ROC 更有用

**为什么？** ROC 曲线使用 FPR = FP / (TN+FP) 作为横轴。当负样本极多时（TN 巨大），即使 FP 的数量变化很大，FPR 也变化很小——ROC 曲线看起来"虚高"，掩盖了模型在正类上的差劲表现。

PR 曲线的横轴是 Recall（TPR），纵轴是 Precision。不涉及 TN，所以在正负样本悬殊时更能反映模型真实质量。

| 对比维度 | ROC 曲线 | PR 曲线 |
|----------|----------|---------|
| 横轴 / 纵轴 | FPR / TPR（Recall） | Recall / Precision |
| 是否受负样本影响 | 是（FPR 的分母是 TN+FP） | 否（两个轴都不涉及 TN） |
| 适用数据 | 正负样本均衡 | **正负样本严重不均衡时** |
| 基线 | 对角线（y=x），AUC=0.5 | y = 正样本比例（如 1% 正类则基线约 0.01） |
| 评估标准 | AUC 越接近 1 越好 | Average Precision（AP），越接近 1 越好 |

```python
from sklearn.metrics import average_precision_score, precision_recall_curve

# PR 曲线
precision_vals, recall_vals, _ = precision_recall_curve(y_true, y_scores_good)
ap = average_precision_score(y_true, y_scores_good)
print(f"Average Precision (AP): {ap:.3f}")
# AP 是 PR 曲线下的面积，等同于 PR 曲线的 "AUC"
```

**面试话术**："正负样本 1:1000 时 ROC 的 FPR 几乎不变，曲线看起来很好，但实际上模型可能一个正样本都找不到。PR 曲线不怕负样本多——你找到了多少正样本（Recall）和找得准不准（Precision），这两指标都和负样本数量无关。"

## 六、调参：Grid Search

```python
from sklearn.model_selection import GridSearchCV

param_grid = {
    "n_estimators": [50, 100, 200],
    "max_depth": [5, 10, 15, None],
    "min_samples_split": [2, 5, 10],
}

grid = GridSearchCV(
    RandomForestClassifier(random_state=42),
    param_grid,
    cv=5,           # 5折交叉验证
    scoring="f1",   # 按 F1 选最优
    n_jobs=-1       # 并行跑
)
grid.fit(X_train, y_train)

print(f"最佳参数: {grid.best_params_}")
print(f"最佳 CV 得分: {grid.best_score_:.3f}")
```

### 三种超参数搜索策略对比

| 策略 | 原理 | 效率 | 适用场景 |
|------|------|------|----------|
| **网格搜索**（Grid Search） | 穷举所有参数组合 | 低（组合爆炸） | 参数少（2-3 个）、范围小 |
| **随机搜索**（Random Search） | 在参数空间中随机采样 N 组 | 中（N 决定开销） | 参数多时比网格搜索更高效 |
| **贝叶斯优化** | 用高斯过程建模"参数→得分"，每次选最有希望的点 | 高（相同预算下效果更好） | 评估一次很贵（如深度学习） |

**为什么随机搜索往往比网格搜索更高效？**

网格搜索在每个参数维度均匀取点——但大多数参数对结果的影响不均匀（有的参数调了基本没变化）。随机搜索在重要参数维度上有更多不同的取值尝试，相当于更有效地探索了参数空间。

```python
# 随机搜索示例
from sklearn.model_selection import RandomizedSearchCV
from scipy.stats import randint, uniform

param_dist = {
    "n_estimators": randint(50, 500),       # [50, 500) 均匀分布
    "max_depth": randint(3, 30),
    "min_samples_split": randint(2, 20),
    "max_features": uniform(0.1, 0.9),       # [0.1, 1.0) 均匀分布
}

random_search = RandomizedSearchCV(
    RandomForestClassifier(random_state=42),
    param_distributions=param_dist,
    n_iter=50,       # 只试 50 组，而非网格搜索的几千组
    cv=5,
    scoring='f1',
    n_jobs=-1,
    random_state=42
)
random_search.fit(X_train, y_train)
print(f"随机搜索最佳: {random_search.best_score_:.3f}")
print(f"参数: {random_search.best_params_}")

# 对比：网格搜索 3×4×3 = 36 组，随机搜索 50 组覆盖更广
# 网格搜索每个参数只试 3-4 个值，随机搜索每个参数试了 50 个不同值
```

**面试话术**："调参通常三阶段：① 先用大范围随机搜索找到大致好的区域；② 在好区域附近做小范围网格搜索精调；③ 用学习曲线验证过拟合/欠拟合。不要一上来就网格搜索——组合数量指数爆炸。"

## 七、样本极度不均衡怎么办？

**面试专题**：这是分类问题中最常见的实际问题。

### 数据层面

| 方法 | 做法 | 优点 | 缺点 |
|------|------|------|------|
| **欠采样**（Under-sampling） | 随机扔掉多数类样本 | 简单，训练快 | 丢失信息，可能丢掉重要样本 |
| **过采样**（Over-sampling） | 简单复制少数类样本 | 不丢信息 | 容易过拟合（同一个样本学很多遍） |
| **SMOTE** | 在少数类样本之间插值生成新样本 | 比简单复制更多样化 | 可能生成噪声、不适合高维数据 |
| **SMOTE+欠采样** | SMOTE 生成少数类 + 随机扔掉多数类 | 平衡最佳 | 调参稍复杂 |

```python
from imblearn.over_sampling import SMOTE
from collections import Counter

# 模拟不均衡数据：正类 20 个，负类 980 个
from sklearn.datasets import make_classification
X_imb, y_imb = make_classification(
    n_samples=1000, n_features=20, weights=[0.98, 0.02], random_state=42
)
print(f"原始分布: {Counter(y_imb)}")
# 输出: Counter({0: 980, 1: 20})

# SMOTE 过采样
smote = SMOTE(random_state=42)
X_resampled, y_resampled = smote.fit_resample(X_imb, y_imb)
print(f"SMOTE 后: {Counter(y_resampled)}")
# 输出: Counter({0: 980, 1: 980})
```

### 算法层面

| 方法 | 做法 |
|------|------|
| **类别权重**（class_weight） | 少数类的错误惩罚更大，sklearn 里设 `class_weight='balanced'` |
| **阈值调整** | 不调模型，调决策阈值（默认 0.5 → 根据需要改） |
| **代价敏感学习** | 为不同错误类型赋予不同代价（FP 代价 1 元，FN 代价 100 元） |
| **集成方法** | EasyEnsemble（多次欠采样+集成）、BalanceCascade |

```python
# 方法一：类别权重（最简单有效）
from sklearn.ensemble import RandomForestClassifier
rf = RandomForestClassifier(class_weight='balanced', random_state=42)
rf.fit(X_train, y_train)

# 方法二：阈值调整
y_proba = rf.predict_proba(X_test)[:, 1]
# 默认阈值 0.5 不用，根据业务需求选阈值
threshold = 0.3  # 更激进地预测正类 → Recall 提高
y_pred_adjusted = (y_proba >= threshold).astype(int)

from sklearn.metrics import classification_report
print("默认阈值 0.5:")
print(classification_report(y_test, rf.predict(X_test)))
print(f"\n调整阈值 {threshold}:")
print(classification_report(y_test, y_pred_adjusted))
```

**面试话术（完整回答）**："样本不均衡分三个层面解决：数据处理最简单，SMOTE 生成合成样本；算法层面用 class_weight='balanced' 对少数类错误加大惩罚，这是改动最小的方法；评估层面不用 Accuracy，用 F1 或 PR-AUC。实际项目里 class_weight + F1 评估是最常见的组合。"

## 八、面试完整框架

> "你怎么评估一个模型？"

1. **先看数据划分**：训练/验证/测试集分层抽样
2. **选对指标**：分类用 F1 而非准确率（尤其样本不均衡时），回归用 MAE/MSE
3. **交叉验证**：至少 5 折，看均值和标准差
4. **对比基线**：比随机猜好多少？比简单规则好多少？
5. **过拟合检查**：训练集和验证集差距是否过大？

**完整面试回答模板**：

"评估模型我分五步走。第一步数据划分，用 StratifiedKFold 保持类别比例。第二步选指标，根据业务定——样本均衡看 Accuracy，不均衡看 F1 或 PR-AUC，回归看 RMSE。第三步交叉验证 5 折，看均值也看方差——方差大说明模型不稳定。第四步设基线对比，和随机猜、简单规则比，看投入产出。第五步诊断过拟合，看学习曲线判断下一步是加数据还是加正则化。"

## 面试高频考点小结

| 考点 | 频率 | 关键点 |
|------|------|--------|
| 过拟合 vs 欠拟合判断 | 极高 | 训练高测试低=过拟合，双低=欠拟合 |
| Precision vs Recall vs F1 | 极高 | 跷跷板关系，F1 调和平均 |
| 交叉验证为什么、怎么做 | 极高 | 防数据划分运气成分；分类用 StratifiedKFold |
| ROC 曲线绘制原理 | 高 | 阈值从 0→1，每个阈值算 (FPR, TPR) |
| AUC 的含义 | 高 | 正样本排在负样本前面的概率 |
| PR 曲线 vs ROC 曲线 | 高 | 样本不均衡用 PR，因为不涉及 TN |
| 样本不均衡处理 | 极高 | class_weight + SMOTE + 不均衡用 F1 |
| 网格搜索 vs 随机搜索 | 中高 | 随机搜索在参数多时更高效 |
| 学习曲线诊断 | 中 | 看两条线分叉=过拟合，都低=欠拟合 |
| StratifiedKFold 的必要性 | 中 | 保持每折叠类别比例 |
| MCC 指标 | 中低 | 最全面的二分类单一指标，不均衡数据也能用 |
| 时间序列交叉验证 | 中低 | 不能用随机 K-Fold，必须用 TimeSeriesSplit |
