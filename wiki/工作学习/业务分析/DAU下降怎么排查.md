# DAU 下降怎么排查

面试经典 case 题：**"某天 DAU 突然掉了 20%，你怎么排查？"**

面试官考察的是你的**分析框架**，不是答案本身。

## 排查框架（按顺序，不要跳）

### 第一步：确认数据是否真实

**先排除数据问题，再分析业务问题。**

- 埋点有没有改动？SDK 升级了？上报断了？
- 数据管道有没有延迟？ETL 任务跑失败了？
- 统计口径变了？比如昨天改了 DAU 的定义？
- 对比其他指标：MAU、新增、启动次数也暴跌 → 大概率数据问题

> 面试话术："首先我会确认数据是不是准确，排查埋点、ETL、统计口径有没有变化。"

### 第二步：定位下降的维度

数据没问题，就要**下钻看是谁掉了**。

| 维度 | 看什么 | 如果异常说明什么 |
|------|--------|-----------------|
| 地区 | 各省/市 DAU | 某个地区暴跌 → 可能是区域性事件（停电、自然灾害） |
| 渠道 | iOS vs Android | 单端暴跌 → 可能是版本问题、应用商店下架 |
| 版本 | 各版本活跃 | 新版本暴跌 → 新版本有 bug |
| 新老用户 | 新增 vs 老用户 | 新增暴跌 → 投放停了/渠道挂了；老用户暴跌 → 产品问题 |
| 时段 | 小时级 DAU | 某个时段暴跌 → 服务器宕机、推送没发 |

> 面试话术："我会按地区、渠道、版本、新老用户四个维度拆解，定位是哪个群体的 DAU 掉了。"

### 第三步：找外部原因

如果是整体下降，先看外面发生了什么：

- 竞品搞了大活动？（比如拼多多百亿补贴，淘宝 DAU 就掉）
- 节假日效应？（春节大家都走亲戚，工具 App 没人用）
- 政策法规？（游戏版号、金融监管）
- 负面舆情？（微博热搜、App Store 差评激增）

### 第四步：找内部原因

如果外部没问题，排查内部改动：

- 最近发了新版本？
- 改了推荐算法？
- 服务器有没有宕机？
- Push 推送是不是停了？
- 某个核心功能挂了？（比如抖音推荐流挂了）

### 第五步：交叉验证

用其他指标验证你的假设：

- 假设是"新版本 bug 导致老用户流失" → 看该版本崩溃率、卸载率是否升高
- 假设是"竞品活动抢了用户" → 看竞品 DAU 是否同步上涨
- 假设是"推送停了" → 看推送点击率数据

## 完整回答模板（面试用）

> "首先确认数据准确性，排除埋点和 ETL 问题。然后按四个维度下钻——地区、渠道、版本、新老用户，定位下降群体。定位后排查外部因素（竞品、节假日、舆情）和内部因素（版本发布、服务状态、运营活动）。最后用相关指标交叉验证假设，确认根因后给出解决方案。"

## 常见坑

1. **一上来就分析业务**，结果是埋点挂了 → 浪费几小时
2. **只看绝对值不看比率** → 日活掉了 20%，新增也掉了 20%，可能是渠道问题
3. **忽略时间因素** → 看是骤降（服务器挂了）还是缓慢下降（产品问题累积）
