# 指标体系搭建

数据分析师的核心能力之一：接到一个产品，知道该看什么指标。

## 第一性原理：北极星指标

任何产品先找到**唯一一个**衡量核心价值的指标。

| 产品类型 | 北极星指标 | 为什么 |
|----------|-----------|--------|
| 内容平台（抖音/B站） | 日均内容消费时长 | 用户花的时间 = 广告收入 |
| 电商（淘宝/拼多多） | GMV（成交总额） | 交易额是最终目的 |
| SaaS 工具 | 周活跃用户数 | 不用就流失了 |
| 社交（微信） | 日均发送消息数 | 沟通是核心价值 |
| 游戏 | 次日留存率 | 第一天不爽就不会再玩 |

北极星指标选错了，整个团队优化的方向就偏了。

## AARRR 海盗模型

从获客到推荐的完整漏斗：

```
获取(Acquisition) → 激活(Activation) → 留存(Retention) → 变现(Revenue) → 推荐(Referral)
```

### 每个阶段的指标

**获取（用户怎么来的）**
- CAC（用户获取成本）= 总投放费用 / 新增用户数
- 渠道转化率：各渠道广告点击 → 下载 → 注册
- 自然量 vs 付费量比例

**激活（用户第一次感受到价值）**
- 注册完成率 = 完成注册 / 下载量
- 关键行为完成率：如电商"首单"、内容"首次点赞"
- 激活率低 → 新手引导有问题，或者来的用户不是目标用户

**留存（用户是否会回来）**
- 次日/7日/30日留存率
- 留存曲线是否健康（好的产品留存曲线趋于水平，差的一直跌）
- [[留存分析实战]] — 详细分析

**变现（怎么赚钱）**
- ARPU（每用户平均收入）= 总收入 / 总用户数
- LTV（用户生命周期价值）= ARPU × 平均使用时长
- 付费率 = 付费用户 / 总用户

**推荐（用户是否愿意传播）**
- K因子 = 邀请数 × 转化率。K > 1 才能病毒增长
- NPS（净推荐值）

## 搭建指标体系的步骤

1. **定北极星** — 问"用户为什么用这个产品"
2. **画 AARRR 漏斗** — 每个阶段列 2-3 个关键指标
3. **拆维度** — 按渠道、地区、用户群、时间段下钻
4. **定阈值** — 什么值算正常？什么值算异常需要报警？
5. **建看板** — 核心指标放第一屏，下钻可交互

## 面试常问

> "给你一个新上线的电商 App，你设计一套指标体系"

回答框架：北极星→GMV，然后按 AARRR 每个阶段讲 2 个指标，最后提拆维度（新老用户、渠道、品类）。
