# 离散数学

离散数学是计算机科学的数学基础。算法岗面试中，图论直接对应 LeetCode 的图算法题，逻辑与集合论影响你写 SQL 和条件判断的方式。

这篇笔记以**面试和实战**为导向——不堆定义，讲清楚"这东西在代码里怎么用"。

## 一、命题逻辑与谓词逻辑

### 命题逻辑

**命题**：能判断真假的陈述句。"今天是周一"是命题，"你好吗"不是命题。

#### 五个逻辑连接词

| 连接词 | 符号 | 含义 | Python 对应 |
|--------|------|------|------------|
| 否定 | ¬P | 非 P | `not P` |
| 合取 | P ∧ Q | P 且 Q | `P and Q` |
| 析取 | P ∨ Q | P 或 Q | `P or Q` |
| 蕴涵 | P → Q | 若 P 则 Q | `not P or Q` |
| 等价 | P ↔ Q | P 当且仅当 Q | `P == Q` |

蕴涵是最容易搞混的：**P → Q 只有当 P 真 Q 假时才为假**。P 为假时，P → Q 恒真（空真）。

```python
# 真值表：蕴涵的真值逻辑
def implies(P, Q):
    """P → Q 等价于 not P or Q"""
    return (not P) or Q

# 打印真值表
print(" P | Q | P→Q")
print("-----------")
for P in [True, False]:
    for Q in [True, False]:
        print(f" {int(P)} | {int(Q)} |  {int(implies(P, Q))}")
#  P | Q | P→Q
# -----------
#  1 | 1 |  1
#  1 | 0 |  0   ← 只有这种情况为假：P 真但 Q 假
#  0 | 1 |  1   ← P 为假时蕴涵恒真（空真）
#  0 | 0 |  1
```

#### 逻辑等价

两个公式在所有可能赋值下都有相同的真值，称为逻辑等价。

常用等价式：

```
¬(P ∧ Q) ≡ ¬P ∨ ¬Q     （德摩根律）
¬(P ∨ Q) ≡ ¬P ∧ ¬Q     （德摩根律）
P → Q ≡ ¬P ∨ Q         （蕴涵等价）
¬(P → Q) ≡ P ∧ ¬Q      （蕴涵的否定，面试常考）
```

**代码中的应用**：简化复杂的 if-else 条件。

```python
# 德摩根律在代码中的应用
# 这两个条件等价：
if not (x > 0 and y > 0):
    print("不在第一象限")

# 等价于：
if not (x > 0) or not (y > 0):
    print("不在第一象限")

# 等价于：
if x <= 0 or y <= 0:
    print("不在第一象限")
```

### 谓词逻辑

谓词逻辑在命题逻辑的基础上加了**量词**：

- **∀（全称量词）**："对所有"，`all()` in Python
- **∃（存在量词）**："存在某个"，`any()` in Python

```python
# 量词与 Python 的对应
nums = [2, 4, 6, 8, 10]

# ∀x, x 是偶数 → Python: all()
all_even = all(x % 2 == 0 for x in nums)  # True

# ∃x, x > 5 → Python: any()
some_large = any(x > 5 for x in nums)     # True

# 面试题：¬∀x P(x) ≡ ∃x ¬P(x)
# "不是所有人都及格" ≡ "存在有人不及格"
```

## 二、集合论基础

### 基本概念

| 运算 | 符号 | 含义 | Python |
|------|------|------|--------|
| 并集 | A ∪ B | 属于 A 或属于 B | `A \| B` 或 `A.union(B)` |
| 交集 | A ∩ B | 同时属于 A 和 B | `A & B` 或 `A.intersection(B)` |
| 差集 | A - B | 属于 A 但不属于 B | `A - B` 或 `A.difference(B)` |
| 补集 | Aᶜ 或 Ā | 全集中去掉 A | `U - A` |
| 对称差 | A ⊕ B | 恰好属于其中一个 | `A ^ B` 或 `A.symmetric_difference(B)` |
| 幂集 | P(A) | A 的所有子集 | `itertools.combinations` |
| 笛卡尔积 | A × B | 所有有序对 (a,b) | `itertools.product` |

```python
# Python 集合操作
A = {1, 2, 3, 4}
B = {3, 4, 5, 6}

print(f"A ∪ B = {A | B}")                    # {1, 2, 3, 4, 5, 6}
print(f"A ∩ B = {A & B}")                    # {3, 4}
print(f"A - B = {A - B}")                    # {1, 2}
print(f"A ⊕ B = {A ^ B}")                    # {1, 2, 5, 6}

# 幂集：生成所有子集
from itertools import chain, combinations

def powerset(iterable):
    """返回集合的所有子集"""
    s = list(iterable)
    return chain.from_iterable(combinations(s, r) for r in range(len(s) + 1))

print("幂集 P({1,2,3}):")
for subset in powerset({1, 2, 3}):
    print(f"  {set(subset)}")
# 共 2³ = 8 个子集

# 笛卡尔积
from itertools import product
cards = list(product(['A', 'K', 'Q'], ['♠', '♥', '♦', '♣']))
print(f"扑克牌笛卡尔积: {len(cards)} 张")  # 3 × 4 = 12
```

### 容斥原理

```
|A ∪ B| = |A| + |B| - |A ∩ B|
|A ∪ B ∪ C| = |A| + |B| + |C| - |A∩B| - |A∩C| - |B∩C| + |A∩B∩C|
```

**LeetCode 中的应用**：计算两个数组的交集大小、计算至少满足一个条件的元素数量。

### 集合的基数

- **有限集**：元素个数有限
- **可数无穷**：能与自然数一一对应（如整数、有理数）
- **不可数无穷**：不能与自然数一一对应（如实数 [0,1]）

**对角论证法**（Cantor）：实数不可数。反证法——假设可数列出所有实数，构造一个不在列表中的实数（对角线上每位取不同值）。

## 三、关系与函数

### 关系的三大性质

| 性质 | 定义 | 例子 | 反例 |
|------|------|------|------|
| 自反性 | ∀a, (a,a) ∈ R | "等于"关系 | "大于"关系 |
| 对称性 | (a,b) ∈ R → (b,a) ∈ R | "同学"关系 | "父子"关系 |
| 传递性 | (a,b)∈R ∧ (b,c)∈R → (a,c)∈R | "祖先"关系 | "朋友"关系 |

```python
# 用 Python 验证关系的性质
def is_reflexive(relation, elements):
    """检查是否自反"""
    return all((a, a) in relation for a in elements)

def is_symmetric(relation):
    """检查是否对称"""
    return all((b, a) in relation for (a, b) in relation)

def is_transitive(relation):
    """检查是否传递"""
    for (a, b) in relation:
        for (c, d) in relation:
            if b == c and (a, d) not in relation:
                return False
    return True

# 测试
elements = {1, 2, 3}
R_eq = {(1, 1), (2, 2), (3, 3), (1, 3), (3, 1)}  # 模拟等价关系

print(f"自反性: {is_reflexive(R_eq, elements)}")
print(f"对称性: {is_symmetric(R_eq)}")
print(f"传递性: {is_transitive(R_eq)}")
```

### 等价关系

同时具有**自反性、对称性、传递性**的关系。

最重要的性质：等价关系把集合**划分**成互不相交的等价类。

**算法应用**：并查集（Union-Find）本质上就是在维护等价关系。

```python
# 并查集 = 等价关系的动态维护
class UnionFind:
    """并查集：判断两个元素是否在同一个等价类中"""
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n

    def find(self, x):
        """查找 x 所在等价类的代表元"""
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])  # 路径压缩
        return self.parent[x]

    def union(self, x, y):
        """合并两个等价类"""
        px, py = self.find(x), self.find(y)
        if px == py:
            return
        if self.rank[px] < self.rank[py]:
            self.parent[px] = py
        elif self.rank[px] > self.rank[py]:
            self.parent[py] = px
        else:
            self.parent[py] = px
            self.rank[px] += 1

    def connected(self, x, y):
        """判断 x 和 y 是否等价（在同一集合中）"""
        return self.find(x) == self.find(y)

# 使用示例：判断图的连通分量
uf = UnionFind(5)
uf.union(0, 1)
uf.union(1, 2)
uf.union(3, 4)
print(f"0 和 2 连通: {uf.connected(0, 2)}")  # True
print(f"0 和 3 连通: {uf.connected(0, 3)}")  # False
```

### 偏序关系

**自反 + 反对称 + 传递**。不是所有元素都能比较——允许"不可比"。

**应用**：拓扑排序（课程依赖）、Hasse 图、任务的偏序关系。

```python
# 拓扑排序：偏序关系的线性化
from collections import defaultdict, deque

def topological_sort(vertices, edges):
    """Kahn 算法：对偏序关系做拓扑排序"""
    in_degree = {v: 0 for v in vertices}
    graph = defaultdict(list)

    for u, v in edges:
        graph[u].append(v)
        in_degree[v] += 1

    queue = deque([v for v in vertices if in_degree[v] == 0])
    result = []

    while queue:
        u = queue.popleft()
        result.append(u)
        for v in graph[u]:
            in_degree[v] -= 1
            if in_degree[v] == 0:
                queue.append(v)

    return result if len(result) == len(vertices) else None  # None 表示有环

# 课程依赖（偏序关系）：A 必须先于 B
courses = ['A', 'B', 'C', 'D']
prereqs = [('A', 'B'), ('A', 'C'), ('B', 'D'), ('C', 'D')]
order = topological_sort(courses, prereqs)
print(f"上课顺序: {order}")  # 可能的顺序: ['A', 'B', 'C', 'D']
```

### BFS 和 DFS

```python
from collections import deque

# BFS：层序遍历，最短路径（无权图）
def bfs(graph, start):
    visited = set([start])
    queue = deque([start])
    order = []

    while queue:
        node = queue.popleft()
        order.append(node)
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)

    return order

# DFS：深度优先，连通分量、拓扑排序、找环
def dfs(graph, node, visited=None, order=None):
    if visited is None:
        visited = set()
    if order is None:
        order = []

    visited.add(node)
    order.append(node)

    for neighbor in graph[node]:
        if neighbor not in visited:
            dfs(graph, neighbor, visited, order)

    return order

graph = {
    0: [1, 2],
    1: [0, 3, 4],
    2: [0],
    3: [1],
    4: [1]
}
print(f"BFS: {bfs(graph, 0)}")  # [0, 1, 2, 3, 4]
print(f"DFS: {dfs(graph, 0)}")  # [0, 1, 3, 4, 2]
```

### 二分图

**定义**：可以把顶点分成两个集合，所有边都跨越两个集合。

**判定**：图可以二染色（每条边两端颜色不同） ↔ 没有奇环。

```python
# 判断二分图（染色法）
def is_bipartite(graph, n):
    color = [-1] * n  # -1: 未染色, 0: 红, 1: 蓝

    def dfs(node, c):
        color[node] = c
        for neighbor in graph[node]:
            if color[neighbor] == -1:
                if not dfs(neighbor, 1 - c):
                    return False
            elif color[neighbor] == c:  # 相邻节点同色 → 不是二分图
                return False
        return True

    for i in range(n):
        if color[i] == -1:
            if not dfs(i, 0):
                return False
    return True

# 应用：人和岗位的匹配问题（推荐系统中的协同过滤也可以建模为二分图）
```

## 五、树

### 树的性质

- n 个顶点的树有 **n-1** 条边
- 树是**无环连通图**
- 任意两点间**恰好一条**简单路径
- 添加任意一条边会**形成恰好一个环**

### 二叉树

```python
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

# 二叉树遍历（递归 vs 迭代）
def inorder_recursive(root):
    """中序遍历（递归）：左 → 根 → 右"""
    if not root:
        return []
    return inorder_recursive(root.left) + [root.val] + inorder_recursive(root.right)

def inorder_iterative(root):
    """中序遍历（迭代）：用栈模拟递归"""
    result, stack = [], []
    curr = root
    while curr or stack:
        while curr:
            stack.append(curr)
            curr = curr.left
        curr = stack.pop()
        result.append(curr.val)
        curr = curr.right
    return result

# 二叉树性质
# 满二叉树：深度 k，节点数 2^k - 1
# 完全二叉树：只有最下层右边可能缺失（堆的底层结构）
# 二叉搜索树 BST：左 < 根 < 右
# 平衡二叉树：左右子树高度差 ≤ 1
```

### 最小生成树

**定义**：连通加权无向图中，边权和最小的生成树。

#### Kruskal 算法

```python
# Kruskal：按边权排序，贪心选边，用并查集防环
def kruskal(n, edges):
    """
    n: 顶点数
    edges: [(u, v, weight), ...]
    返回: 最小生成树的总权重
    """
    uf = UnionFind(n)
    edges.sort(key=lambda x: x[2])  # 按权重排序
    total_weight = 0
    mst_edges = []

    for u, v, w in edges:
        if not uf.connected(u, v):
            uf.union(u, v)
            total_weight += w
            mst_edges.append((u, v, w))
        if len(mst_edges) == n - 1:
            break

    return total_weight, mst_edges

# 示例
edges_mst = [
    (0, 1, 4), (0, 2, 3), (1, 2, 1),
    (1, 3, 2), (2, 3, 4)
]
total, mst = kruskal(4, edges_mst)
print(f"最小生成树权重: {total}")    # 6
print(f"MST 边: {mst}")              # [(1, 2, 1), (1, 3, 2), (0, 2, 3)]
```

#### Prim 算法

```python
import heapq

def prim(n, graph):
    """
    graph: 邻接表, graph[u] = [(v, weight), ...]
    从顶点 0 开始
    """
    visited = [False] * n
    min_heap = [(0, 0, -1)]  # (weight, node, parent)
    total_weight = 0
    mst_edges = []

    while min_heap:
        w, u, parent = heapq.heappop(min_heap)
        if visited[u]:
            continue
        visited[u] = True
        total_weight += w
        if parent != -1:
            mst_edges.append((parent, u, w))

        for v, weight in graph[u]:
            if not visited[v]:
                heapq.heappush(min_heap, (weight, v, u))

    return total_weight, mst_edges
```

| 算法 | 数据结构 | 时间复杂度 | 适用场景 |
|------|---------|-----------|---------|
| Kruskal | 并查集 | O(E log E) | 稀疏图 |
| Prim | 优先队列 | O(E log V) | 稠密图 |

### 鸽巢原理

**n+1 个鸽子放进 n 个笼子，至少一个笼子有 ≥2 只鸽子。**

**面试经典题**：证明在 n 个人中，至少有两人有相同数量的朋友。

```python
# 鸽巢原理的 LeetCode 应用：抽屉原理
# 287. 寻找重复数：n+1 个数（值在 [1, n]），找重复的那个
def find_duplicate(nums):
    """
    鸽巢原理：n+1 个数放在 [1,n] 这 n 个位置上，必有重复。
    用 Floyd 判圈算法（快慢指针）O(n) 时间 O(1) 空间。
    """
    slow = fast = nums[0]
    while True:
        slow = nums[slow]
        fast = nums[nums[fast]]
        if slow == fast:
            break
    # 找到环入口
    slow = nums[0]
    while slow != fast:
        slow = nums[slow]
        fast = nums[fast]
    return slow
```

### 容斥原理

```python
# 容斥原理：至少属于一个集合的元素数量
# |A ∪ B ∪ C| = |A|+|B|+|C| - |A∩B|-|A∩C|-|B∩C| + |A∩B∩C|

def inclusion_exclusion(sets):
    """
    容斥原理的通用模式：
    先加单个，再减两两交集，再加三三交集...
    """
    from itertools import combinations
    n = len(sets)
    result = 0
    universal = set.union(*sets) if sets else set()

    for k in range(1, n + 1):
        sign = 1 if k % 2 == 1 else -1
        for combo in combinations(range(n), k):
            intersection = sets[combo[0]]
            for i in combo[1:]:
                intersection = intersection & sets[i]
            result += sign * len(intersection)

    return result

A = {1, 2, 3, 4}
B = {3, 4, 5, 6}
C = {1, 4, 6, 7}
print(f"|A ∪ B ∪ C| = {inclusion_exclusion([A, B, C])}")
print(f"直接计算验证 = {len(A | B | C)}")
```

| 群 (Group) | 1 个运算 | 封闭、结合、单位元、逆元 | 密码学（椭圆曲线群）、对称性 |
| 环 (Ring) | 2 个运算 (+, ×) | 加法交换群 + 乘法半群 | 整数运算、多项式环 |
| 域 (Field) | 2 个运算 (+, ×) | 加减乘除全封闭 | 有限域 GF(2ⁿ) → AES 加密、Reed-Solomon 纠错码 |

```python
# 模运算构成有限群
# Z_n = {0, 1, ..., n-1} 在模 n 加法下构成群

def mod_group_example(n=7):
    """模 n 加法群"""
    elements = list(range(n))
    # 单位元：0
    # a 的逆元：n - a (mod n)

    print(f"Z_{n} 加法群:")
    for a in elements:
        inverse = (n - a) % n
        assert (a + inverse) % n == 0  # a + (-a) = 0
        print(f"  {a} 的逆元: {inverse}")

mod_group_example(7)

# 1. 等变网络（Equivariant Networks）：利用对称群保证输出随输入对称变换
# 2. 几何深度学习：SO(3) 旋转群上的卷积
# 3. 晶格密码学：基于格的困难问题
```

## 八、面试高频考点

### Q1: BFS 和 DFS 什么时候用哪个？

| 场景 | 选择 | 原因 |
|------|------|------|
| 无权图最短路径 | BFS | 按层扩展，首次到达即最短 |
| 找所有路径/回溯 | DFS | 递归自然地走到底再回溯 |
| 拓扑排序 | BFS(Kahn) / DFS | 两种都行，Kahn 更直观 |
| 连通分量 | 都行 | BFS/DFS 都能遍历整个分量 |
| 二分图判定 | BFS / DFS | 染色过程中发现冲突 |
| 记忆化搜索 | DFS + 记忆 | 树形结构天然适合 DFS |

### Q2: 最小生成树有什么用？

A: (1) 网络布线最小成本 (2) 图像分割（Graph-cut 中的最小割）(3) 聚类（单链聚类的层次树就是 MST）(4) 近似 TSP（MST 的代价 ≤ TSP 最优解）。

### Q3: 并查集能解决什么问题？

A: 动态连通性问题。LeetCode 经典题：岛屿数量、省份数量、冗余连接、等式方程的可满足性。核心操作：find（路径压缩）+ union（按秩合并），均摊近乎 O(1)。

### Q4: 逻辑命题在面试中有用吗？

A: 直接考的不多，但：
- 复杂 SQL 中 WHERE 条件本质是谓词逻辑
- 代码中的早返回（early return）、guard clause 本质是条件化简
- 面试中的"智力题"（谁在说谎、骑士与无赖）本质是命题逻辑

### Q5: 树和图的关系？

A: 树是**无环连通图**。算法上：树天然适合递归（父子关系明确），图需要额外处理环（visited 标记）。面试中，能用树形 DP 解决的问题，在一般图上往往很难（因为环的存在使得 DP 有后效性）。
