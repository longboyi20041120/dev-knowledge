# 概率分布基础

面试会考的分布不多——正态、二项、泊松、指数。把核心特征和使用场景讲清楚就行。

---

## 零、离散 vs 连续：统一视角

先从根上把分布分两类。这个分类决定了你用什么工具、怎么画图、怎么做检验。

| 维度 | 离散分布 | 连续分布 |
|------|---------|---------|
| 取值 | 可数的、孤立的点（0, 1, 2, ...） | 区间内任意实数（0.5, 1.23, 3.14159...） |
| 概率表示 | PMF（概率质量函数）P(X = k) | PDF（概率密度函数）f(x)，面积 = 概率 |
| 求和 vs 积分 | P(a ≤ X ≤ b) = Σ PMF | P(a ≤ X ≤ b) = ∫ PDF dx |
| 典型分布 | 二项、泊松、几何 | 正态、指数、均匀、伽马 |
| 可视化 | 柱状图（每个柱子高度 = 概率） | 密度曲线（曲线下面积 = 1） |
| 常见场景 | 计数数据（次数、人数、个数） | 测量数据（时间、长度、温度） |

**什么时候选哪种？**

```
你的数据是"数数"得来的（整数、非负）？
  ├── 是 → 离散分布
  │     ├── 有上限（如 N 次试验中的成功次数）→ 二项分布
  │     ├── 无上限（如每小时来电次数）→ 泊松分布
  │     └── 直到第一次成功所需的试验次数 → 几何分布
  └── 不是（可以是小数、负数）→ 连续分布
        ├── 对称、钟形 → 正态分布
        ├── 描述等待时间/间隔 → 指数分布
        ├── 在一定范围内均匀取值 → 均匀分布
        └── 描述等待多个事件的时间 → 伽马分布
```

这条决策树在面试时直接画出来，同时嘴里念叨推理过程，比你硬背"泊松描述计数"有说服力得多。

## 一、正态分布

**最关键的分布。** 中心极限定理保证：不管原始数据长什么样，**样本均值的分布趋近正态**。

```python
import numpy as np
import matplotlib.pyplot as plt

# 演示中心极限定理
# 从指数分布（严重右偏）抽 1000 次，每次抽 30 个样本算均值
np.random.seed(42)
sample_means = []
for _ in range(1000):
    sample = np.random.exponential(scale=2.0, size=30)
    sample_means.append(sample.mean())

# sample_means 的分布趋近正态！尽管原始数据是指数分布
```

三个关键数字（68-95-99.7 规则）：
```
在均值 ± 1σ 内：约 68% 的数据
在均值 ± 2σ 内：约 95% 的数据
在均值 ± 3σ 内：约 99.7% 的数据
```

**业务应用**：异常检测——连续 N 天 DAU 超出均值 ± 3σ？基本不可能正常波动，大概率出事了。

### 更多业务案例

| 场景 | 为什么是正态 | 应用 |
|------|------------|------|
| 异常检测 | 正常行为波动应该落在 μ±3σ 内 | DAU 暴跌、支付转化率异常告警 |
| 质量管控 | 工业产品尺寸围绕目标值对称波动 | 六西格玛管理、Cpk 计算 |
| 风控评分 | 大量用户信用分综合的结果趋近正态 | 信用评分卡建模 |
| A/B 测试 | 样本均值的抽样分布是正态（CLT） | t 检验的前提条件 |

## 二、二项分布

**N 次独立试验中成功 K 次的概率。**

业务场景：1000 个用户看到广告，点击率 5%，刚好有 60 个点击的概率？

```python
from scipy.stats import binom

# 1000 个用户，点击率 5%（p=0.05）
n, p = 1000, 0.05

# 刚好 60 个点击
p_exact = binom.pmf(60, n, p)  # 概率 ≈ 2.1%

# 超过 60 个点击（AB 测试里可能是显著的信号）
p_more = 1 - binom.cdf(60, n, p)  # 概率 ≈ 6.7%

# 预期点击人数
expected = n * p  # 50 人
```

### 更多业务案例

**AB 测试样本量计算**

二项分布是 AB 测试样本量计算的基础。因为"用户是否点击"是伯努利试验，N 个用户的总点击次数服从二项分布。用二项分布的性质可以反推需要多少样本才能检测到预期的提升：

```python
from scipy.stats import norm

def sample_size_binomial(baseline, lift, alpha=0.05, power=0.8):
    """
    基于二项分布计算 AB 测试所需样本量
    baseline: 基线转化率（如 10%）
    lift: 预期相对提升（如 10% → 实际提升到 11%）
    alpha: 显著性水平
    power: 统计功效
    """
    p1 = baseline
    p2 = baseline * (1 + lift)

    # 标准差基于二项分布的方差 np(1-p)
    se = np.sqrt(p1 * (1 - p1) + p2 * (1 - p2))

    z_alpha = norm.ppf(1 - alpha / 2)   # 双尾检验
    z_beta = norm.ppf(power)

    effect = abs(p2 - p1)
    n = ((z_alpha + z_beta) ** 2 * (p1 * (1 - p1) + p2 * (1 - p2))) / (effect ** 2)
    return int(np.ceil(n))

print(f"基线 10%，预期提升 10%，每组需要: {sample_size_binomial(0.10, 0.10)} 人")
# 输出：每组约需 14747 人
print(f"基线 5%，预期提升 20%，每组需要: {sample_size_binomial(0.05, 0.20)} 人")
# 输出：每组约需 9601 人 — 基线越低、提升越小，所需样本量越大
```

**质检抽样**

1000 个零件，已知不良率 2%，抽检 50 个发现 3 个不良的概率：

```python
n, p = 50, 0.02
# 恰好发现 3 个不良品的概率
print(f"P(X=3) = {binom.pmf(3, n, p):.4f}")  # ≈ 0.0616

# 发现 3 个或更多不良品的概率（要不要退回整批？）
print(f"P(X>=3) = {1 - binom.cdf(2, n, p):.4f}")  # ≈ 0.0812
# 如果设定阈值为 5%，则不能拒绝"这批合格"的假设
```

## 三、泊松分布

**描述单位时间内事件发生次数的分布。** 本质上二项分布的极限版（n→∞, p→0, np=λ）。

业务场景：
```python
from scipy.stats import poisson

# 网站平均每小时 30 次访问（λ=30）
# 问：下个小时刚好 25 次访问的概率？
p = poisson.pmf(25, 30)
print(f"P(X=25): {p:.3f}")

# 问：下个小时超过 50 次访问的概率？（可能是被攻击了？）
p_high = 1 - poisson.cdf(50, 30)
print(f"P(X>50): {p_high:.6f}")  # 0.000026 — 极不可能，需要排查
```

| 场景 | λ 的估计 | 用途 |
|------|---------|------|
| 客服来电/小时 | 历史均值 | 排班规划 |
| 服务器请求/秒 | 历史均值 | 容量规划 |
| 网站访问/天 | 历史均值 | 异常检测 |

### 更多业务案例

**客服排班规划**

客服中心平均每小时 45 通来电。主管想知道：下一小时来 60 通以上的概率有多大？如果概率超过 20%，就该多安排一个客服。

```python
mu = 45
# 超过 60 通来电的概率
prob_overload = 1 - poisson.cdf(60, mu)
print(f"超过 60 通的概率: {prob_overload:.3f}")  # ≈ 0.012

# 更实用的问法：需要准备多少线路，才能 95% 的情况下不占线？
capacity = poisson.ppf(0.95, mu)  # ppf = 百分位点函数（逆 CDF）
print(f"95% 情况下不超过 {int(capacity)} 通来电，需准备这么多个客服")
```

**服务器故障预测**

服务器集群平均每天 0.5 次故障（λ=0.5）。问：某天发生 3 次故障的概率？

```python
p_crash = poisson.pmf(3, 0.5)
print(f"一天 3 次故障的概率: {p_crash:.6f}")  # ≈ 0.012636 — 虽然低，但一个月内大概率发生一次

# 一个月内至少有一天 3 次故障的概率？
p_any_day = 1 - poisson.cdf(2, 0.5)
print(f"30 天内至少发生一次'3 次故障日'的概率: {1 - (1 - p_any_day)**30:.3f}")
```

**网站流量异常检测**

用泊松分布建立每天访问量的置信区间，超出即告警：

```python
lambda_daily = 5000  # 日均访问量
lower = poisson.ppf(0.001, lambda_daily)  # 下界（千分之一分位）
upper = poisson.ppf(0.999, lambda_daily)  # 上界
print(f"正常范围: [{int(lower)}, {int(upper)}]")  # 如 [4670, 5337]
# 今天 6000 访问？超出了上界 → 告警（可能是被刷量/爬虫）
# 今天 4000 访问？低于下界 → 告警（可能是服务挂了用户进不来）
```

## 四、指数分布

**泊松的"镜像"：事件之间的间隔时间服从指数分布。**

如果每小时平均 30 次访问，两次访问之间的平均间隔 = 2 分钟。

```python
from scipy.stats import expon

# 平均间隔 = 1/λ = 1/30 小时 = 2 分钟
lambda_param = 30  # 每小时30次
scale = 1 / lambda_param  # 期望间隔

# 下一次访问在 1 分钟以内的概率
p = expon.cdf(1/60, scale=scale)  # ≈ 39%

# 超过 5 分钟没有访问（用户是不是都跑了？）
p_long = 1 - expon.cdf(5/60, scale=scale)  # ≈ 8%
```

### 重要性质：无记忆性

指数分布最特殊的性质：**"已经等了多久，不影响还要等多久"**。

```
P(T > t+s | T > s) = P(T > t)

比如：一台服务器已经连续运行 1000 小时了，
它在接下来 100 小时内出故障的概率，
和一台全新的服务器在接下来 100 小时内出故障的概率一样。
```

现实中这意味着：如果设备的故障服从指数分布，**预防性维护没有意义**（因为新旧设备的故障率一样）。但现实中设备都有老化，所以 Weibull 分布更常用。

### 更多业务案例

| 场景 | λ 的含义 | 问题 |
|------|---------|------|
| 用户会话时长 | 用户平均每 λ 秒离开一次 | 用户会在 30 秒内离开吗？ |
| 设备寿命（早期） | 早期失效率恒定 | 在保修期内免费换新的概率 |
| 网页加载时间 | 平均加载时间 = 1/λ | 超过 3 秒的概率 |
| 交易间隔 | 每秒平均 λ 笔交易 | 下一笔交易在 5 秒内到来的概率 |

## 五、均匀分布、几何分布、伽马分布

### 5.1 均匀分布（Uniform）

**在一定范围内每个值出现的概率相同。** 最简单的连续分布。

- 参数：a（下限）、b（上限）
- 期望：(a+b)/2，方差：(b-a)²/12
- **业务场景**：随机抽奖的中奖号码、随机分流用户的 user_id 哈希值、蒙特卡洛模拟中生成随机数
- **面试注意**：如果面试官问"你的 AB 测试分流是否均匀"，可以先画各组 user_id 的分布直方图——理论上应该接近均匀分布

### 5.2 几何分布（Geometric）

**直到第一次成功所需的试验次数。** 泊松是时间维度，几何是试验维度。

- 参数：p（单次成功的概率）
- 期望：1/p，方差：(1-p)/p²
- **业务场景**：一个新用户访问多少页后才首次点击广告；销售打多少个电话才能成交一单
- 与指数分布的关系：指数是"第一个事件发生前的时间"，几何是"第一个成功前的试验次数"

### 5.3 伽马分布（Gamma）

**独立指数分布的和。** 可以理解为"等到第 k 个事件发生的时间"。

- 参数：k（形状，事件个数）、θ（尺度，平均等待时间）
- 当 k=1 时退化为指数分布；k 很大时趋近正态
- **业务场景**：客服中心等到第 5 通来电的时间、设备在第 k 次冲击后失效的时间（疲劳寿命）
- 实际中伽马常被用作**先验分布**（贝叶斯统计中），因为它的共轭性质很好

## 六、Q-Q 图：判断数据近似什么分布

**Q-Q 图（Quantile-Quantile Plot）是把你的数据和理论分布比一比，看它们像不像。**

原理很简单：把你的数据的分位数（从小到大排序后各位置的值）画在 y 轴上，把理论分布对应分位数的值画在 x 轴上。如果两个分布一致，点应该落在对角线 y=x 上。

```python
import numpy as np
import matplotlib.pyplot as plt
from scipy import stats

# 模拟三种不同的数据
np.random.seed(42)
n = 500

data_normal = np.random.normal(0, 1, n)       # 正态数据
data_exp = np.random.exponential(1, n)         # 指数数据（右偏）
data_uniform = np.random.uniform(-3, 3, n)     # 均匀数据

fig, axes = plt.subplots(1, 3, figsize=(15, 4))

titles = ['正态数据 vs 正态分布', '指数数据 vs 正态分布', '均匀数据 vs 正态分布']
datasets = [data_normal, data_exp, data_uniform]

for ax, data, title in zip(axes, datasets, titles):
    stats.probplot(data, dist="norm", plot=ax)
    ax.set_title(title, fontsize=12)
    ax.get_lines()[0].set_markersize(3)
    ax.get_lines()[1].set_color('red')  # 参考线

plt.tight_layout()
# 解读：
# 左图：点几乎全在对角线上 → 数据是正态的
# 中图：两端偏离对角线（特别是右端翘起）→ 数据右偏，不是正态
# 右图：中间贴得很紧，但两端突然陡峭 → 均匀分布（尾部比正态薄）
```

**面试话术**："Q-Q 图是我最喜欢的分布检验工具，因为一眼就能看出偏离的方向和程度。直方图受分箱大小影响大，统计检验（如 Shapiro-Wilk）在大样本时过于敏感——有一点点偏离就显著。Q-Q 图在视觉和量化之间取得了平衡。"

### Q-Q 图解读口诀

| 看到的现象 | 说明什么 |
|-----------|---------|
| 点全在 y=x 线上 | 数据完美符合该分布 |
| 上端（右边）向上翘 | 右偏（比正态分布尾部更厚） |
| 下端（左边）向下弯 | 左偏 |
| 两端都偏离 | 可能是有离群值，或者分布本身不是正态 |
| S 形弯曲 | 数据比正态分布更"集中"，峰度更高 |

## 七、实战：用真实数据拟合分布

用 iris 数据集（最经典的统计入门数据）演示分布拟合和检验的完整流程。

```python
import numpy as np
import pandas as pd
from scipy import stats
import matplotlib.pyplot as plt
from sklearn.datasets import load_iris

# 加载真实数据
iris = load_iris()
df = pd.DataFrame(iris.data, columns=iris.feature_names)

# 取花瓣长度（petal length）作为分析对象
petal_length = df['petal length (cm)']

# 第一步：看直方图 + 密度曲线，形成假设
fig, axes = plt.subplots(1, 2, figsize=(12, 4))

axes[0].hist(petal_length, bins=20, density=True, alpha=0.6, edgecolor='black')
axes[0].set_title('花瓣长度直方图')
axes[0].set_xlabel('cm')

# 第二步：Q-Q 图
stats.probplot(petal_length, dist="norm", plot=axes[1])
axes[1].set_title('花瓣长度 Q-Q 图（正态）')
axes[1].get_lines()[1].set_color('red')

plt.tight_layout()
# 目测：直方图似乎有两个峰（三种花混在一起），Q-Q 图有明显弯曲
# 结论：整体不是正态分布

# === 第三步：按品种分开后再看 ===
df['species'] = iris.target_names[iris.target]

fig, axes = plt.subplots(2, 3, figsize=(15, 8))
species_list = df['species'].unique()

for i, species in enumerate(species_list):
    data = df[df['species'] == species]['petal length (cm)']

    # 直方图
    axes[0, i].hist(data, bins=10, density=True, alpha=0.6, edgecolor='black')
    axes[0, i].set_title(f'{species}\n偏度={stats.skew(data):.2f}, 峰度={stats.kurtosis(data):.2f}')

    # Q-Q 图
    stats.probplot(data, dist="norm", plot=axes[1, i])
    axes[1, i].set_title(f'{species} Q-Q')
    axes[1, i].get_lines()[1].set_color('red')

plt.tight_layout()
# 通常：setosa 的花瓣长度接近正态，versicolor 和 virginica 的分布更复杂

# === 第四步：定量检验（Shapiro-Wilk） ===
print("=" * 50)
print("Shapiro-Wilk 正态性检验")
print("H0: 数据服从正态分布")
print("=" * 50)

for species in species_list:
    data = df[df['species'] == species]['petal length (cm)']
    stat, p = stats.shapiro(data)
    conclusion = "不能拒绝正态" if p > 0.05 else "拒绝正态，数据不是正态分布"
    print(f"{species:<15} | W={stat:.4f}, p={p:.4f} → {conclusion}")

# === 第五步：尝试拟合其他分布 ===
# 用 K-S 检验比较数据和几种候选分布的拟合程度
print("\n" + "=" * 50)
print("K-S 检验：哪个分布拟合最好？")
print("（D 统计量越小，拟合越好）")
print("=" * 50)

target_data = df[df['species'] == 'setosa']['petal length (cm)']

distributions = {
    'norm': '正态分布',
    'expon': '指数分布',
    'gamma': '伽马分布',
    'uniform': '均匀分布',
}

for dist_name, dist_label in distributions.items():
    # 用最大似然估计（MLE）拟合参数
    dist_obj = getattr(stats, dist_name)
    params = dist_obj.fit(target_data)
    # K-S 检验
    D, p = stats.kstest(target_data, dist_name, args=params)
    print(f"{dist_label:<10} | D={D:.4f}, p={p:.4f} | {'拟合良好' if p > 0.05 else '拟合不佳'}")

# 输出示例（具体数值取决于数据）：
# 正态分布    | D=0.0965, p=0.8021 | 拟合良好 ← D 最小
# 指数分布    | D=0.2731, p=0.0031 | 拟合不佳
# 伽马分布    | D=0.1059, p=0.6873 | 拟合良好
# 均匀分布    | D=0.3381, p=0.0002 | 拟合不佳
```

**关键建议**：实际工作中，不要机械地跑一圈 test 看哪个 p 值最大就选哪个。应该先理解业务含义（比如等待时间理论上是指数分布），再用统计方法**验证**你的假设，而不是让统计方法代替你思考。

## 八、面试专题：怎么判断数据是不是正态分布？

这是面试中高频到爆的一道题。标准答案应该包含 **"先看、再画、最后测"** 三层：

### 方法对比

| 方法 | 原理 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|---------|
| **直方图** | 看数据的频数分布形状 | 直观，一秒看出偏态 | 受分箱数影响大，主观 | 快速 EDA，初步判断 |
| **Q-Q 图** | 比较样本分位数 vs 理论分位数 | 能看出偏离的具体位置和方向 | 需要经验解读 | 正式分析首选 |
| **Shapiro-Wilk** | 检验数据与正态分布的偏离是否显著 | 客观的 p 值 | 样本大时过于敏感（n>5000几乎总显著） | 中小样本（n < 5000） |
| **偏度 + 峰度** | 偏度=0 对称，峰度=3 正态 | 给出量化的偏离方向和程度 | 只是描述性指标 | 补充参考 |

### Python 代码：四种方法一条龙

```python
import numpy as np
from scipy import stats

def normality_check(data, name="数据"):
    """跑四种正态性检查并给出综合结论"""
    print(f"\n{'='*50}")
    print(f"正态性检查: {name}")
    print(f"{'='*50}")
    print(f"样本量: {len(data)}")

    # 方法1: 偏度 & 峰度
    skew = stats.skew(data)
    kurt = stats.kurtosis(data)  # Fisher 定义，正态分布 kurtosis=0
    print(f"\n[方法1] 偏度 = {skew:.3f}, 峰度(超额) = {kurt:.3f}")
    print(f"  偏度解释: {'左偏' if skew < -0.5 else '右偏' if skew > 0.5 else '基本对称'}")
    print(f"  峰度解释: {'厚尾（离群值多）' if kurt > 1 else '薄尾' if kurt < -1 else '正常'}")

    # 方法2: Shapiro-Wilk
    stat_sw, p_sw = stats.shapiro(data)
    print(f"\n[方法2] Shapiro-Wilk: W={stat_sw:.4f}, p={p_sw:.4f}")
    print(f"  结论: {'拒绝正态，数据不是正态分布' if p_sw < 0.05 else '不能拒绝正态分布'}")

    # 方法3: D'Agostino K² 检验（基于偏度+峰度的综合检验）
    stat_da, p_da = stats.normaltest(data)
    print(f"\n[方法3] D'Agostino K²: stat={stat_da:.4f}, p={p_da:.4f}")
    print(f"  结论: {'拒绝正态' if p_da < 0.05 else '不能拒绝正态'}")

    # 方法4: Anderson-Darling（更稳健的正态检验）
    result = stats.anderson(data, dist='norm')
    print(f"\n[方法4] Anderson-Darling: A²={result.statistic:.4f}")
    for sig_level, crit_val in zip(result.significance_level, result.critical_values):
        verdict = "拒绝正态" if result.statistic > crit_val else "不能拒绝正态"
        print(f"  显著性 {sig_level}%: 临界值={crit_val:.4f} → {verdict}")

    return {
        'skewness': skew,
        'kurtosis': kurt,
        'shapiro_p': p_sw,
        'normaltest_p': p_da,
    }

# 模拟几种场景
np.random.seed(42)
normality_check(np.random.normal(0, 1, 200), "标准正态数据 (n=200)")
normality_check(np.random.exponential(1, 200), "指数分布数据 (n=200)")
```

### 面试追问：如果数据不是正态分布，能用什么方法？

**面试官问这句话的时候，是想看你是否知道走"另一条路"。** 标准回答：

> "两条路。**第一条路：数据变换。** 对数据做 Box-Cox 变换、对数变换或平方根变换，把右偏的数据'拉正'，然后继续用正态方法。这在金融（对数收益率）和生物统计中很常见。
>
> **第二条路：非参数检验。** 如果变换后还是不正态，或者变换后结果不好解释，就用不依赖正态假设的方法：
> - 两组对比：用 Mann-Whitney U 检验代替 t 检验
> - 多组对比：用 Kruskal-Wallis 代替 ANOVA
> - 相关性：用 Spearman 秩相关代替 Pearson 相关
>
> 非参数方法的代价是统计功效略低（同样样本量下更难检测到差异），但在数据严重偏离正态时反而更可靠。"

```python
# Box-Cox 变换示例
from scipy.stats import boxcox

data_exp = np.random.exponential(2, 500)  # 严重右偏

# 变换前
print(f"变换前偏度: {stats.skew(data_exp):.3f}")
print(f"Shapiro-Wilk p: {stats.shapiro(data_exp)[1]:.6f}")

# Box-Cox 变换（自动选择最优 λ）
data_transformed, lambda_opt = boxcox(data_exp + 0.001)
print(f"\n最优 λ: {lambda_opt:.3f}")
print(f"变换后偏度: {stats.skew(data_transformed):.3f}")
print(f"Shapiro-Wilk p: {stats.shapiro(data_transformed)[1]:.4f}")
# 通常变换后 p 值大幅提高，偏度接近 0
```

## 九、面试速查表

### 分布速查

| 分布 | 类型 | 参数 | 均值 | 方差 | 典型场景 |
|------|------|------|------|------|---------|
| 正态 N(μ,σ²) | 连续 | μ, σ | μ | σ² | 身高、成绩、测量误差、CLT |
| 二项 B(n,p) | 离散 | n, p | np | np(1-p) | 点击率、转化率、AB测试样本量 |
| 泊松 Po(λ) | 离散 | λ | λ | λ | 访问量、来电数、故障次数、流量监控 |
| 指数 Exp(λ) | 连续 | λ | 1/λ | 1/λ² | 间隔时间、等待时间、用户会话 |
| 均匀 U(a,b) | 连续 | a, b | (a+b)/2 | (b-a)²/12 | 随机分流、随机数生成、蒙特卡洛 |
| 几何 Geo(p) | 离散 | p | 1/p | (1-p)/p² | 首次成功前的试验次数、客户触达 |
| 伽马 Γ(k,θ) | 连续 | k, θ | kθ | kθ² | 第k次事件等待时间、贝叶斯先验 |

**面试常问："什么时候用泊松分布？"**

答："描述计数数据，特别是稀有事件在固定时间/空间内的发生次数。关键是 λ（发生率）已知且事件独立。如果方差远大于均值说明数据过度离散，泊松不合适。"

### 面试高频考点小结

| 考点 | 高频程度 | 核心要点 |
|------|---------|---------|
| 中心极限定理 | ★★★★★ | 不管原分布是什么，样本均值的分布趋近正态。这是所有参数检验的基石 |
| 正态分布 68-95-99.7 | ★★★★★ | 六西格玛、异常检测的基础 |
| 判断数据是否正态 | ★★★★☆ | 直方图→Q-Q图→Shapiro-Wilk→偏度峰度，四层递进 |
| 泊松 vs 二项 | ★★★★☆ | 泊松是二项的极限（n大p小）；二项有上限，泊松没有 |
| 指数分布无记忆性 | ★★★☆☆ | 等了多久不影响还要等多久——这是指数最特殊的性质 |
| 非正态怎么办 | ★★★★☆ | 数据变换（Box-Cox）或非参数方法 |
| 什么时候选什么分布 | ★★★☆☆ | 计数→离散（有没有上限？），测量→连续（等待时间？对称？） |
