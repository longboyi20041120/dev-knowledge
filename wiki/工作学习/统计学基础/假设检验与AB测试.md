# 假设检验与 AB 测试

数据分析师面试必问。面试官不一定让你算 p 值，但一定会问**"怎么设计一个 AB 测试"**。

## 一、假设检验的核心逻辑

**反证法**：先假设"没区别"（零假设 H₀），然后看数据是不是"太不可能"在这个假设下出现。如果是 → 拒绝 H₀，认为有显著差异。

```
H₀（零假设）：新算法和旧算法没区别
H₁（备择假设）：新算法有区别

p 值 = 在 H₀ 为真的情况下，观察到这个结果（或更极端）的概率

p < 0.05 → "在 H₀ 下这个结果太不可能了" → 拒绝 H₀ → 有显著差异
p ≥ 0.05 → "在 H₀ 下这个结果完全可能" → 不能拒绝 H₀ → 没有显著证据
```

**面试话术**："p 值不是'H₀ 为真的概率'，而是'在 H₀ 为真的前提下，看到这个数据的概率'。p < 0.05 只是说数据与 H₀ 不一致，不代表 H₁ 一定对。"

## 二、常见检验及选择

| 场景 | 用什么检验 | 例子 |
|------|---------|------|
| 两组的均值是否不同 | t 检验 | AB 测试：新版 vs 旧版转化率 |
| 多组的均值是否不同 | 方差分析（ANOVA） | 三种推荐算法的效果对比 |
| 两个分类变量是否独立 | 卡方检验 | 用户性别和购买行为是否相关 |
| 分布是否正态 | Shapiro-Wilk | 先检验数据是否符合正态分布 |

```python
from scipy import stats
import numpy as np

# t 检验：两组均值对比（AB测试最常用）
np.random.seed(42)
control = np.random.normal(100, 15, 1000)    # 对照组：均值100
treatment = np.random.normal(103, 15, 1000)  # 实验组：均值103（提升了3%）

t_stat, p_value = stats.ttest_ind(treatment, control)
print(f"t = {t_stat:.3f}, p = {p_value:.4f}")
# t = 4.212, p = 0.0000 → p < 0.05，显著！新算法确实更好

# 卡方检验：分类变量独立性
# 观察值：买了/没买 按性别分组
observed = np.array([[80, 120], [90, 110]])  # [男(买,没买), 女(买,没买)]
chi2, p_val, dof, expected = stats.chi2_contingency(observed)
print(f"卡方 = {chi2:.3f}, p = {p_val:.4f}")
# 如果 p > 0.05，性别和购买行为没有显著关系
```

## 三、AB 测试完整流程

面试官说："假设你要测试新的推荐算法能不能提升点击率，怎么做？"

### 标准回答框架

**1. 确定指标**
- 核心指标：点击率 CTR = 点击次数 / 展示次数
- 辅助指标：转化率、用户停留时长（确保 CTR 涨了但其他没掉）

**2. 确定样本量**
```python
# 样本量计算（简化版）
# 需要：预期提升幅度、当前基线、显著性水平(α=0.05)、统计功效(1-β=0.8)
from math import sqrt

def sample_size(baseline_rate, expected_lift, alpha=0.05, power=0.80):
    # 基线 10% 的 CTR，预期提升到 11%（1%提升）
    p1, p2 = baseline_rate, baseline_rate * (1 + expected_lift)
    z_alpha = 1.96   # α=0.05 对应
    z_beta = 0.84    # power=0.8 对应
    n = ((z_alpha + z_beta)**2 * (p1*(1-p1) + p2*(1-p2))) / ((p2 - p1)**2)
    return int(n)

print(f"每组需要 {sample_size(0.10, 0.10)} 用户")
# 每组需要约 14,700 用户（所以 AB 测试不能只跑几百人）
```

**3. 随机分流**
- 用 user_id 哈希确保同一用户始终在同一组
- 检查分流是否均匀（各组用户量、历史行为分布类似）

**4. 运行测试**
- 至少跑一个完整周期（一周，覆盖工作日和周末）
- 防止"新奇效应"（用户对新东西天然会多点几次）
- 不能中途偷看结果提前停止（peeking 问题）

**5. 得出结论**
```python
# 对照组: 10000 次展示，1000 次点击（CTR 10%）
# 实验组: 10000 次展示，1100 次点击（CTR 11%）
from statsmodels.stats.proportion import proportions_ztest

success = [1000, 1100]  # 点击次数
nobs = [10000, 10000]   # 展示次数
z_stat, p_val = proportions_ztest(success, nobs)
print(f"z = {z_stat:.2f}, p = {p_val:.4f}")
# p < 0.05 → 显著提升，可以上线
```

### 面试话术模板

> "首先定核心指标（点击率）和辅助指标（转化率、停留时长防副作用）。算好样本量确保统计功效够。用 user_id 哈希做随机分流，跑至少一周覆盖周期效应。最后用 t 检验或比例检验判断显著，同时检查各组分布是否均匀。"

## 四、常见陷阱

| 陷阱 | 问题 | 解法 |
|------|------|------|
| 偷看（Peeking） | 天天看 p 值，等 p<0.05 就停 | 提前定好样本量和结束时间，到点再看 |
| 多重比较 | 同时测 20 个指标，总有一个 p<0.05 | Bonferroni 矫正：α / 比较次数 |
| 新奇效应 | 新东西刚出来数据好，过几天回归均值 | 跑够时间，去掉前几天数据 |
| 样本污染 | 同一用户同时看到新旧版本 | user_id 哈希固化分组 |
