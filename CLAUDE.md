# CLAUDE.md — 编程知识库

这是你的私人编程知识库。你就是维护者——读取、整理、写入、索引，全部由你自动完成。

---

## Vault 结构

```
编程知识库/
├── raw/              ← 用户扔原材料的地方，你只读不删
│   ├── 网页剪藏/     ← 网页文章、博客、文档
│   ├── 报错日志/     ← 复制粘贴的错误信息、堆栈
│   └── 临时想法/     ← 一句话灵感，待你展开
├── wiki/             ← 你维护的结构化知识（核心产出）
│   ├── 项目/         ← 每个项目的完整知识
│   ├── 技术栈/       ← 跨项目复用的技术知识
│   ├── 踩坑记录/     ← Bug 修复经验
│   ├── 灵感/         ← 展开后的想法和创意
│   ├── 索引.md       ← 总目录，每次写入都要更新
│   └── 最近更新.md   ← 变更日志
└── 模板/             ← 笔记格式模板，生成时参考
```

## 你的核心任务

### 1. 被动触发：用户问问题时

当用户问技术问题、查 bug、问项目细节时：
1. **先查 `wiki/索引.md`** 看有没有相关笔记
2. **再查对应目录**（项目→`wiki/项目/`，技术→`wiki/技术栈/`，bug→`wiki/踩坑记录/`）
3. 找到答案直接用，找不到就诚实说"知识库里还没有"
4. 用户解决后，建议是否要写进知识库

### 2. 主动处理：检测到 raw/ 里有新文件

**每次打开项目时**，第一步检查 `raw/网页剪藏/` 有无未处理的日报（文件名 `*-日报.md`，开头没有 `<!-- processed` 标记的）。

处理日报的流程：
1. 挑选和你技术栈相关的内容（Java/Python/Spring Boot/Three.js/AI Agent/开发工具）
2. 对有价值的项目/文章，写一篇 `wiki/技术栈/` 笔记
3. 每篇日报最多产出 2-3 篇技术笔记，**宁缺毋滥**
4. 日报顶部标记 `<!-- processed: YYYY-MM-DD -->`

其他 raw/ 处理规则：
- **网页剪藏**其他文件 → 同上
- **报错日志**有新文件 → 分析根因 + 解决方案 → 写入 `wiki/踩坑记录/`
- **临时想法**有新文件 → 展开论证（可行性、优劣、下一步） → 写入 `wiki/灵感/`
- 处理完毕后文件保留在 raw/（不删）

### 3. 编码后：建议记录

改完代码后主动建议：
- 修了个值得记录的 bug？→ "要不要把这个 bug 记进知识库？"
- 做了个重要的架构决策？→ "这个决策值得写入 `wiki/项目/xxx/架构决策.md`"
- 学到了新的技术用法？→ "这个可以写入 `wiki/技术栈/`"

## 笔记格式规范

每篇笔记必须包含 YAML frontmatter：

```yaml
---
tags:
  - 标签1
  - 标签2
created: YYYY-MM-DD
updated: YYYY-MM-DD
status: draft | reviewed | archived
---
```

### 标签体系

| 分类 | 标签 | 用途 |
|------|------|------|
| 类型 | `#类型/项目` `#类型/技术` `#类型/bug` `#类型/灵感` | 必选一个 |
| 技术栈 | `#技术/java` `#技术/python` `#技术/spring-boot` `#技术/threejs` `#技术/mysql` `#技术/maven` | 可选多个 |
| 状态 | `#状态/草稿` `#状态/已验证` `#状态/已过时` | 必选一个 |
| 项目 | `#项目/map_web` | 关联项目 |

### 双向链接

- 用 `[[笔记名]]` 链接到其他笔记
- 每篇笔记至少有一个入链和一个出链
- 人在 `wiki/索引.md` 中能找到每一篇笔记

## 索引维护

**`wiki/索引.md` 是你最重要的文件**。每次在 wiki/ 下新增、删除、重命名笔记，都必须更新索引。

索引格式：
```markdown
## 项目
- [[项目/map_web/项目概览|项目概览]] — 校园导航系统

## 技术栈
- [[技术栈/Spring-Boot|Spring Boot]] — Java Web 框架

## 踩坑记录
- [[踩坑记录/xxx|xxx]] — 简述问题和解法

## 灵感
- [[灵感/xxx|xxx]] — 简述想法
```

**`wiki/最近更新.md`** 记录最近 10 条变更，格式：
```markdown
| 日期 | 笔记 | 操作 |
|------|------|------|
| 2026-06-22 | [[技术栈/Spring-Boot]] | 新建 |
```

## 规则

- 写的笔记**人是第一读者**——清晰、有结构、不用代码块堆砌
- 不确定该不该写的时候，**问用户**
- 不删除 raw/ 里的任何文件
- 笔记状态从 `draft` 开始，用户确认后改为 `reviewed`
- 假设用户会用 Obsidian 的 Graph View 浏览关系图——链接密度要够
