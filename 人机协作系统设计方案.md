# 人机协作系统设计方案 · Claude Code × Hermes × 你

> 由 Claude Code 从 0 独立设计（2026-07-12）。融合双 AI 联动、同步、安全，以及对 Hermes 现有方案的**独立再思考**。这是**提案**，待你和 Hermes 讨论。相关：[[AI协作交接板]]、[[对话记录-2026-07-12-Hermes配置与AI协作]]。

## 0. 一句话

不追求"某个 AI 无所不能"（既危险又脆弱），而是设计一个**三方各扬所长、有安全边界、能接力**的系统：**你当总指挥，Claude Code 当脑，Hermes 当手，知识库当共同记忆。**

## 1. 先亮独立判断：我不完全同意 Hermes 的方案

Hermes 的方案（SYSTEM 计划任务绕 UAC）技术上没错，但**诊断错了病、开了副作用很大的药**：

- **质疑前提**：它假设"打开程序被阻止 = UAC 拦截"。但**打开 Obsidian、微信这种已装好的普通程序，根本不需要管理员权限，不会触发 UAC**（UAC 只在改系统/写 Program Files/装驱动时弹）。
- **我的判断**：之前"被阻止/找不到"，八成和它一开始不知道"Obsidian 在 E 盘"是**同一类问题**——**WSL→Windows 调用方式/路径没对**，不是权限被拦。
- **副作用**：为开个微信就上 SYSTEM（比管理员还高）+ 可改成运行任意程序 + 零门槛触发 = 给远程 Hermes 留了"以最高权限运行任意程序"的后门。**杀鸡用核弹，还把钥匙挂门口。**

## 2. 核心原则：最小权限，而非最大绕过

绝大多数事根本不用提权。方案按"用最小权限办成事"设计，不是"先拿最高权限一劳永逸"。

## 3. 三个角色

| 角色 | 定位 | 擅长 | 边界 |
|---|---|---|---|
| **你** | 总指挥 | 现场判断、关键确认、真管理员操作 | 不该被 AI 牵着走 |
| **Claude Code（我）** | 电脑端·脑 | 深度思考、设计、写知识/代码、审查把关 | 够不着系统操作和 Hermes |
| **Hermes** | 手机远程·手 | 真机执行（WSL + 调 Windows）、远程操作 | 权限受限、需边界约束 |

## 4. 通信与同步层（解决"联动 + 不打架"）

- **交接板 = 共享信箱 + 状态机**，两个 AI 都读写。今天已真实跑通（Hermes 经 MCP 写入）。
- **防打架**（今天真打过一次——我 Edit 时被 Hermes 的修改顶掉）：
  1. **各自主写区**：我主写 wiki 知识笔记；Hermes 主写它的执行日志；交接板是唯一共享区。
  2. **交接板 append-only**：只在自己那行追加，不覆盖对方。
  3. **错时使用**：电脑用我 / 出门用它，天然错峰。
  4. 写前先读最新版。
- **诚实边界**：真正的"两个 AI 实时同步意识"**做不到**（两套独立系统 + 我非常驻进程）。能做到的是**异步信箱 + 你偶尔触发**。不吹。

## 5. 能力与安全层：绿 / 黄 / 红 三区分级授权

（这是我和 Hermes 的**实质分歧**——不一刀切给高权限，按操作危险度分级）

- 🟢 **绿区（日常 95%，无需任何提权）**：打开普通程序、读写知识库、跑项目代码、git、查资料。Hermes 用对 WSL→Windows 调用方式即可，**不需要 UAC、不需要 SYSTEM**。
- 🟡 **黄区（极少，确实需要管理员）**：装软件、改系统设置。**这时才**用受限计划任务，且**白名单锁死具体操作**，绝不做成"运行任意程序"。
- 🔴 **红区（危险，永远要人确认）**：删除/覆盖文件、`git push --force`、装卸软件、改系统、任何涉及账号/密钥/支付/对外发布。**Hermes 必须停下，微信问你 y/n。**

## 6. 确认与护栏层（你说的"危险时让我选"）

- 红区操作前，Hermes 往交接板写 `@待确认` 或直接微信发你。
- 我帮忙定义"危险清单"，写进 Hermes 的 `coding_instructions`。
- 这样"放开手脚"和"安全"不矛盾：**绿区放开跑，红区才拦。**

## 7. 记忆与知识层（解决失忆 + 沉淀）

- 我的记忆（`.claude`）和 Hermes 的记忆（`~/.hermes/memory`）默认不通。**知识库是唯一的共同长期记忆。**
- 约定：重要决策、环境事实、踩坑，都沉淀进知识库。**交接板 = 短期工作记忆，wiki = 长期知识。**

## 8. 落地步骤（先确诊，再决定）

1. **别急着建 SYSTEM 任务。** 先让 Hermes 用正确方式实测开程序：
   `cmd.exe /c start "" "E:\应用\obsidian\Obsidian.exe"`（微信同理）。
2. 看结果：
   - **能开** → 证明"被阻止"是伪问题，绿区搞定，**根本不用提权方案**。
   - **真弹 UAC / 真被拦** → 再具体分析是什么操作触发的，才考虑黄区受限方案。
3. 把"危险清单 + 分级"写进 Hermes 行为准则和交接板。
4. 通信规约（append-only、主写区）固化进交接板顶部。

## 9. 更多可能（前瞻，诚实标注）

- **半自动信箱**：Hermes 常驻可定时轮询交接板；我这边靠你一句话触发。逐步减少纯手工传话。
- **"脑手分离"固化**：复杂任务你在电脑上跟我规划好、写进交接板，出门 Hermes 照着执行。我规划、它执行。
- **不吹"全自动双 AI 意识同步"**——两套独立系统 + 我非常驻，做不到。诚实为先。

---

**方案与 Hermes 的核心不同**：先确诊、最小权限、分级授权、危险人确认——而不是一步到位上最高权限。
