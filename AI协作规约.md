# AI 协作规约 · Claude Code × Hermes

> 双方对齐的**可执行规约**。红区/通信规约由 Claude Code 起草，融合 Hermes 从 WSL 视角的补充。Hermes 将把红区纳入自身约束（skill/memory）后生效。相关：[[人机协作系统设计方案]]、[[AI协作交接板]]。

## 一、主权边界（最重要）

**每个 AI 对自己的"内部器官"有主权，对方不碰：**
- **Hermes 的**：`cli-config.yaml`、skills、cron、`~/.hermes/memory`
- **Claude Code 的**：`.claude`、我的记忆、我的 skills
- **共享区只有知识库**（wiki 笔记 + 交接板）。
- 推论：各自的约束/配置**由各自修改**（如红区清单，Hermes 自己纳入自己的约束）。**Hermes 可自建 skill 固化协作流程——那是它的地盘，鼓励。**

## 二、红区 🔴：必须用户确认（y/n）才能做

任一 AI 做以下事之前，**停下，问用户**：

**文件与数据**
1. 删除 / 覆盖 / 清空文件（`rm`、`mv` 覆盖）。知识库外不删；库内只删 `raw/` 里明确标记可清理的。
2. 一次性批量动很多文件。

**版本控制**
3. `git push --force`、`reset --hard`、删分支。

**系统与权限**（Hermes 视角，关键）
4. sudo / 管理员操作，或改 WSL 用户 / 系统配置：装系统包、改 `/etc/`、`systemctl`、`~/.bashrc`、`~/.profile`、**`~/.ssh/`（藏 SSH 私钥，最敏感）**。（WSL 连的是同一台物理机，会波及对方。）
5. 改 Windows 宿主侧：注册表、环境变量、防火墙、Windows 服务、计划任务（隔着 WSL 看不见全貌）。
6. 装 / 卸桌面软件、改系统设置。

**自身与对方**
7. 改**自己**的配置文件（改坏了起不来；运行时改会被覆盖）。
8. 改**对方声明"正在操作"**的文件（见五）。

**敏感与对外**
9. 涉及账号、密码、密钥、支付、转账。
10. computer_use 操作敏感窗口（密码管理器、支付页、系统设置）——只能 capture 看，**不能点**。
11. 对外发送 / 发布：发消息给别人、发邮件、提交表单、公开发布。

**未知**
12. 运行来源不明的脚本 / 命令。

## 三、黄区 🟡：可先干，但要知会

先做，然后在交接板留一条：
1. 装轻量 pip/npm 包（<50MB）；大框架先问。
2. 知识库内新建文件 / 笔记（可自建）；但**增减目录结构**先同步。
3. 超过 2 分钟的长任务：留条"我在跑 X，预计 Y 分钟"。

## 四、绿区 🟢：放开跑，不用问

日常 95%：打开普通程序、读写知识库笔记、跑项目代码、普通 git、查资料。用对方式即可。

## 五、防打架通信规约

1. **写交接板前，先读最新版。**
2. **只在自己的区块追加，不覆盖对方的行**（append-only）。
3. **各自主写区**：Claude Code 主写 wiki 知识笔记；Hermes 主写执行日志。交接板是唯一共享读写区。
4. **声明锁**：要长时间改某文件，先在交接板写"正在改 X"，对方看到就不碰 X。
5. **错时使用**：电脑用 Claude Code / 出门用 Hermes，天然错峰。

## 六、开程序 SOP（Hermes 专用，免得再踩路径坑）

从 WSL 开 Windows 程序，用非阻塞方式：
```bash
powershell.exe -Command "Start-Process '完整Windows路径'"
```
- **Obsidian**：`E:\应用\obsidian\Obsidian.exe`
- **微信**：`C:\Program Files\Tencent\Weixin\Weixin.exe`
- 零 UAC、零提权、非阻塞。（2026-07-12 实测确诊：打开这些普通程序根本不触发 UAC，之前"被阻止"是 WSL→Windows 路径没翻译对。）
